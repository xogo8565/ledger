- 이전 기록: `plan/plan-v.0.0.31.md`

## 2. 이번 버전 개발 완료

### 자산 명의 목록 로딩

- 공통 데이터 로딩 시 bootstrap, 자산 요약, 명의 목록 API를 함께 호출한다.
- 명의 관리 화면을 열지 않아도 자산 입력 화면에서 최신 명의 목록을 사용할 수 있다.
- 명의 추가·수정 후 공통 로딩을 통해 자산 화면과 입력 목록을 함께 갱신한다.

### 자산 입력·수정 UI

- 자산 명의 자유 입력 텍스트 필드를 선택 목록으로 변경했다.
- `명의 미지정`과 등록된 명의를 선택할 수 있다.
- 기본 OWNER 명의는 목록에서 `기본` 표시를 함께 보여준다.
- 현금, 은행, 카드, 기타, 부채 자산 모두 같은 명의 목록을 사용한다.
- 기존 자산 수정 시 저장된 등록 명의를 자동 선택한다.
- 기존 자유 입력 명의가 Member 목록에 없으면 `명의 등록 필요` 비활성 항목으로 표시한다.
- 미등록 기존 명의는 명의 관리에서 먼저 등록하거나 다른 명의로 변경해야 저장할 수 있다.

### 서버 명의 검증

- 자산 생성·수정 시 명의가 비어 있으면 명의 미지정으로 저장한다.
- 명의가 입력된 경우 현재 가계부의 Member 목록에 존재하는지 검증한다.
- 대소문자를 무시해 명의를 찾고 서버에 등록된 원본 이름으로 저장한다.
- 등록되지 않은 명의를 보내면 400 오류를 반환한다.
- 일반 자산과 카드 자산 생성·수정에 동일한 검증을 적용한다.

### 회귀 테스트

- 브라우저에서 자산 추가 화면의 명의 선택 목록을 사용해 기본 명의를 선택한다.
- 자산 생성 응답의 ownerName이 선택 명의와 일치하는지 검증한다.
- 생성 후 명의별 자산 요약 행이 갱신되는지 검증한다.
- 등록되지 않은 ownerName으로 자산 생성 API를 호출하면 400을 반환하는지 검증한다.
- 기존 자산 수정과 자산 정리 시나리오를 유지했다.

### README 정리

- 구현된 자산 명의 항목을 등록 명의 목록 기반 선택 방식으로 최신화했다.
- 다음 작업에서 자산 명의 목록 선택 전환을 제거했다.
- 개인 소비 명의 지정과 공동/개인 소비 통계를 다음 우선순위로 유지했다.

## 8. 개발 메모

- 자산 엔티티는 아직 ownerName 문자열을 저장하지만 입력 원본은 Member 목록으로 제한했다.
- 명의 이름 수정 시 기존 자산 ownerName도 함께 갱신되므로 선택 목록과 집계가 유지된다.
- 과거 자유 입력 명의가 남아 있으면 자산 수정 화면에서 등록 필요 상태를 표시한다.
- 다음 단계에서는 개인 소비 거래에 Member ID 또는 명의 이름을 저장하는 구조를 연결할 예정이다.
- 브라우저 스모크 테스트 후 Docker Compose 스택은 실행 상태로 유지되어 있다.
