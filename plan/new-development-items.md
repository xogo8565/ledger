- v0.1 라인에서는 파기했지만, 신규 개발항목으로 재추가한다.

1. OCR 요청 처리 시간 측정 로그 추가 - 완료
2. 시간 초과 기준과 사용자 안내 메시지 정의 - 완료, 30초 기준 유지
3. OCR 실패/timeout 케이스 집계 기준 검토 - 부분 완료, 로그 status 기준 정의
4. 운영 환경에서 OCR 처리 지연을 확인할 수 있는 로그 포맷 정리 - 완료

### P3. 영수증 품목별 상세 저장 모델 검토

출처:

- `plan-v.0.1.10.md`

현재 상태:

- 현재 OCR은 거래 초안과 품목 요약 메모 중심으로 처리한다.

- 품목 상세 저장 모델은 `receipt_items` 신규 테이블로 분리하는 설계안을 정리했다.

신규 개발 범위:

1. 품목명, 수량, 단가, 금액 저장 모델 검토 - 완료, `receipt_items` 설계안 정리
2. 거래 1건과 영수증 품목 N건의 관계 설계 - 완료, `TransactionRecord 1:N ReceiptItem` 우선

4. OCR 품목 후보와 실제 저장 품목의 수정 UX 검토 - 부분 완료, 저장 전 검토/수정 화면 필요

2. 재분석 API에도 후보 DTO와 정책 DTO를 반환하도록 확장 검토 - 완료
