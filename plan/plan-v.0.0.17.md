# 가계부 앱 개발 진행 기록 v.0.0.17

작성일: 2026-06-23

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.16.md`
- 이번 개발 범위: v.0.0.17 권장 항목 중 연별 통계와 카테고리 선택 상세 이동
- 검증 방식: Docker Compose 컨테이너 빌드, Gradle 테스트 컨테이너 실행

## 2. 이번 버전 개발 완료

### 연별 통계 API

- `GET /api/summary/yearly?year=YYYY` API를 추가했다.
- 연간 수입, 지출, 이체 합계를 반환한다.
- 1월부터 12월까지 월별 수입, 지출, 이체 합계를 반환한다.
- 연간 카테고리별 지출 랭킹을 반환한다.
- 기존 월별 카테고리 지출 응답에 `categoryId`를 포함해 상세 이동 필터와 연결할 수 있게 했다.

### 통계 화면 연별 보기

- 통계 화면의 기간 버튼을 월별/연별 전환 버튼으로 동작하게 했다.
- 연별 보기에서 선택 월의 연도 기준 총 지출과 12개월 지출 막대를 표시한다.
- 연별 보기에서도 카테고리별 지출 랭킹을 표시한다.
- 월을 바꾸면 해당 연도의 연별 통계가 다시 로드된다.

### 카테고리 선택 상세 이동

- 통계 카테고리 랭킹 행을 선택 가능한 버튼으로 변경했다.
- 카테고리를 누르면 가계부 탭으로 이동한다.
- 거래 목록 필터를 지출 유형과 선택 카테고리로 자동 설정한다.
- 기존 월별 거래 목록/검색/필터 UI를 재사용한다.

### README 정리

- 구현된 MVP 목록에 월별/연별 통계와 통계 카테고리 선택 후 거래 목록 이동을 반영했다.
- 다음 작업 목록에서 연별 통계와 카테고리 상세 이동 항목을 제거했다.
- 다음 우선순위를 데이터 내보내기로 갱신했다.

## 5. 아직 부분 완료인 항목

### 통계

- 월별/연별 통계와 카테고리 선택 상세 이동은 구현되어 있다.
- 사용자 지정 시작일/종료일 기반 기간 필터는 아직 미완료다.
- 연별 보기에서 특정 월 막대를 눌러 해당 월로 이동하는 흐름은 추가 개선 여지가 있다.

### 카드 관리

- 2025~2027년 공휴일 캘린더는 코드 내 정적 데이터로 반영되어 있다.
- 2028년 이후 공휴일은 데이터 갱신 또는 외부 캘린더 연동이 필요하다.
- 카드 결제 예정 금액과 할부 청구 월 연결 검증이 필요하다.

### 영수증 첨부

- 신규 거래 첨부와 업로드 API는 구현되어 있다.
- 첨부 이미지 조회, 미리보기, 삭제 UI는 구현되어 있다.
- 기존 거래 수정 중 신규 영수증 추가와 다중 첨부 흐름 고도화는 아직 미완료다.

### 문자 자동 입력

- 원문 붙여넣기, 후보 추출, 거래 입력 폼 확정 이동은 구현되어 있다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

## 8. 개발 메모

- 연별 통계는 선택된 월의 연도를 기준으로 조회한다.
- 카테고리 상세 이동은 현재 월 거래 목록 필터를 재사용한다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드와 Gradle 테스트는 정상 통과했다.
