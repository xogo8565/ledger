# 편한가계부 후속 개발 진행 기록 v.0.0.37

작성일: 2026-06-24

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.36.md`

- 검증 방식: Gradle 전체 테스트, Docker Compose 이미지 빌드, Playwright 브라우저 스모크 테스트

## 2. 이번 버전 개발 완료

### 사용자 지정 기간 통계 API

- `GET /api/summary/range` API를 추가했다.
- `startDate`, `endDate` 기준으로 기간 내 거래를 집계한다.
- 기간 요약 응답은 기간 라벨, 시작일, 종료일, 수입 합계, 지출 합계, 이체 합계를 제공한다.
- 기간 지출의 카테고리별, 소비 태그별, 개인/공동 소비 구분별, 소비 명의별 통계를 함께 제공한다.
- 종료일이 시작일보다 빠르면 400 오류로 처리한다.

### 통계 화면 기간 모드

- 통계 탭에 `월별`, `연별`, `기간` 전환을 추가했다.
- 기간 모드에서는 시작일과 종료일을 직접 선택할 수 있다.
- 선택한 기간의 지출 총액과 카테고리·소비 태그·개인/공동·명의별 랭킹을 표시한다.
- 기간 모드는 통계 탭에만 적용하고, 예산·내용 탭은 기존 월 기준 화면을 유지한다.
- 잘못된 기간을 선택하면 통계 요청을 중단하고 기간 확인 안내를 표시한다.

### 집계 로직 정리

- 월별·연별·기간 통계가 같은 카테고리 집계 helper를 사용하도록 정리했다.
- 기간 통계 단위 테스트를 추가해 총액과 지출 분해 집계를 검증했다.

### 테스트 및 스모크 보강

- 브라우저 스모크 테스트에서 기간 통계 API가 생성한 테스트 거래를 포함하는지 검증한다.
- 통계 화면에서 `기간` 탭과 날짜 입력, 기간 통계 카테고리 랭킹 표시를 검증한다.

## 4. 아직 부분 완료인 항목

### 기존 개인 지출 데이터

- 소비 명의가 없는 기존 개인 지출은 명의별 통계에서 `명의 미지정`으로 집계된다.
- OWNER 명의로 일괄 연결하는 마이그레이션 정책은 아직 미완료다.

### 예산·주간 통계

- 예산 화면은 아직 월별 예산 중심이다.
- 연간 예산 화면에서 연도 이동을 기본으로 하는 흐름과 주간별 통계는 아직 미완료다.

### 기타 후속 항목

- 프론트엔드 화면 단위 컴포넌트 분리는 아직 미완료다.
- 영수증 첨부 파일 정리 정책 고도화는 아직 미완료다.

## 7. 개발 메모

- 기간 통계는 예산·자산 현재값과 분리된 순수 거래 기간 집계로 구현했다.
- 기간 모드는 통계 탭에만 적용해 예산 화면의 월 기준 정책과 섞이지 않게 했다.
- 기간 통계의 랭킹 클릭은 기존 카테고리·태그·소비구분·명의별 가계부 필터 이동 흐름을 그대로 재사용한다.
