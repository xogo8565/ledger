작성일: 2026-06-24

## 2. 이번 버전 개발 완료

### 명의별 소비 통계 API

- 월별 요약 응답에 `memberSpends`를 추가했다.
- 연별 요약 응답에도 같은 명의별 소비 집계를 추가했다.
- 각 집계는 소비 명의 ID, 소비 명의 이름, 지출 합계, 거래 건수를 제공한다.
- 개인 소비 지출만 명의별 통계에 포함하고 공동 소비, 수입, 이체는 제외한다.
- 소비 명의가 없는 기존 개인 지출은 `명의 미지정` 항목으로 집계한다.
- 지출 금액 내림차순, 거래 건수 내림차순, 이름순으로 명의별 항목을 정렬한다.

### 월별·연별 통계 UI

- 지출 통계 기준에 `명의별` 탭을 추가했다.
- 월별 통계와 연별 통계에서 동일한 명의별 소비 랭킹을 확인할 수 있다.
- 명의별 항목에는 지출 비율, 거래 건수, 금액을 표시한다.
- 데이터가 없으면 전용 빈 상태 안내를 표시한다.

### 거래 필터 연동

- 가계부 필터에 `소비 명의` 선택 항목을 추가했다.
- 명의별 통계 항목을 누르면 지출 유형, 개인 소비, 해당 소비 명의 필터가 자동 선택되어 거래 목록으로 이동한다.
- 소비 명의 필터를 직접 선택하면 지출 유형과 개인 소비 필터를 함께 적용한다.
- 수입, 이체, 공동 소비 필터를 선택하면 소비 명의 필터를 자동으로 비운다.

### 테스트 및 스모크 보강

- 개인 소비 지출만 명의별로 집계하고 공동 소비와 수입을 제외하는 단위 테스트를 추가했다.
- 월별 API가 선택한 소비 명의를 `memberSpends`에 포함하는지 브라우저 테스트에서 검증한다.
- 통계 화면의 `명의별` 항목 선택 시 가계부의 소비 명의 필터가 적용되는지 검증한다.
- 브라우저 스모크 테스트가 빈 자산/카테고리 데이터베이스에서도 최소 시드 데이터를 만들고 종료 시 정리하도록 보강했다.

## 4. 아직 부분 완료인 항목

### 기존 개인 지출 데이터

- 소비 명의가 없는 기존 개인 지출은 명의별 통계에서 `명의 미지정`으로 집계된다.

- 할부 그룹 전체 수정 시 여러 회차에 첨부 파일을 어떻게 배분할지에 대한 정책과 UI는 아직 미완료다.

### 기타 후속 항목

- 예산 관련 연간 이동 기본값과 주간별 통계는 미완료다.

## 7. 개발 메모

- 명의별 소비 통계는 `TransactionRecord.getConsumer()`를 기준으로 집계하므로 명의 이름 수정 시 최신 이름을 사용한다.
- 기존 데이터 호환성을 위해 소비 명의가 없는 개인 지출은 `명의 미지정`으로 남겨 두었다.
- 로컬 `npm run build`는 `frontend/node_modules`가 없어 직접 실행하지 못했으며, Docker 이미지 빌드 안의 `npm run build`로 프론트엔드 빌드를 검증했다.
