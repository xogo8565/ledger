# 가계부 앱 개발 진행 기록 v.0.0.24

작성일: 2026-06-24

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.23.md`
- 이번 개발 범위: v.0.0.24 권장 항목 중 소비 태그별 통계
- 검증 방식: Docker Compose 컨테이너 빌드, Gradle 테스트 컨테이너 실행, 요약 API 확인, 브라우저 스모크 테스트

## 2. 이번 버전 개발 완료

### 소비 태그 통계 집계

- 월 요약과 연 요약 응답에 `tagSpends` 목록을 추가했다.
- 지출 거래 중 소비 태그가 있는 거래만 집계한다.
- 쉼표로 구분한 여러 태그를 개별 태그로 정규화해 집계한다.
- 태그 앞의 `#`, 앞뒤 공백, 중복 공백을 제거한다.
- 한 거래 안에서 같은 태그가 중복 입력된 경우 한 번만 집계한다.
- 대소문자만 다른 태그는 동일 태그로 합산한다.
- 각 태그 응답에 태그명, 지출 합계, 거래 건수를 포함한다.
- 지출 합계, 거래 건수, 태그명 순으로 정렬한다.

### 통계 화면

- 통계 탭에 `카테고리`와 `소비 태그` 기준 전환 버튼을 추가했다.
- 월별·연별 화면 모두 소비 태그별 지출 합계와 거래 건수를 표시한다.
- 각 태그의 비율은 해당 기간 전체 지출 대비 비율로 표시한다.
- 소비 태그 통계가 없으면 전용 빈 상태를 표시한다.
- 태그 항목을 누르면 가계부 화면으로 이동해 지출 유형과 해당 태그 검색어를 자동 적용한다.
- 기존 카테고리 통계 및 카테고리 선택 상세 이동은 유지했다.

### 회귀 테스트

- 쉼표 구분 태그 집계를 검증하는 `LedgerServiceTagStatsTest`를 추가했다.
- 수입 거래와 빈 태그가 제외되는지 검증했다.
- 한 거래의 중복 태그와 `#` 접두어가 정규화되는지 검증했다.
- 태그별 금액 합계와 거래 건수 정렬을 검증했다.

### README 정리

- 구현된 MVP의 통계를 카테고리별·소비 태그별 통계로 최신화했다.
- 다음 작업에서 소비 태그별 통계를 제거했다.
- v0.0.23 기록에 추가된 거래 입력 방식 선택 UX를 다음 작업에 반영했다.

## 5. 아직 부분 완료인 항목

### 영수증 첨부

- 신규 거래 첨부, 기존 거래 수정 중 첨부, 조회, 미리보기, 삭제 UI는 구현되어 있다.
- 다중 첨부 일괄 업로드와 할부 그룹 수정 중 첨부 흐름은 아직 미완료다.

### 문자 자동 입력

- 카드 승인, 체크카드 출금, 은행 입금, 취소/환불 기본 규칙은 구현되어 있다.
- 문자 파싱 기반 카테고리 추천과 사용자 정의 파싱 규칙 관리는 아직 미완료다.
- 거래 추가 버튼에서 직접 입력/문자 자동 입력을 선택하는 진입 UX는 아직 미완료다.

### 통계

- 월별/연별 카테고리 및 소비 태그 통계는 구현되어 있다.
- 기간 필터 기준 요약 금액 재계산은 아직 미완료다.
- 다중 태그 거래는 태그 분석마다 거래 전액을 반영하므로 태그 비율 합계가 100%를 넘을 수 있다.

### 자산 명의/공동 가계부

- 자산 명의 저장과 표시, Household/Member 도메인은 구현되어 있다.
- 명의별 자산 집계와 공동 가계부 초대/권한 UI는 아직 미완료다.

## 8. 개발 메모

- 소비 태그는 별도 마스터 테이블 없이 기존 문자열 필드를 집계한다.
- 쉼표 구분 다중 태그는 각 태그에 거래 전액을 반영한다.
- 태그 클릭 상세 이동은 기존 가계부 검색 기능을 재사용한다.
- 브라우저 스모크 테스트 후 Docker Compose 스택은 실행 상태로 유지되어 있다.
