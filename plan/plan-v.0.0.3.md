# 가계부 앱 개발 진행 기록 v.0.0.3

작성일: 2026-06-22

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.2.md`
- 이번 개발 범위: 1차 MVP 잔여 항목 중 거래 상세/수정/삭제 UI, 카드 자산 생성 UI, README 한글 문서 정리, 빌드 검증
- 검증 방식: Docker Compose 컨테이너 빌드

## 2. 이번 버전 개발 완료

### 거래 상세 화면

- 가계부 일일 목록과 메모 목록의 거래 행을 선택할 수 있도록 변경했다.
- 거래 상세 패널을 추가했다.
- 상세 화면 표시 항목:
  - 유형
  - 날짜
  - 분류
  - 자산
  - 금액
  - 제목/내용
  - 메모
  - 할부 정보
- 할부 거래는 상세 화면에서 할부 내역 화면으로 이동할 수 있다.

### 거래 수정 UI

- 거래 상세 화면에서 수정 버튼을 누르면 기존 거래 입력 화면을 수정 모드로 재사용한다.
- 수정 폼에 기존 거래 정보를 채운다.
  - 거래 유형
  - 거래일
  - 금액
  - 카테고리
  - 자산
  - 이체 출금/입금 자산
  - 내용
  - 메모
  - 할부 개월 수
- 수정 모드 저장 시 `PUT /api/transactions/{id}`를 호출한다.
- 수정 모드에서는 신규 영수증 첨부와 이체 수수료 자동 추가를 비활성화했다.

### 거래 삭제 UI

- 거래 상세 화면에 삭제 버튼을 추가했다.
- 삭제 전 브라우저 확인창을 표시해 즉시 삭제 실수를 줄이도록 했다.
- 삭제 시 `DELETE /api/transactions/{id}`를 호출한다.
- 삭제 후 상세 패널을 닫고 월별 데이터를 다시 불러온다.

### 카드 자산 생성 UI

- 자산 추가 화면에서 유형을 `카드`로 선택하면 카드 결제 설정 영역이 표시되도록 했다.
- 카드 생성 시 결제 계좌, 결제금액 확정일, 결제일, 자동 결제 여부를 입력할 수 있다.
- 결제 계좌 선택 목록은 카드와 부채를 제외한 일반 자산만 표시한다.
- 신규 카드 저장 시 일반 자산 API가 아니라 `POST /api/assets/card`를 호출해 `CardProfile`까지 함께 생성한다.
- 카드 설정 입력으로 폼이 길어져도 저장 버튼이 밀리지 않도록 자산 폼 입력 영역에 스크롤을 적용했다.

### README 정리

- 기존 README의 한글 인코딩 깨짐을 수정했다.
- Docker 실행, 로컬 개발 실행, 구현된 MVP, 다음 작업, 스모크 테스트 설명을 현재 상태 기준으로 다시 정리했다.

### 빌드 검증

- `docker compose build frontend` 통과
- `docker compose build backend` 통과
- `docker compose build --no-cache frontend` 통과

## 5. 아직 부분 완료인 항목

### 카드 관리

- 카드 결제 예약과 실행은 구현되어 있다.
- 신규 카드 자산 생성 시 카드 프로필 입력과 생성 API 연결은 구현되어 있다.
- 기존 카드 자산의 카드 프로필 수정 UI는 아직 미완료다.
- 실제 청구 기간, 말일 처리, 휴일 처리, 중복 자동 결제 방지 정책은 보강이 필요하다.
- 카드 결제 예정 금액을 사용자가 이해하기 쉽게 보여주는 UI 설명도 더 필요하다.

### 할부 관리

- 할부 생성과 조회는 구현되어 있다.
- 할부 그룹 전체 수정/삭제는 아직 미완료다.
- 카드 결제 예정 금액과 할부 청구 월 연결 검증이 필요하다.

### 예산

- 월별 전체/카테고리 예산 저장은 구현되어 있다.
- 카테고리별 예산 사용률, 초과 표시, 전월 예산 복사는 미완료다.

### 통계

- 월별 및 카테고리별 지출 통계는 구현되어 있다.
- 연별 통계, 기간 필터, 카테고리 선택 후 거래 목록 이동은 미완료다.

### 영수증 첨부

- 신규 거래 첨부와 업로드 API는 구현되어 있다.
- 첨부 이미지 조회, 미리보기, 삭제, 다중 첨부 UI는 미완료다.

### 문자 자동 입력

- 원문 붙여넣기와 후보 추출은 구현되어 있다.
- 파싱 결과를 거래 입력 폼으로 확정 저장하는 흐름은 미완료다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

### 공동 가계부

- Household/Member 도메인은 있다.
- 인증, 초대, 권한, 사용자 전환 UI는 미완료다.

## 7. 개발 메모

- 프론트엔드는 여전히 `frontend/src/main.jsx` 단일 파일 비중이 크다. 다음 기능 확장 전에 화면 단위 컴포넌트 분리를 고려하는 것이 좋다.
- 거래 수정 모드에서는 영수증 재첨부를 우선 제외했다. 기존 첨부 관리 기능이 추가될 때 함께 정리하는 편이 안전하다.
- 거래 삭제는 확인창을 거치도록 개선했다. 이후 앱 디자인에 맞는 커스텀 확인 모달 또는 실행 취소 토스트로 바꾸면 더 좋다.
- Docker 빌드는 통과했지만, 실제 브라우저 스모크 테스트는 이번 작업에서 실행하지 않았다.
- 프론트 no-cache 빌드 중 `npm audit` 기준 moderate 1건, high 1건 취약점 알림이 표시되었다. 의존성 점검이 필요하다.
