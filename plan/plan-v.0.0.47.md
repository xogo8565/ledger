# 편한가계부 후속 개발 진행 기록 v.0.0.47

작성일: 2026-06-25

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.46.md`
- 이번 개발 범위: v.0.0.47 권장 항목 중 예산 설정 화면과 하단 탐색·앱 셸 컴포넌트 분리
- 검증 방식: Gradle 전체 테스트, Docker Compose 이미지 빌드, Playwright 브라우저 스모크 테스트

## 2. 이번 버전 개발 완료

### 예산 설정 화면 분리

- `frontend/src/screens/BudgetSettingsScreen.jsx`를 추가했다.
- 월 이동, 전월 예산 복사, 전체 예산 입력, 카테고리별 예산 입력과 저장 UI를 화면 모듈로 이동했다.
- 공통 `AppHeader`, `BackButton`과 원장의 `MonthNav`를 재사용해 기존 동작과 화면 구조를 유지했다.
- App은 예산 상태와 저장·복사 callback만 화면에 전달하도록 정리했다.

### 앱 셸과 하단 탐색 분리

- `frontend/src/components/AppShell.jsx`를 추가했다.
- 페이지 프레임, 휴대폰 셸, 하단 탭 탐색, 거래 추가 FAB를 앱 셸 모듈로 이동했다.
- 메인 탭 화면은 `content` 슬롯에, 전체 화면 패널은 children 영역에 배치해 기존 DOM 순서와 패널 겹침 순서를 유지했다.
- 하단 탭 정의를 셸 내부 상수로 모아 App의 표시 책임을 줄였다.

### App import와 렌더 책임 정리

- `main.jsx`에서 예산 화면 전용 `AppHeader`, `BackButton`, `MonthNav` import를 제거했다.
- `main.jsx`는 메인 탭별 데이터·callback 연결과 전체 화면 패널 전환을 담당하고, 공통 외곽 레이아웃은 `AppShell`에 위임한다.
- 기존 예산 설정 화면과 `BottomNav` 지역 컴포넌트 정의를 제거했다.

### 파일 규모 변화

- `main.jsx`는 1,151줄에서 1,096줄로 감소했다.
- 앱 셸 47줄과 예산 설정 화면 54줄을 독립 모듈로 구성했다.
- 프론트엔드 프로덕션 빌드 모듈 수는 39개에서 41개로 증가했다.

## 4. 아직 부분 완료인 항목

### 프론트엔드 상태와 API 구조

- `main.jsx`가 여러 도메인의 API 호출, 로딩 상태와 패널 전환을 함께 관리한다.
- 거래, 예산, 자산, 일정별 API 모듈과 custom hook으로 책임을 나눌 필요가 있다.
- API 오류 처리와 재시도 정책도 도메인별로 일관되게 정리할 필요가 있다.

### 백엔드 구조

- `LedgerService`가 거래, 검색, 예산, 통계, 명의와 자산 관리 책임을 함께 담당한다.
- 도메인별 서비스 분리와 검색 조건 객체 분리가 필요하다.

### 기타 후속 항목

- 검색 결과 페이지네이션과 정렬 선택은 아직 미완료다.
- 영수증 첨부 파일 정리 정책 고도화는 아직 미완료다.

## 7. 개발 메모

- 셸의 `content`와 children을 분리해 메인 콘텐츠, 하단 탐색·FAB, 전체 화면 패널의 기존 렌더 순서를 보존했다.
- 예산 화면은 상태를 소유하지 않고 전달받은 값과 callback만 사용하므로 이후 예산 전용 hook을 도입하기 쉽다.
- 화면 컴포넌트 분리 단계가 마무리되어 다음 버전부터는 API와 상태 조정 책임을 줄이는 작업에 집중할 수 있다.
