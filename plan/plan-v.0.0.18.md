## 2. 이번 버전 개발 완료

### 거래 CSV 내보내기 API

- `GET /api/export/transactions.csv` API를 추가했다.
- `month=YYYY-MM` 파라미터로 월별 거래 CSV를 내려받을 수 있게 했다.
- `year=YYYY` 파라미터로 연별 거래 CSV도 내려받을 수 있게 했다.
- CSV 컬럼은 기간, 거래일, 유형, 금액, 카테고리, 자산, 출금자산, 입금자산, 제목, 메모, 할부회차, 할부개월로 구성했다.
- CSV 셀은 따옴표 이스케이프를 적용했다.
- 엑셀 호환성을 위해 UTF-8 BOM과 `Content-Disposition` 다운로드 헤더를 추가했다.

### 프론트엔드 다운로드 흐름

- 선택 월 기준 거래 CSV 다운로드 함수를 추가했다.
- 가계부 요약 화면의 기존 내보내기 버튼을 실제 월 거래 CSV 다운로드로 연결했다.
- 더보기 화면의 백업/내보내기 항목을 월 거래 CSV 내보내기 버튼으로 연결했다.
- 다운로드 실패 시 사용자 알림을 표시하도록 했다.

### README 정리

- 구현된 MVP 목록에 월별 거래 CSV 내보내기를 추가했다.
- 다음 작업 목록에서 데이터 내보내기를 제거했다.
- 다음 우선순위를 자산 명의/소비 태그 구조로 갱신했다.

## 5. 아직 부분 완료인 항목

### 데이터 내보내기

- 월별/연별 거래 CSV 내보내기는 구현되어 있다.
- 예산, 자산, 카테고리, 통계 요약의 통합 내보내기는 아직 미완료다.
- 필터 결과 그대로 내보내기와 날짜 범위 직접 선택은 추가 개선 여지가 있다.

### 통계

- 월별/연별 통계와 카테고리 선택 상세 이동은 구현되어 있다.
- 사용자 지정 시작일/종료일 기반 기간 필터는 아직 미완료다.
- 연별 보기에서 특정 월 막대를 눌러 해당 월로 이동하는 흐름은 추가 개선 여지가 있다.

### 카드 관리

- 2025~2027년 공휴일 캘린더는 코드 내 정적 데이터로 반영되어 있다.
- 2028년 이후 공휴일은 데이터 갱신 또는 외부 캘린더 연동이 필요하다.
- 카드 결제 예정 금액과 할부 청구 월 연결 검증이 필요하다.

### 영수증 첨부

- 신규 거래 첨부와 업로드 API는 구현되어 있다.
- 첨부 이미지 조회, 미리보기, 삭제 UI는 구현되어 있다.
- 기존 거래 수정 중 신규 영수증 추가와 다중 첨부 흐름 고도화는 아직 미완료다.

### 문자 자동 입력

- 원문 붙여넣기, 후보 추출, 거래 입력 폼 확정 이동은 구현되어 있다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

## 8. 개발 메모

- 프론트엔드 다운로드는 현재 선택 월 기준 CSV를 내려받는다.
- 연별 CSV API는 백엔드에서 지원하지만 아직 별도 UI는 없다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드와 Gradle 테스트는 정상 통과했다.
