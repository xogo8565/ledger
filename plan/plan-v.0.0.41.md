# 편한가계부 후속 개발 진행 기록 v.0.0.41

작성일: 2026-06-25

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.40.md`
- 이번 개발 범위: v.0.0.41 권장 항목 1순위인 프론트엔드 화면 단위 컴포넌트 분리 1차
- 검증 방식: Gradle 전체 테스트, Docker Compose 이미지 빌드, Playwright 브라우저 스모크 테스트

## 2. 이번 버전 개발 완료

### 공통 UI 컴포넌트 분리

- `frontend/src/components/ui.jsx`를 추가했다.
- 다음 공통 UI를 `main.jsx`에서 분리했다.
  - `AppHeader`
  - `BackButton`
  - `IconButton`
  - `EmptyState`
- 기존 화면은 공통 UI 모듈을 import해 같은 마크업과 CSS class를 사용한다.
- 시각 스타일과 이벤트 인터페이스를 유지해 기능 변경 없이 재사용 구조를 만들었다.

### 더보기 화면 모듈 분리

- `frontend/src/screens/MoreScreens.jsx`를 추가했다.
- 하단 탭의 `MoreScreen`을 별도 화면 모듈로 이동했다.
- 카테고리 관리, 명의 관리, 반복 거래, CSV 내보내기 진입 콜백은 기존처럼 상위 App에서 전달한다.

### 명의 관리 화면 모듈 분리

- `MemberManagerScreen`을 `MoreScreens.jsx`로 이동했다.
- 명의 추가·수정·삭제와 기존 개인 지출 OWNER 명의 연결 UI를 함께 화면 모듈로 묶었다.
- 서버 요청과 상태 갱신은 App에 유지하고 화면은 props 기반으로 렌더링과 사용자 이벤트를 담당한다.

### 카테고리 관리 화면 모듈 분리

- `CategoryManagerScreen`을 `MoreScreens.jsx`로 이동했다.
- 지출·수입 전환, 카테고리 추가·수정·삭제 UI를 화면 모듈로 묶었다.
- 새 카테고리 폼 초기화는 상위에서 전달한 callback을 사용해 기존 폼 기본값 정책을 유지한다.

### 디렉터리 구조 정리

- 재사용 UI는 `components`, 화면 단위 UI는 `screens`에 배치하는 기준을 세웠다.
- `main.jsx`는 애플리케이션 상태, API 요청, 화면 전환과 조합 역할을 유지한다.
- 이번 분리로 `main.jsx`에서 공통 UI 4개와 화면 컴포넌트 3개 정의를 제거했다.

## 4. 아직 부분 완료인 항목

### 프론트엔드 구조

- 가계부, 통계, 자산, 거래 입력 화면은 아직 `main.jsx`에 남아 있다.
- 화면별 helper와 표시 전용 하위 컴포넌트도 각 화면 모듈로 함께 옮길 필요가 있다.
- API 요청과 상태 로직을 custom hook 또는 별도 API 모듈로 분리하는 작업은 아직 미완료다.

### 백엔드 구조

- `LedgerService`가 거래, 검색, 예산, 통계, 명의와 자산 관리 책임을 함께 담당한다.
- 도메인별 서비스 분리와 검색 조건 객체 분리가 필요하다.

### 기타 후속 항목

- 검색 결과 페이지네이션과 정렬 선택은 아직 미완료다.
- 영수증 첨부 파일 정리 정책 고도화는 아직 미완료다.

## 7. 개발 메모

- 화면 분리는 API 호출까지 한 번에 이동하지 않고, 먼저 표시 컴포넌트를 props 기반으로 분리하는 방식으로 진행했다.
- 이 방식은 기존 App 상태 흐름을 유지해 회귀 위험을 낮추면서 이후 hook·API 모듈 분리를 위한 경계를 만든다.
- CSS class와 스타일 파일은 이번 버전에서 변경하지 않아 화면 모듈 이동 전후의 시각적 결과를 동일하게 유지했다.
