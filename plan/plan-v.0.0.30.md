작성일: 2026-06-24

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.29.md`
- 이번 개발 범위: v.0.0.30 권장 항목 중 거래 공동/개인 소비 구분
- 검증 방식: Docker Compose 컨테이너 빌드, Gradle 테스트 컨테이너 실행, 공동 소비 거래 생성 브라우저 회귀 테스트

## 2. 이번 버전 개발 완료

### 소비 구분 도메인

- `ConsumptionScope` 열거형을 추가했다.
- 소비 구분 값은 `PERSONAL`과 `SHARED`로 구성한다.
- 지출 거래에 소비 구분 필드를 저장한다.
- 신규 지출의 기본값은 개인 소비다.
- 기존 지출 데이터처럼 필드가 비어 있는 경우에도 조회 시 개인 소비로 해석한다.
- 수입과 이체 거래는 소비 구분을 저장하지 않고 응답에서도 비워 둔다.

### 거래 API

- 거래 생성/수정 요청에 `consumptionScope` 필드를 추가했다.
- 거래 조회 응답에 소비 구분을 포함한다.
- 일반 거래, 할부 생성, 할부 그룹 전체 수정에 소비 구분을 반영한다.
- 카드 자동 결제 이체와 반복 거래 자동 생성은 기존 호환성을 유지하며 소비 구분을 비워 둔다.
- Hibernate 자동 스키마 갱신으로 기존 transactions 테이블에 소비 구분 열을 추가한다.

### 거래 입력·수정 UI

- 지출 입력 화면에 `소비 구분` 선택 항목을 추가했다.
- 개인 소비와 공동 소비 중 하나를 선택할 수 있다.
- 신규 지출은 개인 소비가 기본 선택된다.
- 기존 거래 수정 시 저장된 소비 구분을 불러온다.
- 거래 유형을 수입/이체로 변경하면 소비 구분을 개인 기본값으로 초기화하고 서버에는 저장하지 않는다.
- 할부 전체 수정에서도 선택한 소비 구분을 모든 회차에 반영한다.

### 목록·상세·검색·CSV

- 거래 목록의 지출 항목에 개인/공동 배지를 표시한다.
- 공동 소비는 별도 강조 색상으로 표시한다.
- 거래 상세 화면에 소비 구분을 표시한다.
- 거래 검색 대상에 `개인`, `공동` 소비 구분 문구를 포함한다.
- 서버 월별/연별 CSV와 화면 필터 결과 CSV에 소비 구분 열을 추가했다.

### 회귀 테스트

- 값이 없는 기존 지출이 개인 소비로 해석되는 도메인 테스트를 추가했다.
- 공동 소비 값이 유지되는지 검증했다.
- 수입 거래에 공동 소비를 전달해도 응답 값이 제거되는지 검증했다.
- 브라우저에서 공동 소비를 선택해 거래를 생성하고 API 응답이 `SHARED`인지 검증했다.

### README 정리

- 구현된 MVP 목록에 지출 거래 공동/개인 소비 구분을 추가했다.
- 다음 작업에서 거래 소비 구분 항목을 제거했다.
- 명의 관리 메뉴와 자산 명의 선택 방식 전환을 다음 우선순위로 갱신했다.

## 5. 아직 부분 완료인 항목

### 명의/소비 구분

- 자산 명의 저장·집계와 거래 공동/개인 소비 구분은 구현되어 있다.
- 개인 소비에 실제 소비 명의를 지정하는 필드는 아직 미완료다.
- 더보기에서 명의를 추가·수정·삭제하는 관리 메뉴는 아직 미완료다.
- 자산 명의는 자유 입력 문자열이며 Member 목록 선택 방식으로 전환이 필요하다.
- 인증과 권한 인가는 현재 개인용 범위에서 제외한다.

### 통계

- 월별/연별 카테고리 및 소비 태그 통계는 구현되어 있다.
- 공동/개인 소비 비중 통계와 기간 필터 기반 통계는 아직 미완료다.

### 영수증 첨부

- 신규/기존 거래의 최대 10장 일괄 첨부, 조회, 미리보기, 개별 삭제는 구현되어 있다.
- 할부 그룹 전체 수정 중 영수증 첨부 정책과 UI는 아직 미완료다.

### 문자 자동 입력

- 클립보드 문자 분석, 카테고리 추천, 거래 폼 자동 입력은 구현되어 있다.
- 사용자 정의 문자 파싱·추천 규칙 관리는 아직 미완료다.

## 8. 개발 메모

- 소비 구분은 현재 지출 거래에만 의미가 있으며 수입·이체에는 저장하지 않는다.
- 기존 데이터 마이그레이션 없이 getter에서 빈 지출 값을 개인 소비로 호환 처리한다.
- 소비 명의는 다음 단계의 Member 관리 및 거래 명의 지정과 함께 연결할 예정이다.
- 브라우저 스모크 테스트 후 Docker Compose 스택은 실행 상태로 유지되어 있다.
