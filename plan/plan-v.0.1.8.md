# 편한가계부 신규 개발 진행 기록 v.0.1.8

작성일: 2026-06-26

## 1. 기준

- 이전 기록: `plan/plan-v.0.1.7.md`
- 이번 개발 범위: OCR 결과 후보 다중 선택 UI 1차 구현 및 Docker 재개 후 누적 검증
- 배경: OCR 결과는 한 번의 자동 파싱으로 확정하기 어렵다. 특히 영수증에는 여러 품명과 여러 금액이 함께 등장하므로, 사용자가 후보 중 적절한 거래 제목/금액을 선택할 수 있어야 한다.

## 2. 이번 버전 개발 완료

### OCR 제목/품명 후보 선택

- OCR 결과 화면에서 `OCR 후보 선택` 영역을 추가했다.
- OCR 원문에서 다음 값을 제목/품명 후보로 추출한다.
  - 현재 파싱된 가맹점/품명
  - `품명/단가/수량/금액` 표의 품목명
  - 날짜, 금액, 합계, 사업자, 전화번호 등 노이즈가 아닌 일반 텍스트 라인
- 사용자가 후보 칩을 선택하면 거래 입력에 반영될 `merchant` 후보가 즉시 갱신된다.

### OCR 금액 후보 선택

- OCR 원문에서 숫자 금액 후보를 최대 6개까지 추출한다.
- 현재 파싱된 금액을 우선 후보로 노출한다.
- 사용자가 금액 후보 칩을 선택하면 거래 입력에 반영될 `amount` 후보가 즉시 갱신된다.

### 기존 흐름과의 연결

- 후보 선택 후 `거래 입력에 사용`을 누르면 선택된 제목/금액이 거래 입력 초안에 반영된다.
- v0.1.4의 OCR 원문 수정 후 재분석 흐름과 함께 동작한다.
- v0.1.5의 품목표 여러 행 메모 요약 기능은 유지한다.

### Docker 재개 후 누적 검증

- Docker Desktop이 재개되어 v0.1.6~v0.1.8 누적 변경 검증을 수행했다.
- 백엔드 테스트, 백엔드 이미지 빌드, 프론트엔드 이미지 빌드, 브라우저 스모크 테스트를 통과했다.

## 4. 아직 부분 완료인 항목

### 후보 품질

- 현재 후보 추출은 프론트의 간단한 정규식/라인 필터 기반이다.
- 후보의 신뢰도 점수나 서버 기반 후보 DTO는 아직 없다.
- 필요하면 다음 단계에서 백엔드가 후보 목록을 명시적으로 내려주는 구조로 확장하는 것이 좋다.

### 후보 범위

- 현재는 제목/품명과 금액 후보만 선택할 수 있다.
- 날짜, 카테고리, 소비 태그 후보 선택은 아직 없다.
