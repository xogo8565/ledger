## 2. 이번 버전 개발 완료

### 명의 관리 API

- `GET /api/members` 명의 목록 API를 추가했다.
- `POST /api/members` 명의 추가 API를 추가했다.
- `PUT /api/members/{id}` 명의 이름 수정 API를 추가했다.
- `DELETE /api/members/{id}` 명의 삭제 API를 추가했다.
- 명의 목록은 기본 OWNER 명의를 먼저 표시하고 이름순으로 정렬한다.
- 신규 명의는 인증 권한과 무관한 일반 명의 용도로 `EDITOR` 역할로 저장한다.

### 명의 정책

- 명의 이름의 앞뒤 공백과 중복 공백을 정규화한다.
- 같은 가계부 안에서 대소문자를 무시한 중복 명의를 등록할 수 없다.
- 기본 OWNER 명의는 삭제할 수 없다.
- 자산에서 사용 중인 명의는 삭제할 수 없다.
- 사용 중인 명의 삭제 시 자산 명의를 먼저 변경하도록 안내한다.
- 명의 이름을 수정하면 해당 문자열을 사용 중인 자산의 명의도 함께 변경한다.
- 인증, 초대, 사용자 전환, 권한 인가는 이번 범위에 포함하지 않는다.

### 더보기 명의 관리 화면

- 더보기 메뉴에 `명의 관리` 항목을 추가했다.
- 명의 목록에서 기본 명의와 일반 명의를 구분해 표시한다.
- 한 화면에서 명의 추가와 이름 수정을 지원한다.
- 수정 중 취소 버튼으로 신규 입력 상태로 돌아갈 수 있다.
- 일반 명의는 확인창 후 삭제할 수 있다.
- 기본 명의의 삭제 버튼은 비활성화한다.
- API 오류 메시지를 사용자 안내창으로 표시한다.

### 회귀 테스트

- 더보기에서 명의 관리 화면이 열리는지 검증했다.
- 기본 명의 삭제 버튼이 비활성화되어 있는지 검증했다.
- 고유 명의를 UI에서 추가하고 목록에 표시되는지 검증했다.
- 추가한 명의의 이름을 수정하고 목록이 갱신되는지 검증했다.
- 수정한 일반 명의를 확인창 후 삭제하고 목록에서 제거되는지 검증했다.

### README 정리

- 구현된 MVP 목록에 더보기 명의 관리 기능을 추가했다.
- 다음 작업에서 명의 관리 메뉴를 제거했다.
- 자산 명의 목록 선택과 개인 소비 명의 지정을 다음 우선순위로 갱신했다.

## 5. 아직 부분 완료인 항목

### 명의/소비 구분

- 명의 관리, 자산 명의 저장·집계, 거래 공동/개인 소비 구분은 구현되어 있다.
- 자산 입력 화면은 아직 명의 자유 입력이며 Member 목록 선택 방식으로 전환이 필요하다.
- 개인 소비 거래에 실제 소비 명의를 지정하는 기능은 아직 미완료다.
- 명의 이름 수정 시 자산 문자열은 갱신하지만 거래 소비 명의 필드는 아직 존재하지 않는다.
- 인증과 권한 인가는 현재 개인용 범위에서 제외한다.

### 통계

- 월별/연별 카테고리 및 소비 태그 통계는 구현되어 있다.
- 공동/개인 소비 통계와 기간 필터 기반 통계는 아직 미완료다.

### 영수증 첨부

- 신규/기존 거래의 최대 10장 일괄 첨부, 조회, 미리보기, 개별 삭제는 구현되어 있다.
- 할부 그룹 전체 수정 중 영수증 첨부 정책과 UI는 아직 미완료다.

### 문자 자동 입력

- 클립보드 문자 분석, 카테고리 추천, 거래 폼 자동 입력은 구현되어 있다.
- 사용자 정의 문자 파싱·추천 규칙 관리는 아직 미완료다.

## 8. 개발 메모

- Member 역할 값은 기존 도메인 호환을 위해 유지하지만 현재 UI에서는 권한 기능으로 사용하지 않는다.
- 명의 삭제 보호는 OWNER 여부와 자산 ownerName 사용 여부를 검사한다.
- 명의 이름 수정은 현재 자산 ownerName 문자열도 함께 갱신한다.
- 다음 단계에서 자산 입력을 Member 선택 방식으로 바꾸면 문자열 불일치를 더 줄일 수 있다.
- 브라우저 스모크 테스트 후 Docker Compose 스택은 실행 상태로 유지되어 있다.
