# 가계부 앱 개발 진행 기록 v.0.0.16

작성일: 2026-06-23

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.15.md`
- 이번 개발 범위: v.0.0.16 권장 항목 중 법정 공휴일/대체공휴일 캘린더 기반 결제일 보정
- 검증 방식: Docker Compose 컨테이너 빌드, Gradle 테스트 컨테이너 실행

## 2. 이번 버전 개발 완료

### 한국 공휴일 캘린더 기반 영업일 보정

- `KoreanHolidayCalendar` 서비스를 추가했다.
- 결제일이 토요일, 일요일, 법정 공휴일, 대체공휴일이면 다음 영업일까지 반복 이동하도록 했다.
- 고정 양력 공휴일과 2025~2027년 설날, 부처님오신날, 추석, 대체공휴일 캘린더를 반영했다.
- 기존 카드 결제일 계산이 `KoreanHolidayCalendar.nextBusinessDay()`를 사용하도록 변경했다.
- 2026년 개천절 토요일과 10월 5일 대체공휴일이 이어질 때 10월 6일로 보정되는 테스트를 추가했다.
- 2026년 추석 연휴 결제일이 다음 영업일로 이동하는 테스트를 추가했다.

### 카드 결제 UI 문구

- 카드 결제 상세의 보정 안내 문구를 `주말 보정`에서 `영업일 보정`으로 변경했다.
- 공휴일/대체공휴일 보정까지 포함하는 의미로 README의 카드 결제 MVP 설명을 최신화했다.

## 5. 아직 부분 완료인 항목

### 카드 관리

- 2025~2027년 공휴일 캘린더는 코드 내 정적 데이터로 반영되어 있다.
- 2028년 이후 공휴일은 데이터 갱신 또는 외부 캘린더 연동이 필요하다.
- 카드 결제 예정 금액과 할부 청구 월 연결 검증이 필요하다.

### 영수증 첨부

- 신규 거래 첨부와 업로드 API는 구현되어 있다.
- 첨부 이미지 조회, 미리보기, 삭제 UI는 구현되어 있다.
- 기존 거래 수정 중 신규 영수증 추가와 다중 첨부 흐름 고도화는 아직 미완료다.

### 문자 자동 입력

- 원문 붙여넣기, 후보 추출, 거래 입력 폼 확정 이동은 구현되어 있다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

### 통계

- 연별 통계, 기간 필터, 카테고리 선택 후 거래 목록 이동은 미완료다.

## 8. 개발 메모

- 2026년 공휴일 목록은 공개 공휴일 캘린더를 기준으로 반영했다.
- 결제일 보정은 원 결제일을 유지하면서 실제 예정일만 다음 영업일로 계산한다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드와 Gradle 테스트는 정상 통과했다.
