## 2. 이번 버전 개발 완료

### 서버 기반 거래 검색 API

- `GET /api/transactions/search` API를 추가했다.
- 다음 조건을 선택적으로 조합할 수 있다.
  - 시작일과 종료일
  - 거래 유형
  - 카테고리
  - 개인/공동 소비 구분
  - 소비 명의
  - 자산
  - 최소 금액과 최대 금액
  - 키워드
- 자산 조건은 일반 거래 자산뿐 아니라 이체의 출금·입금 자산도 함께 검색한다.
- 키워드는 제목, 메모, 소비 태그, 카테고리명, 자산명, 이체 자산명, 소비 명의를 대상으로 검색한다.
- 여러 조건은 AND로 결합하고, 결과는 거래일과 ID 역순으로 반환한다.

### JPA 동적 조건 검색

- `TransactionRepository`에 `JpaSpecificationExecutor`를 적용했다.
- 조건이 지정된 항목만 Criteria predicate에 추가하도록 구성했다.
- 연관 데이터 키워드 검색은 LEFT JOIN을 사용해 카테고리나 자산이 없는 거래도 누락되지 않게 했다.
- Household 조건을 항상 적용해 다른 가계부 데이터가 섞이지 않도록 했다.

### 검색 조건 검증

- 종료일이 시작일보다 빠르면 400 오류로 처리한다.
- 최소·최대 금액이 음수이면 400 오류로 처리한다.
- 최대 금액이 최소 금액보다 작으면 400 오류로 처리한다.
- 잘못된 조건이 저장소 조회 전에 거절되는 단위 테스트를 추가했다.

### 가계부 고급 필터 UI

- 기존 검색어·유형·카테고리·소비 구분·명의·기간 필터에 자산 필터를 추가했다.
- 최소 금액과 최대 금액 입력을 추가했다.
- 필터가 활성화되면 현재 월 범위 또는 사용자가 지정한 기간을 기준으로 서버 검색 API를 호출한다.
- 검색어 입력은 250ms 지연 후 요청해 타이핑 중 과도한 API 호출을 줄였다.
- 서버 결과를 기존 일일·달력·월별·요약·메모 화면과 필터 결과 CSV에 공통 적용한다.
- 필터 초기화 시 현재 월 기본 거래 목록으로 되돌아간다.

### 테스트 및 스모크 보강

- 검색 날짜와 금액 범위 검증 단위 테스트를 추가했다.
- 브라우저 스모크 테스트에서 키워드·유형·자산·정확한 금액 범위를 조합한 API 검색을 검증한다.
- 화면에서 자산과 금액 범위를 적용했을 때 서버 검색 요청과 거래 목록 반영을 검증한다.

- 주요 화면과 상태가 `frontend/src/main.jsx`에 집중되어 있다.

- 도메인별 서비스 분리와 검색 조건 객체 분리가 필요하다.

## 7. 개발 메모

- 날짜를 지정하지 않은 고급 필터는 현재 선택 월의 시작일과 마지막 날을 서버 검색 범위로 사용한다.
- 날짜 한쪽만 지정하면 나머지 경계는 현재 선택 월의 시작일 또는 마지막 날로 보완한다.
- 현재 데이터 규모에서는 전체 검색 결과를 반환하며, 데이터 증가에 대비한 페이지네이션은 다음 단계로 남겼다.
