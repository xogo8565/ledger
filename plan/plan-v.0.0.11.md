## 2. 이번 버전 개발 완료

### 카드 결제일 주말 보정 정책

- 카드 결제 예정일이 토요일이면 다음 월요일로 보정한다.
- 카드 결제 예정일이 일요일이면 다음 월요일로 보정한다.
- 결제 예정일이 평일이면 기존 날짜를 유지한다.
- 청구 기간 계산은 보정된 결제일이 아니라 원래 결제일이 속한 달을 기준으로 유지한다.
  - 예: 원래 결제일이 일요일이라 다음 달 월요일로 밀려도 청구 기간은 원래 결제월 기준으로 계산한다.
- 월에 없는 결제일은 기존처럼 말일로 먼저 보정한 뒤 주말 보정을 적용한다.

### 카드 상세 응답 확장

- `GET /api/cards/{cardAssetId}` 응답에 다음 값을 추가했다.
  - `originalPaymentDate`
  - `paymentDateAdjusted`
- `nextPaymentDate`는 실제 표시/예정에 사용할 보정된 결제일을 반환한다.
- `originalPaymentDate`는 주말 보정 전 원래 결제일을 반환한다.
- `paymentDateAdjusted`는 결제일 보정 여부를 나타낸다.

### 카드 결제 UI

- 카드 결제 화면의 예정일 문구에 주말 보정 여부를 표시한다.
- 결제일이 보정된 경우 `원래 결제일 주말 보정` 문구를 함께 보여준다.

### README 정리

- 구현된 MVP 목록의 카드 결제 항목에 주말 결제일 보정을 반영했다.
- 다음 작업 목록에서 카드 결제일 휴일 보정 정책 항목을 제거했다.

## 5. 아직 부분 완료인 항목

### 카드 관리

- 카드 결제 예약과 실행은 구현되어 있다.
- 카드 자산 생성/수정 시 카드 프로필 입력과 저장 API 연결은 구현되어 있다.
- 자동 결제 중복 실행 방지는 `PROCESSING` 상태와 낙관적 잠금으로 보강했다.
- 실패 상태 저장과 UI 표시는 구현되어 있다.
- 실패한 카드 결제 재시도와 재예약 UI는 구현되어 있다.
- 결제 예정 금액은 다음 결제 예정일에 대응하는 청구 기간 기준으로 산정한다.
- 말일 처리는 구현되어 있다.
- 주말 결제일 보정은 구현되어 있다.
- 법정 공휴일/대체공휴일 데이터 기반 보정은 아직 미완료다.

### 할부 관리

- 할부 생성, 조회, 그룹 단위 수정/삭제는 구현되어 있다.
- 그룹 수정 시 기존 회차 수 변경은 아직 미지원이다.
- 카드 결제 예정 금액과 할부 청구 월 연결 검증이 필요하다.

### 예산

- 월별 전체/카테고리 예산 저장은 구현되어 있다.
- 카테고리별 예산 사용률과 초과 표시는 구현되어 있다.
- 전월 예산 복사는 구현되어 있다.

### 통계

- 월별 및 카테고리별 지출 통계는 구현되어 있다.
- 연별 통계, 기간 필터, 카테고리 선택 후 거래 목록 이동은 미완료다.

### 영수증 첨부

- 신규 거래 첨부와 업로드 API는 구현되어 있다.
- 첨부 이미지 조회, 미리보기, 삭제, 다중 첨부 UI는 미완료다.

### 문자 자동 입력

- 원문 붙여넣기와 후보 추출은 구현되어 있다.
- 파싱 결과를 거래 입력 폼으로 확정 저장하는 흐름은 미완료다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

### 공동 가계부

- Household/Member 도메인은 있다.
- 인증, 초대, 권한, 사용자 전환 UI는 미완료다.
- 인증, 권한, 사용자 전환은 불필요하더라도 자산 명의나 관련 소비 태그로 활용할 구조가 필요하다.

## 8. 개발 메모

- 이번 버전은 공휴일 데이터 없이 확정 가능한 주말 보정을 구현했다.
- 한국 법정 공휴일과 대체공휴일은 매년 변동될 수 있으므로, 별도 휴일 캘린더 데이터 또는 외부 API/수동 등록 구조가 필요하다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드는 정상 통과했다.
