## 2. 이번 버전 개발 완료

### 품목표 우선 파싱

- `ImportTextService`에 영수증 품목표 감지 로직을 추가했다.
- OCR 텍스트에서 `품명`, `단가`, `수량`, `금액`이 포함된 헤더 라인을 찾는다.
- 품목표가 감지되면 다음 유효 품목 행을 우선 거래 제목 후보로 사용한다.
- 품목 행 파싱 방식:
  - 첫 숫자 앞까지를 품명으로 판단한다.
  - 행의 마지막 숫자를 해당 품목 금액으로 판단한다.
- 합계 금액이 있으면 거래 금액은 합계를 우선 사용한다.
- 합계 금액이 없으면 첫 품목 행의 금액을 거래 금액으로 사용한다.

### 기존 정책과의 관계

- 품목표가 없는 영수증은 기존처럼 첫 유효 영수증 라인을 가맹점 후보로 사용한다.
- 품목표가 있어도 합계가 있으면 총 거래 금액은 합계를 우선한다.
- 품목표가 있고 합계가 없으면 첫 품목의 금액을 사용한다.

### 회귀 테스트 추가

- `ImportTextServiceTest`에 품목표 우선 파싱 테스트를 추가했다.
- `품명 단가 수량 금액` 헤더가 있는 영수증에서 제목 후보가 첫 품명으로 잡히는지 검증했다.
- 합계가 있는 경우 금액은 합계를 우선하는지 검증했다.
- 합계가 없는 경우 첫 품목 금액을 사용하는지 검증했다.

## 4. 아직 부분 완료인 항목

### OCR UX

- OCR 실패/낮은 신뢰도 안내는 아직 더 보강할 수 있다.
- OCR 원문을 사용자가 수정한 뒤 재파싱하는 UI/API는 아직 없다.

### 실제 이미지 OCR 회귀

- 조건부 실제 이미지 OCR 테스트 파일은 추가되어 있으나, OCR 결과는 환경과 폰트에 따라 변동될 수 있다.
- 더 안정적인 fixture 이미지와 기대값 조정이 필요하다.

### 품목표 고도화

- 여러 품목을 선택하는 UI는 아직 없다.
- 현재는 첫 품목만 거래 제목 후보로 사용한다.
- 여러 품목 합산, 대표 품명 선택, 품목 전체 메모화는 추후 개선 대상이다.

- 사용자가 명시한 `품명/단가/수량/금액` 구조를 OCR 영수증 입력에서 가장 강한 신호로 사용하도록 했다.

- 다만 여러 품목 영수증에서는 첫 품목만으로 거래를 대표하기 어렵기 때문에, 다음 단계에서 품목 요약 메모 또는 품목 선택 UI를 붙이는 것이 좋다.
