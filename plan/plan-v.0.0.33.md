# 편한가계부 후속 개발 진행 기록 v.0.0.33

작성일: 2026-06-24

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.32.md`
- 이번 개발 범위: v.0.0.33 권장 항목 1순위인 개인 소비 명의 지정
- 검증 방식: Docker Compose 이미지 빌드, Gradle 전체 테스트, Playwright 브라우저 스모크 테스트

## 2. 이번 버전 개발 완료

### 거래 소비 명의 도메인

- 거래에 Member 기반 소비 명의 관계를 추가했다.
- 지출 거래의 소비 구분이 `PERSONAL`일 때만 소비 명의를 유지한다.
- `SHARED` 지출과 수입·이체 거래에서는 전달된 소비 명의를 서버에서 자동으로 제거한다.
- 개인 지출 요청에 명의가 없으면 현재 가계부의 OWNER 명의를 기본값으로 지정한다.
- 다른 가계부의 Member ID는 소비 명의로 사용할 수 없도록 검증한다.
- 개인 소비 거래에서 사용 중인 명의는 삭제할 수 없도록 보호한다.

### 거래 API

- 거래 생성·수정 요청에 `consumerMemberId`를 추가했다.
- 거래 응답에 `consumerMemberId`, `consumerMemberName`을 추가했다.
- 할부 거래 생성·전체 수정에도 같은 소비 명의를 각 회차에 적용한다.
- 카드 자동 결제와 반복 거래 생성 경로는 소비 명의를 비우는 기존 정책을 유지한다.

### 거래 입력·조회 UI

- 개인 소비 선택 시 등록 명의 목록에서 소비 명의를 선택할 수 있다.
- 신규 개인 지출은 OWNER 명의를 기본 선택한다.
- 공동 소비로 전환하면 소비 명의 입력을 숨기고 값을 비운다.
- 수입·이체로 전환해도 소비 명의를 제거한다.
- 거래 수정 시 저장된 소비 명의를 다시 선택한다.
- 거래 목록의 개인 소비 배지에 명의 이름을 함께 표시한다.
- 거래 상세에 개인 소비 명의를 표시한다.
- 거래 검색 대상에 소비 명의 이름을 포함했다.

### CSV 내보내기

- 서버 월별·연별 CSV에 `소비명의` 열을 추가했다.
- 화면 필터 결과 CSV에도 `소비명의` 열을 추가했다.
- 공동 소비와 비지출 거래는 소비 명의를 빈 값으로 내보낸다.

### 회귀 테스트

- 개인 지출만 소비 명의를 유지하는 도메인 테스트를 추가했다.
- 브라우저에서 개인 소비 명의를 선택해 저장하고 응답 이름이 일치하는지 검증한다.
- 공동 소비 요청에 명의를 함께 보내도 서버 응답에서 제거되는지 검증한다.
- 기존 자산·명의·카테고리·예산·반복 거래·카드 결제·영수증·클립보드 입력 회귀 시나리오를 유지했다.

## 4. 아직 부분 완료인 항목

### 소비 통계

- 거래별 공동/개인 소비 구분과 개인 소비 명의 지정은 완료됐다.
- 공동/개인 소비 금액 비교 통계와 명의별 소비 통계는 아직 미완료다.
- 다음 버전에서는 월별·연별 요약에 소비 구분 및 명의 집계를 추가할 수 있다.

### 기존 개인 지출 데이터

- 기존 개인 지출 중 소비 명의가 없는 데이터는 API 응답에서 명의가 비어 있을 수 있다.
- 해당 거래를 수정·저장하면 기본 또는 선택 명의가 연결된다.
- 운영 데이터 규모가 커지면 OWNER 명의를 일괄 연결하는 별도 마이그레이션을 고려한다.

### 영수증·문자 규칙·기간 통계

- 할부 그룹 전체 수정 중 영수증 첨부 정책과 UI는 미완료다.
- 사용자 정의 문자 파싱·추천 규칙 관리는 미완료다.
- 사용자 지정 기간 기반 통계는 미완료다.

## 7. 개발 메모

- 소비 명의는 문자열 복사가 아니라 Member 관계로 저장하므로 명의 이름 수정 시 거래 표시도 최신 이름을 사용한다.
- 소비 구분 변경 시 명의 정합성은 프론트엔드와 서버 양쪽에서 처리한다.
- 실행 중인 Docker Compose 스택은 최신 이미지로 재생성했으며 프론트엔드는 `http://localhost:8081`에서 확인할 수 있다.
