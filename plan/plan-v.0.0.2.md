# 가계부 앱 개발 진행 기록 v.0.0.2

작성일: 2026-06-22

## 1. 기준 문서

- 기준 기획: `plan/plan-v.0.0.0.md`
- 현재 코드 기준: Spring Boot 백엔드, React/Vite 프론트엔드, Docker Compose 실행 환경
- 목적: 최초 기획 대비 현재 개발 진행 사항과 잔여 항목을 기능 단위로 기록한다.

## 2. 현재 개발 요약

v.0.0.2 기준으로 1차 MVP의 핵심 골격은 구현되어 있다. 거래 입력, 일일/월별 목록, 수입/지출/이체 처리, 자산/카테고리/예산 관리, 카드 결제 예약, 반복 거래, 할부 거래, 영수증 첨부 API, 문자 텍스트 파싱 초안까지 포함되어 있다.

프론트엔드는 모바일 앱 형태의 단일 React 화면으로 구성되어 있으며, 하단 탭 기준으로 `가계부`, `통계`, `자산`, `더보기` 화면을 제공한다. 백엔드는 JPA 도메인과 REST API를 갖추고 MySQL을 기본 저장소로 사용한다.

## 4. 부분 완료 항목

### 카드 결제 규칙

- 결제금액 확정일 기준 청구 기간 계산과 결제 예정 금액 표시는 구현되어 있다.
- 다만 실제 카드사별 청구 기간 규칙, 말일 결제일, 결제일 휴일 처리, 결제 예정 금액 자동 생성 정책은 더 정교화가 필요하다.
- 자동 결제 스케줄러 구조는 있으나 운영 정책과 중복 실행 방지 검증이 더 필요하다.

### 카드 미결제 금액

- 카드 지출과 완료된 결제 예약 금액을 기준으로 미결제 금액을 계산한다.
- 카드 자산의 잔액 자체는 카드 지출 시 직접 변경하지 않는 정책으로 구현되어 있다.
- 카드 대금 결제 거래가 순자산/부채 계산과 사용자 인식에 맞게 표시되는지는 추가 검증이 필요하다.

### 할부 관리

- 할부 거래 분할 생성과 조회는 구현되어 있다.
- 할부 수정/삭제 시 전체 할부 그룹 단위 조정 UI는 아직 부족하다.
- 카드 결제 예정 금액과 할부 청구 월의 연결 검증이 필요하다.

### 예산

- 전체 예산과 카테고리별 예산 저장/표시는 구현되어 있다.
- 카테고리별 예산 사용률, 초과 상태 강조, 전월 예산 복사는 미완료다.
- 예산 상세 화면의 사용성 고도화가 필요하다.

### 통계

- 월별 통계와 카테고리별 지출 비중은 구현되어 있다.
- 연별 통계, 기간 필터, 카테고리 클릭 후 거래 목록 이동, 자산별 통계는 미완료다.

### 문자 자동 입력

- 정규식 기반 후보 추출 초안은 있다.
- 카드사/은행별 규칙, 사용자가 수정/확정한 뒤 거래로 저장하는 흐름, 자동 카테고리 추천은 미완료다.

### 영수증 첨부

- 업로드 API와 거래 입력 시 첨부 흐름은 있다.
- 첨부 이미지 조회, 미리보기, 삭제, 다중 첨부 UI는 미완료다.

### 공동 가계부

- `Household`, `Member`, 역할 도메인은 구현되어 있다.
- 실제 초대, 권한, 사용자 전환, 인증/인가 흐름은 미완료다.

## 8. 개발 메모

- 현재 구현은 MVP 기능 확인용으로 충분한 폭을 갖췄지만, 운영 수준의 정확성을 위해 카드 결제, 반복 거래, 할부, 예산 계산은 추가 테스트가 필요하다.
- 프론트엔드는 빠르게 기능을 확장한 단일 파일 구조이므로 이후 컴포넌트 분리가 필요하다.
- 일부 문서와 코드 주석에서 한글 인코딩이 깨진 흔적이 있어, 배포 전 UTF-8 기준으로 정리해야 한다.
- 자동화 테스트는 스모크 테스트 중심으로 준비되어 있으며, 다음 단계에서는 서비스 단위 테스트와 주요 계산 로직 테스트를 보강하는 것이 좋다.
