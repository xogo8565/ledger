## 2. 이번 버전 개발 완료

### 할부 그룹 수정 중 영수증 보존 정책

- 할부 그룹 전체 수정 시 기존 회차 거래를 가능한 한 유지하도록 수정했다.
- 기존 회차의 영수증은 같은 회차 거래에 그대로 남긴다.
- 회차 수를 줄일 때 삭제되는 회차에 있던 영수증은 마지막으로 남는 회차로 이동한다.
- 회차 수를 늘릴 때 기존 회차 영수증은 유지하고 추가 회차만 새 거래로 생성한다.
- 이 정책으로 영수증이 있는 할부 그룹을 수정해도 첨부 FK 오류나 첨부 유실이 발생하지 않도록 했다.

### 할부 전체 수정 영수증 UI

- 할부 전체 수정 화면에서도 영수증 파일을 새로 첨부할 수 있게 했다.
- `첨부 회차` 선택 항목을 추가해 업로드할 회차를 명시적으로 고를 수 있다.
- 회차 수를 줄이면 선택된 첨부 회차가 새 회차 수를 넘지 않도록 화면에서 보정한다.
- 저장 후 선택된 회차 거래에 기존 영수증 일괄 첨부 API를 재사용해 업로드한다.

### 테스트 및 스모크 보강

- 브라우저 스모크 테스트에 할부 전체 수정 중 영수증 업로드 시나리오를 추가했다.
- 스모크 테스트에서 선택한 회차에 영수증이 실제로 첨부되는지 API로 검증한다.
- 검증 후 테스트 영수증은 삭제해 후속 정리에 영향을 주지 않게 했다.

## 4. 아직 부분 완료인 항목

### 기존 개인 지출 데이터

- 소비 명의가 없는 기존 개인 지출은 명의별 통계에서 `명의 미지정`으로 집계된다.
- OWNER 명의로 일괄 연결하는 마이그레이션 정책은 아직 미완료다.

### 기간 기반 통계

- 가계부 거래 목록에는 시작일·종료일 필터가 있다.
- 통계 화면 자체의 사용자 지정 기간 집계는 아직 미완료다.

### 기타 후속 항목

- 예산 관련 연간 이동 기본값과 주간별 통계는 미완료다.
- 프론트엔드 화면 단위 컴포넌트 분리는 아직 미완료다.

## 7. 개발 메모

- 할부 전체 수정은 더 이상 기존 회차를 모두 삭제하지 않고, 유지 가능한 회차를 업데이트한다.
- 삭제되는 회차의 영수증을 마지막 유지 회차로 이동하는 정책은 첨부 유실을 막기 위한 보수적 기본값이다.
- 신규 첨부는 프론트엔드에서 선택한 회차에만 저장하며, 여러 회차에 같은 파일을 복제하지 않는다.
