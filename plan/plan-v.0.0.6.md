- 검증 방식: Docker Compose 컨테이너 빌드

## 2. 이번 버전 개발 완료

### 카드 결제 예정 금액 산정 정책

- 카드 결제 예정 금액을 카드 프로필의 결제일과 결제금액 확정일 기준으로 계산하도록 변경했다.
- 다음 결제 예정일을 먼저 산정한 뒤, 그 결제일에 대응하는 청구 기간을 계산한다.
- 청구 기간 정책:
  - 다음 결제 예정일이 속한 달의 전월 확정일을 청구 종료일로 본다.
  - 청구 시작일은 그 전 청구 종료일의 다음 날로 본다.
  - 확정일 또는 결제일이 해당 월에 없으면 그 달 말일로 보정한다.
- `GET /api/cards/{cardAssetId}` 응답에 다음 값을 추가했다.
  - `nextPaymentDate`
  - `billingStartDate`
  - `billingEndDate`
- `GET /api/cards/{cardAssetId}/billing-period`도 같은 청구 기간 정책을 사용하도록 맞췄다.

### 카드 결제 UI 설명 보강

- 카드 결제 화면의 요약 영역에 예정 금액의 기준 청구 기간을 표시한다.
- 다음 결제 예정일도 함께 표시해 사용자가 예정 금액의 기준을 바로 알 수 있게 했다.

### README 정리

- 구현된 MVP 목록의 카드 결제 항목에 청구 기간 기반 예정 금액 산정을 반영했다.
- 다음 작업 목록에서 일반적인 청구 기간 계산 고도화 항목을 제거하고, 남은 휴일 보정 정책으로 좁혀 기록했다.

## 5. 아직 부분 완료인 항목

### 카드 관리

- 카드 결제 예약과 실행은 구현되어 있다.
- 카드 자산 생성/수정 시 카드 프로필 입력과 저장 API 연결은 구현되어 있다.
- 자동 결제 중복 실행 방지는 `PROCESSING` 상태와 낙관적 잠금으로 보강했다.
- 실패 상태 저장과 UI 표시는 구현되어 있다.
- 결제 예정 금액은 다음 결제 예정일에 대응하는 청구 기간 기준으로 산정한다.
- 말일 처리는 구현되어 있다.
- 공휴일/주말 결제일 보정 정책은 아직 미완료다.
- 실패한 카드 결제를 사용자가 다시 예약하거나 재시도하는 UI는 아직 미완료다.

### 할부 관리

- 할부 생성과 조회는 구현되어 있다.
- 할부 그룹 전체 수정/삭제는 아직 미완료다.
- 카드 결제 예정 금액과 할부 청구 월 연결 검증이 필요하다.

### 예산

- 월별 전체/카테고리 예산 저장은 구현되어 있다.
- 카테고리별 예산 사용률, 초과 표시, 전월 예산 복사는 미완료다.

### 통계

- 월별 및 카테고리별 지출 통계는 구현되어 있다.
- 연별 통계, 기간 필터, 카테고리 선택 후 거래 목록 이동은 미완료다.

### 영수증 첨부

- 신규 거래 첨부와 업로드 API는 구현되어 있다.
- 첨부 이미지 조회, 미리보기, 삭제, 다중 첨부 UI는 미완료다.

### 문자 자동 입력

- 원문 붙여넣기와 후보 추출은 구현되어 있다.
- 파싱 결과를 거래 입력 폼으로 확정 저장하는 흐름은 미완료다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

### 공동 가계부

- Household/Member 도메인은 있다.
- 인증, 초대, 권한, 사용자 전환 UI는 미완료다.
- 인증 등, 권한, 사용자 전환 등 기능은 불필요하지만, 자산 명의 / 관련 소비 등에 태그될 수 있도록 구성한다.

## 8. 개발 메모

- 카드 청구 기간은 결제 예정일의 전월 확정일을 청구 종료일로 보는 정책으로 구현했다.
- 공휴일 데이터가 아직 없으므로 결제일 휴일 보정은 다음 작업으로 남겼다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드는 정상 통과했다.
