# 가계부 앱 개발 진행 기록 v.0.0.15

작성일: 2026-06-23

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.14.md`
- 이번 개발 범위: v.0.0.15 권장 항목 중 문자 파싱 결과를 거래 입력 폼으로 확정 저장
- 검증 방식: Docker Compose 컨테이너 빌드

## 2. 이번 버전 개발 완료

### 문자 자동 입력 확정 흐름

- 문자 분석 후 즉시 거래 입력 화면으로 이동하던 흐름을 미리보기 확인 방식으로 분리했다.
- 더보기 화면의 문자 자동 입력 영역에서 분석 결과를 가맹점, 거래일, 유형, 금액, 메모, 원문으로 확인할 수 있게 했다.
- 미리보기 카드에 `거래 입력으로 이동` 버튼을 추가했다.
- 확정 버튼을 누르면 파싱된 거래 유형, 거래일, 금액, 가맹점, 메모가 거래 입력 폼에 채워진다.
- 확정 후 기존 거래 등록 로직을 그대로 사용해 저장되도록 했다.
- 확정 시 수정/할부 편집 상태와 영수증 파일 상태를 초기화해 신규 거래 입력으로 처리되게 했다.

### README 정리

- 구현된 MVP 목록의 문자 자동 입력 항목을 후보 추출 및 거래 입력 확정 기준으로 최신화했다.
- 다음 작업 목록에서 문자 파싱 결과 확정 저장 항목을 제거했다.
- 다음 카드 관리 작업을 법정 공휴일/대체공휴일 결제일 보정으로 올렸다.

## 5. 아직 부분 완료인 항목

### 영수증 첨부

- 신규 거래 첨부와 업로드 API는 구현되어 있다.
- 첨부 이미지 조회, 미리보기, 삭제 UI는 구현되어 있다.
- 기존 거래 수정 중 신규 영수증 추가와 다중 첨부 흐름 고도화는 아직 미완료다.

### 문자 자동 입력

- 원문 붙여넣기, 후보 추출, 거래 입력 폼 확정 이동은 구현되어 있다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

### 카드 관리

- 법정 공휴일/대체공휴일 데이터 기반 결제일 보정은 아직 미완료다.
- 카드 결제 예정 금액과 할부 청구 월 연결 검증이 필요하다.

### 통계

- 연별 통계, 기간 필터, 카테고리 선택 후 거래 목록 이동은 미완료다.

## 8. 개발 메모

- 문자 분석 API는 미리보기 전용으로 유지하고, 실제 저장은 기존 거래 생성 API를 사용한다.
- 확정 후 거래 입력 화면에서 사용자가 자산, 카테고리, 제목, 메모를 최종 수정할 수 있다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드는 정상 통과했다.
