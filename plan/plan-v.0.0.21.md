## 2. 이번 버전 개발 완료

### 문자 파싱 규칙 확장

- 카드 승인, 체크카드 출금, 은행 입금, 승인취소/환불 문자를 구분하도록 파싱 규칙을 확장했다.
- `YYYY.MM.DD`, `YYYY-MM-DD`, `MM/DD`, `M월D일` 형태의 날짜를 인식하도록 했다.
- 잔액, 누적, 합계, 한도, 사용가능 금액 등 보조 금액을 제거한 뒤 거래 금액을 추출하도록 했다.
- 카드사/은행명, 카드번호 일부, 시간, 승인번호, 승인/출금/입금 같은 노이즈 단어를 제거한 뒤 가맹점/입금자 후보를 추출하도록 했다.
- 기관명이나 노이즈 단어가 서로 포함되는 경우를 줄이기 위해 긴 단어부터 제거하도록 했다.
- 파싱 메모에 `카드 승인`, `체크/출금`, `입금`, `취소/환불` 구분을 남기도록 했다.

### 파서 회귀 테스트

- 카드 승인 문자에서 잔액 금액을 거래 금액으로 오인하지 않는 테스트를 추가했다.
- 체크카드 출금 문자를 지출로 파싱하는 테스트를 추가했다.
- 은행 입금 문자를 수입으로 파싱하는 테스트를 추가했다.
- 승인취소 문자를 수입으로 파싱하는 테스트를 추가했다.

### README 정리

- 구현된 MVP 목록의 문자 자동 입력 항목을 카드/은행 문자 기준으로 최신화했다.
- 다음 작업 목록에서 카드사/은행별 문자 파싱 규칙 확장 항목을 제거했다.
- 다음 우선순위를 사용자 지정 기간 필터로 갱신했다.

## 8. 개발 메모

- 파싱 규칙은 아직 코드 기반 정규식이며, `TextImportRule` 엔티티를 통한 사용자 정의 규칙 관리는 추후 과제로 남겼다.
- 취소/환불은 현재 별도 거래 유형이 없어 수입 후보로 매핑한다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드와 Gradle 테스트는 정상 통과했다.
