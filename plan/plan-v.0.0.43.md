작성일: 2026-06-25

- 이전 기록: `plan/plan-v.0.0.42.md`

- 검증 방식: Gradle 전체 테스트, Docker Compose 이미지 빌드, Playwright 브라우저 스모크 테스트

## 2. 이번 버전 개발 완료

### 통계 화면 모듈 분리

- `frontend/src/screens/StatsScreen.jsx`를 추가했다.
- `main.jsx`에 있던 통계 화면과 다음 하위 컴포넌트를 전용 모듈로 이동했다.
  - 월별·연별·사용자 기간 전환
  - 수입·지출 표시
  - 카테고리 통계와 도넛 차트
  - 소비 태그 통계
  - 개인·공동 소비 통계
  - 소비 명의 통계
  - 연간 월별 지출 막대
  - 기간 통계 헤더
  - 거래 내용별 집계

### 통계 랭킹·이동 흐름 분리

- 카테고리, 태그, 소비 구분, 명의 랭킹 컴포넌트를 통계 모듈 내부로 이동했다.
- 랭킹 선택 후 가계부 필터로 이동하는 callback은 기존 App에서 전달한다.
- 카테고리 색상과 통계 차트 palette는 통계 모듈이 직접 소유한다.
- 통계 기간과 세부 분류 상태는 App에 유지해 기존 화면 전환 흐름을 보존한다.

### 예산 화면 모듈 분리

- 월별 예산 사용률, 카테고리별 예산, 주간 지출 통계를 통계 모듈로 이동했다.
- 연간 예산 합계와 12개월 예산 대비 지출 화면도 같은 모듈로 이동했다.
- 예산 설정 진입 callback과 월·연도 변경 상태는 App에서 전달한다.
- 공통 `ProgressBar`, `MonthNav`, `YearNav`를 재사용해 화면 간 중복을 만들지 않았다.

### 파일 규모 변화

- `main.jsx`는 약 2,436줄에서 약 1,945줄로 감소했다.
- 통계·예산 표시 로직 약 490줄을 `StatsScreen.jsx`로 이동했다.
- 프론트엔드 빌드 모듈 수는 36개로 증가해 화면 단위 분리 구조가 확장됐다.

## 4. 아직 부분 완료인 항목

### 프론트엔드 구조

- 자산과 카드 결제 화면은 아직 `main.jsx`에 남아 있다.
- 거래 입력, 거래 상세, 할부, 반복 거래 화면도 아직 `main.jsx`에 남아 있다.
- API 요청과 상태 로직을 custom hook 또는 API 모듈로 분리하는 작업은 아직 미완료다.

### 백엔드 구조

- `LedgerService`가 거래, 검색, 예산, 통계, 명의와 자산 관리 책임을 함께 담당한다.
- 도메인별 서비스 분리와 검색 조건 객체 분리가 필요하다.

### 기타 후속 항목

- 검색 결과 페이지네이션과 정렬 선택은 아직 미완료다.
- 영수증 첨부 파일 정리 정책 고도화는 아직 미완료다.

## 7. 개발 메모

- 통계 화면은 데이터 요청을 직접 수행하지 않고 App이 제공한 월·연·기간 요약을 표시한다.
- 화면 내부의 차트, 랭킹, 예산 계산용 표시 로직을 함께 이동해 통계 도메인의 응집도를 높였다.
- 기존 CSS class와 사용자 이동 callback을 유지해 화면 분리 전후 동작을 동일하게 유지했다.
