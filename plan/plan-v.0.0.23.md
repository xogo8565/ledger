## 2. 이번 버전 개발 완료

### 필터 결과 CSV 생성

- 가계부 화면에 표시 중인 거래 결과를 그대로 CSV로 내려받도록 했다.
- 검색어, 거래 유형, 카테고리, 시작일, 종료일 필터가 모두 내보내기 결과에 반영된다.
- 시작일 또는 종료일만 지정한 부분 기간 필터도 화면 결과와 동일하게 반영된다.
- CSV에는 기간, 거래일, 유형, 금액, 카테고리, 소비 태그, 자산, 출금/입금 자산, 제목, 메모, 할부 정보를 포함한다.
- 이체 거래의 출금/입금 자산 ID를 현재 자산 목록의 이름으로 변환해 기록한다.
- Excel 한글 호환을 위해 UTF-8 BOM을 포함하고, 쉼표·따옴표·줄바꿈이 포함된 값은 CSV 규칙에 맞게 이스케이프한다.
- 파일명에는 선택 월 또는 기간과 `filtered` 표시를 포함한다.

### 가계부 내보내기 UI

- 필터 영역에 현재 결과 건수와 `결과 CSV` 버튼을 추가했다.
- 결과가 0건이면 내보내기 버튼을 비활성화한다.
- 가계부 요약 화면의 내보내기 버튼도 현재 필터 결과를 사용하도록 연결했다.
- 필터가 활성화되면 버튼 문구를 `필터 결과 CSV 내보내기`로 표시한다.
- 더보기 화면의 기존 월 거래 CSV 내보내기는 유지했다.

### README 정리

- 구현된 MVP의 CSV 내보내기 항목을 월별/필터 결과 기준으로 최신화했다.
- 다음 작업에서 필터 결과 CSV 항목을 제거하고 소비 태그별 통계를 최우선으로 갱신했다.

## 8. 개발 메모

- 필터 결과는 프론트엔드에서 이미 확정된 거래 배열을 사용해 CSV로 생성하므로 화면과 내보내기 결과가 어긋나지 않는다.
- 필터가 없을 때 가계부 화면의 결과 CSV는 현재 선택 월 거래를 대상으로 한다.
- 더보기 화면의 월 CSV는 기존 백엔드 API를 계속 사용한다.
- 브라우저 스모크 테스트 후 Docker Compose 스택은 실행 상태로 유지되어 있다.
