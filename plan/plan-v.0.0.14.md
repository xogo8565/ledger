# 가계부 앱 개발 진행 기록 v.0.0.14

작성일: 2026-06-23

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.13.md`
- 이번 개발 범위: v.0.0.14 권장 항목 중 영수증 이미지 조회/삭제 UI
- 검증 방식: Docker Compose 컨테이너 빌드

## 2. 이번 버전 개발 완료

### 영수증 조회/파일 API

- `GET /api/transactions/{transactionId}/receipts` API를 추가했다.
- 거래에 첨부된 영수증 목록을 반환한다.
- `GET /api/transactions/{transactionId}/receipts/{receiptId}/file` API를 추가했다.
- 영수증 파일을 `inline` 응답으로 내려 미리보기와 새 탭 열기를 지원한다.
- `ReceiptDto`에 콘텐츠 타입과 파일 크기를 추가했다.

### 영수증 삭제 API

- `DELETE /api/transactions/{transactionId}/receipts/{receiptId}` API를 추가했다.
- 삭제 시 DB 첨부 레코드와 업로드 파일을 함께 삭제한다.
- 업로드 파일이 이미 없어도 DB 레코드는 삭제되도록 했다.

### 거래 상세 UI

- 거래 상세 화면을 열 때 영수증 목록을 함께 불러온다.
- 거래 상세 화면에 영수증 섹션을 추가했다.
- 첨부 이미지 썸네일과 파일명을 표시한다.
- 썸네일/파일명을 누르면 원본 파일을 새 탭에서 볼 수 있다.
- 영수증별 삭제 버튼을 추가했다.
- 삭제 전 브라우저 확인창을 표시한다.
- 첨부된 영수증이 없으면 빈 상태 문구를 표시한다.

### README 정리

- 구현된 MVP 목록의 영수증 항목을 첨부/조회/미리보기/삭제 기준으로 최신화했다.
- 다음 작업 목록에서 영수증 이미지 조회/삭제 UI 항목을 제거했다.

## 5. 아직 부분 완료인 항목

### 영수증 첨부

- 신규 거래 첨부와 업로드 API는 구현되어 있다.
- 첨부 이미지 조회, 미리보기, 삭제 UI는 구현되어 있다.
- 기존 거래 수정 중 신규 영수증 추가와 다중 첨부 흐름 고도화는 아직 미완료다.

### 문자 자동 입력

- 원문 붙여넣기와 후보 추출은 구현되어 있다.
- 파싱 결과를 거래 입력 폼으로 확정 저장하는 흐름은 미완료다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

### 카드 관리

- 법정 공휴일/대체공휴일 데이터 기반 결제일 보정은 아직 미완료다.
- 카드 결제 예정 금액과 할부 청구 월 연결 검증이 필요하다.

### 통계

- 연별 통계, 기간 필터, 카테고리 선택 후 거래 목록 이동은 미완료다.

## 8. 개발 메모

- 영수증 파일 응답은 현재 인증 없는 내부 MVP 기준으로 열려 있다.
- 운영 전환 시 파일 접근 권한과 파일명 헤더 인코딩 처리를 점검해야 한다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드는 정상 통과했다.
