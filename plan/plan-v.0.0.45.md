# 편한가계부 후속 개발 진행 기록 v.0.0.45

작성일: 2026-06-25

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.44.md`
- 이번 개발 범위: v.0.0.45 권장 항목 1순위인 거래 입력·상세 화면 컴포넌트 분리
- 검증 방식: Gradle 전체 테스트, Docker Compose 이미지 빌드, Playwright 브라우저 스모크 테스트

## 2. 이번 버전 개발 완료

### 거래 화면 모듈 분리

- `frontend/src/screens/TransactionScreens.jsx`를 추가했다.
- 직접 입력과 클립보드 문자 자동 입력을 선택하는 `EntryChoiceSheet`를 이동했다.
- 신규 거래와 기존 거래 수정에 사용하는 `EntryScreen`을 이동했다.
- 거래 상세 조회와 영수증 목록·삭제, 거래 수정·삭제 진입 화면을 이동했다.

### 거래 입력 폼 분리

- 수입·지출·이체 유형 전환과 유형별 입력 필드를 거래 모듈로 이동했다.
- 날짜, 금액, 출금·입금 자산, 카테고리, 자산, 수수료, 할부, 내용 입력을 함께 이동했다.
- 소비 태그, 개인·공동 소비 구분, 소비 명의 선택 흐름을 유지했다.
- 할부 그룹 수정 중 첨부 회차 선택과 다중 영수증 파일 목록 관리 UI를 유지했다.

### 계산기·금액 표현식 분리

- 거래 입력 계산기 키패드를 거래 모듈로 이동했다.
- 사칙연산 표현식을 금액으로 계산하는 helper도 거래 모듈 내부로 이동했다.
- 계산 결과 반영과 편집·신규 저장 버튼 동작은 기존 callback 흐름을 유지한다.

### 거래 상세·영수증 화면 분리

- 거래 유형·날짜·분류·자산·태그·소비 구분·명의·할부·메모 상세 표시를 이동했다.
- 영수증 이미지 미리보기, 새 탭 파일 조회, 개별 삭제 UI를 이동했다.
- 할부 내역 진입, 거래 수정, 거래 삭제 callback은 App에서 전달한다.

### 화면 전용 helper 정리

- 거래 유형 라벨과 소비 구분 라벨을 거래 모듈 내부에 배치했다.
- 기본 OWNER 소비 명의 선택 helper를 거래 입력 모듈 내부로 이동했다.
- 공통 `AppHeader`, `BackButton`, `IconButton`, `LineField`, `EmptyState`와 가계부의 `KeyValue`·거래 아이콘 helper를 재사용했다.

### 파일 규모 변화

- `main.jsx`는 약 1,674줄에서 약 1,334줄로 감소했다.
- 거래 입력·상세·계산기 표시 로직 약 350줄을 `TransactionScreens.jsx`로 이동했다.
- 프론트엔드 빌드 모듈 수는 38개로 증가했다.

## 4. 아직 부분 완료인 항목

### 프론트엔드 구조

- 반복 거래와 할부 일정 화면은 아직 `main.jsx`에 남아 있다.
- 예산 설정 화면과 하단 탐색도 아직 `main.jsx`에 남아 있다.
- 거래 API 요청과 상태 로직은 App에 유지되어 있으며 custom hook/API 모듈 분리는 미완료다.

### 백엔드 구조

- `LedgerService`가 거래, 검색, 예산, 통계, 명의와 자산 관리 책임을 함께 담당한다.
- 도메인별 서비스 분리와 검색 조건 객체 분리가 필요하다.

### 기타 후속 항목

- 검색 결과 페이지네이션과 정렬 선택은 아직 미완료다.
- 영수증 첨부 파일 정리 정책 고도화는 아직 미완료다.

## 7. 개발 메모

- 거래 화면은 표시 컴포넌트와 화면 전용 계산 helper를 함께 이동해 입력 도메인의 응집도를 높였다.
- API 요청, 폼 상태와 저장 정책은 App에 남겨 기존 비즈니스 흐름을 보존했다.
- 영수증 파일 객체와 할부 첨부 회차 상태는 props로 전달해 브라우저 파일 입력 동작을 그대로 유지했다.
