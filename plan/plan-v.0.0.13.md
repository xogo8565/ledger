# 가계부 앱 개발 진행 기록 v.0.0.13

작성일: 2026-06-23

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.12.md`
- 이번 개발 범위: v.0.0.13 권장 항목 중 검색/필터
- 검증 방식: Docker Compose 컨테이너 빌드

## 2. 이번 버전 개발 완료

### 월 내 거래 검색/필터

- 가계부 화면에 검색/필터 영역을 추가했다.
- 검색 대상:
  - 거래 제목
  - 메모
  - 카테고리명
  - 자산명
  - 거래일
  - 이체 자산 라벨
- 거래 유형 필터를 추가했다.
  - 전체
  - 수입
  - 지출
  - 이체
- 카테고리 필터를 추가했다.
- 유형을 선택하면 해당 유형에 맞는 카테고리만 표시되도록 했다.
- 활성 필터가 있을 때 결과 건수를 표시한다.
- 활성 필터가 있을 때 `초기화` 버튼을 표시한다.

### 필터 적용 범위

- 현재 월에 이미 불러온 거래 목록을 기준으로 클라이언트에서 필터링한다.
- 필터 적용 화면:
  - 일일
  - 달력
  - 요약의 거래 기반 목록
  - 메모
- 월별 요약 화면은 기존 월 합계 기준 표시를 유지한다.

### README 정리

- 구현된 MVP 목록에 월 내 거래 검색/필터를 반영했다.
- 다음 작업 목록에서 검색/필터 항목을 제거했다.

## 5. 아직 부분 완료인 항목

### 검색/필터

- 현재 월에 로드된 거래 기준 검색/필터는 구현되어 있다.
- 기간을 넘나드는 서버 검색, 금액 범위 필터, 자산 필터는 아직 미완료다.

### 카드 관리

- 법정 공휴일/대체공휴일 데이터 기반 결제일 보정은 아직 미완료다.
- 카드 결제 예정 금액과 할부 청구 월 연결 검증이 필요하다.

### 할부 관리

- 할부 생성, 조회, 그룹 단위 수정/삭제, 회차 수 변경은 구현되어 있다.
- 이미 확정/결제된 회차 보호 정책은 아직 없다.

### 통계

- 연별 통계, 기간 필터, 카테고리 선택 후 거래 목록 이동은 미완료다.

### 영수증 첨부

- 첨부 이미지 조회, 미리보기, 삭제, 다중 첨부 UI는 미완료다.

### 문자 자동 입력

- 파싱 결과를 거래 입력 폼으로 확정 저장하는 흐름은 미완료다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

## 8. 개발 메모

- 이번 검색/필터는 프론트엔드 클라이언트 필터로 구현했다.
- 월 범위를 넘어서는 검색이나 대용량 데이터 검색이 필요해지면 백엔드 검색 API로 확장하는 편이 좋다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드는 정상 통과했다.
