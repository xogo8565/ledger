# 편한가계부 신규 개발 진행 기록 v.0.1.4

작성일: 2026-06-26

## 1. 기준

- 이전 기록: `plan/plan-v.0.1.3.md`
- 이번 개발 범위: OCR 원문 수정 후 재파싱 UI 연결
- 배경: Tesseract OCR 결과는 이미지 품질에 따라 품명, 금액, 날짜가 흔들릴 수 있으므로 사용자가 원문을 직접 보정한 뒤 같은 파싱 규칙으로 다시 거래 후보를 계산할 수 있어야 한다.

## 2. 이번 버전 개발 완료

### OCR 원문 편집

- 영수증 OCR 결과 화면에 OCR 원문 편집용 텍스트 영역을 추가했다.
- 파일을 새로 선택하면 이전 OCR 결과와 편집 원문을 초기화한다.
- OCR 분석이 끝나면 서버가 반환한 `rawText`를 편집 원문 기본값으로 채운다.

### 수정 원문 재파싱

- OCR 화면에서 `수정한 원문으로 다시 분석` 버튼을 제공한다.
- 재파싱은 기존 문자/영수증 텍스트 파싱 API인 `/api/import/text/parse`를 재사용한다.
- 별도 OCR 전용 재파싱 API를 늘리지 않고 `ImportTextService`의 단일 파싱 규칙을 공유한다.
- 따라서 v0.1.3에서 추가한 `품명/단가/수량/금액` 우선 규칙도 수정 원문 재파싱에 그대로 적용된다.

### 거래 입력 반영

- 재파싱 성공 시 OCR 결과 화면의 날짜, 금액, 가맹점/품명, 추천 분류 후보를 최신 값으로 갱신한다.
- 이후 `거래 입력에 사용`을 누르면 재파싱된 최신 후보와 수정된 원문이 거래 입력 초안에 반영된다.
- OCR에 사용한 원본 파일은 기존처럼 거래 저장 시 영수증 첨부 후보로 유지한다.

## 4. 아직 부분 완료인 항목

### OCR UX

- OCR 실패/낮은 신뢰도 안내는 아직 더 보강할 수 있다.
- 현재는 서버 경고와 실패 메시지를 표시하지만, 수동 입력 전환 안내나 촬영 가이드는 더 구체화할 여지가 있다.

### 품목표 고도화

- 현재는 품목표의 첫 번째 품목을 거래 제목 후보로 사용한다.
- 여러 품목 요약, 대표 품명 선택, 품목 전체 메모화는 아직 미완료다.

### 실제 이미지 OCR 회귀

- 조건부 실제 이미지 OCR 테스트 파일은 있으나, 안정적인 fixture 이미지를 기반으로 한 고정 회귀 검증은 추가 개선 대상이다.

## 7. 개발 메모

- 재파싱은 새로운 백엔드 엔드포인트를 만들지 않고 기존 텍스트 파싱 API를 재사용했다.
- 이렇게 하면 문자 자동 입력, OCR 최초 분석, OCR 수정 원문 재분석이 같은 파싱 정책을 공유하므로 향후 영수증 규칙을 추가해도 중복 수정이 줄어든다.
- 사용자가 OCR 원문에서 `품명 단가 수량 금액` 같은 헤더나 품목 금액을 직접 보정하면 즉시 최신 거래 후보를 얻을 수 있다.
