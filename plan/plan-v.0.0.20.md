## 2. 이번 버전 개발 완료

### 기존 거래 수정 중 영수증 추가

- 거래 수정 화면에서도 영수증 파일 입력을 표시하도록 했다.
- 신규 거래 입력에서는 기존처럼 `영수증 사진`으로 표시하고, 기존 거래 수정에서는 `영수증 추가`로 표시하도록 했다.
- 거래 수정 저장 후 선택한 영수증 파일을 기존 거래 ID로 업로드하도록 했다.
- 업로드 대상 거래 ID를 찾지 못하면 사용자 알림을 표시하도록 했다.
- 영수증 업로드 실패 시 사용자 알림을 표시하고 수정 화면 상태를 유지하도록 했다.
- 할부 그룹 전체 수정은 여러 거래를 한 번에 수정하므로 이번 범위에서는 영수증 첨부 입력을 계속 숨긴다.

### 기존 흐름 유지

- 신규 거래 생성 후 영수증 첨부 흐름은 유지했다.
- 거래 상세 화면의 영수증 조회, 미리보기, 삭제 흐름은 유지했다.
- 백엔드 영수증 업로드 API는 기존 거래에도 이미 동작하므로 프론트 연결 중심으로 진행했다.

### README 정리

- 구현된 MVP 목록의 영수증 항목을 신규/기존 거래 첨부 기준으로 최신화했다.
- 다음 작업 목록에서 기존 거래 수정 중 영수증 추가 항목을 제거했다.
- 다음 우선순위를 카드사/은행별 문자 파싱 규칙 확장으로 갱신했다.

## 5. 아직 부분 완료인 항목

### 영수증 첨부

- 신규 거래 첨부, 기존 거래 수정 중 첨부, 조회, 미리보기, 삭제 UI는 구현되어 있다.
- 다중 첨부 일괄 업로드와 할부 그룹 수정 중 첨부 흐름은 아직 미완료다.

### 문자 자동 입력

- 원문 붙여넣기, 후보 추출, 거래 입력 폼 확정 이동은 구현되어 있다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

### 자산 명의/소비 태그

- 자산 명의와 소비 태그의 기본 저장/표시/검색은 구현되어 있다.
- 명의별 자산 집계, 태그별 통계, 태그 추천/관리 UI는 아직 미완료다.
- 반복 거래 규칙 자체에 소비 태그를 저장하는 구조는 아직 미완료다.

### 데이터 내보내기

- 월별/연별 거래 CSV 내보내기는 구현되어 있다.
- CSV에는 소비 태그가 포함되어 있다.
- 예산, 자산, 카테고리, 통계 요약의 통합 내보내기는 아직 미완료다.

### 통계

- 월별/연별 통계와 카테고리 선택 상세 이동은 구현되어 있다.
- 사용자 지정 시작일/종료일 기반 기간 필터는 아직 미완료다.
- 소비 태그별 통계는 아직 미완료다.

## 8. 개발 메모

- 기존 거래 수정 후 영수증 첨부는 거래 수정 API 응답의 ID 또는 수정 대상 거래 ID를 사용한다.
- 업로드 실패 시 거래 수정 자체는 이미 반영되었을 수 있으므로 화면을 유지하고 사용자에게 실패를 알린다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드와 Gradle 테스트는 정상 통과했다.
