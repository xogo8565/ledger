## 2. 이번 버전 개발 완료

### 다중 영수증 업로드 API

- `POST /api/transactions/{transactionId}/receipts/batch` API를 추가했다.
- multipart 요청의 `files` 항목으로 여러 이미지를 한 번에 받는다.
- 한 요청에서 최대 10장까지 업로드할 수 있다.
- 빈 파일과 이미지가 아닌 파일은 저장 전에 거부한다.
- 기존 단일 영수증 업로드 API는 호환성을 위해 유지했다.
- 단일 업로드도 공통 다중 저장 로직을 사용하도록 정리했다.
- 일괄 저장 도중 오류가 발생하면 이번 요청에서 생성한 물리 파일을 삭제한다.
- 파일 저장 예외 발생 시 데이터베이스 트랜잭션도 롤백하도록 설정했다.
- 최대 요청 크기를 20MB에서 50MB로 확장했고, 개별 파일 최대 크기 10MB는 유지했다.

### 신규/기존 거래 영수증 UI

- 거래 입력과 기존 거래 수정 화면의 파일 입력에 다중 선택을 적용했다.
- 선택한 파일은 최대 10장까지만 유지한다.
- 선택 파일명을 목록으로 표시한다.
- 저장 전 선택 목록에서 파일을 개별 제외할 수 있다.
- 거래 저장 후 선택한 파일 전체를 한 번의 일괄 업로드 요청으로 전송한다.
- 일괄 첨부 실패 시 파일 형식과 개수를 확인하도록 안내한다.
- 할부 그룹 전체 수정에서는 기존처럼 영수증 입력을 숨긴다.

### 회귀 테스트

- 이미지 2장을 한 요청으로 저장하는 서비스 테스트를 추가했다.
- 10장을 초과하는 요청이 파일 저장 전에 거부되는지 검증했다.
- 이미지가 아닌 파일이 거부되는지 검증했다.
- 기존 문자 파싱, 공휴일, 소비 태그 집계 테스트를 포함한 전체 테스트를 통과했다.

### README 정리

- 구현된 MVP의 영수증 항목을 신규/기존 거래 다중 첨부 기준으로 최신화했다.
- 다음 작업에서 다중 영수증 일괄 업로드를 제거했다.
- 할부 그룹 수정 중 영수증 첨부는 별도 후속 항목으로 유지했다.

## 5. 아직 부분 완료인 항목

### 영수증 첨부

- 신규/기존 거래의 최대 10장 일괄 첨부, 조회, 미리보기, 개별 삭제는 구현되어 있다.
- 할부 그룹 전체 수정 중 첨부 대상을 어느 회차로 할지 정하는 정책과 UI는 아직 미완료다.
- 업로드 전 브라우저 이미지 미리보기와 촬영/갤러리 선택 구분 UI는 아직 미완료다.

### 문자 자동 입력

- 카드 승인, 체크카드 출금, 은행 입금, 취소/환불 기본 규칙은 구현되어 있다.
- 문자 파싱 기반 카테고리 추천과 사용자 정의 파싱 규칙 관리는 아직 미완료다.
- 거래 추가 버튼에서 직접 입력/문자 자동 입력을 선택하는 진입 UX는 아직 미완료다.
- 더보기에서 직접 문자를 붙여넣는 형식은 삭제 처리 한다.
- 거래 추가 버튼에서 문자 자동 입력 클릭시 추출된 텍스트가 직접 입력 화면에 자동 입력되도록 진행 / 붙여넣을 데이터 없을 시 alert 창 띄우기

### 통계

- 월별/연별 카테고리 및 소비 태그 통계는 구현되어 있다.
- 기간 필터 기준 요약 금액 재계산은 아직 미완료다.

### 자산 명의/공동 가계부

- 자산 명의 저장과 표시, Household/Member 도메인은 구현되어 있다.
- 명의별 자산 집계 / 소비 시 명의를 구분할 수 있도록 항목 추가(공동 , 개인 소비 구분)
- 명의 관리 위한 메뉴 추가(더보기/ member 도메인 / 인증, 권한 인가 등 기능은 불필요)

## 8. 개발 메모

- 파일 선택 시 브라우저에서 최대 10장만 상태에 보관하고 서버에서도 같은 제한을 검증한다.
- 데이터베이스 롤백만으로 물리 파일이 삭제되지는 않으므로 서비스에서 생성 경로를 추적해 예외 시 정리한다.
- 거래 저장 성공 후 영수증 업로드가 실패하면 거래 자체는 이미 반영될 수 있으며, 화면을 유지하고 실패를 안내한다.
- 브라우저 스모크 테스트 후 Docker Compose 스택은 실행 상태로 유지되어 있다.
