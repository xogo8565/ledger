## 2. 이번 버전 개발 완료

### 명의별 자산 요약 API

- 기존 `GET /api/assets/summary` 응답에 명의별 요약 목록을 추가했다.
- 각 명의별 자산 합계, 부채 합계, 순자산, 자산 수를 반환한다.
- 현금, 은행, 기타 자산은 자산 합계에 포함한다.
- 카드와 부채 자산은 절대값 기준 부채 합계에 포함한다.
- 순자산은 자산 합계에서 부채 합계를 차감해 계산한다.
- 전체 자산·부채·순자산 계산과 명의별 계산이 같은 자산 목록을 사용한다.

### 명의 정규화

- 명의 앞뒤 공백을 제거한다.
- 명의 안의 연속 공백을 하나로 정규화한다.
- 명의가 없거나 공백만 있으면 `명의 미지정`으로 집계한다.
- `명의 미지정`은 명의 목록의 마지막에 표시한다.
- 그 외 명의는 순자산이 큰 순서로 정렬하고, 같으면 명의명 순으로 정렬한다.

### 자산 화면

- 자산 화면이 bootstrap과 자산 요약 API를 함께 불러오도록 했다.
- 기존 전체 자산·부채·합계 영역은 자산 전용 요약 API 값을 사용한다.
- 전체 요약 아래에 `명의별 자산` 영역을 추가했다.
- 각 명의별 자산 수, 자산, 부채, 순자산을 표시한다.
- 명의별 데이터가 없으면 전용 빈 상태를 표시한다.
- 기존 자산 그룹별 목록과 자산 행의 명의 표시는 유지했다.

### 회귀 테스트

- 본인 계좌, 본인 부채, 공동 현금, 명의 미지정 카드를 이용한 집계 단위 테스트를 추가했다.
- 공백이 포함된 동일 명의가 하나로 합쳐지는지 검증했다.
- 카드/부채의 절대값 합계와 순자산 계산을 검증했다.
- 브라우저에서 고유 명의의 자산을 생성한 뒤 해당 명의 요약 행이 나타나는지 검증했다.

### README 정리

- 구현된 자산 명의 항목에 명의별 자산·부채·순자산 집계를 반영했다.
- 다음 작업에서 명의별 자산 집계를 제거했다.
- 거래 공동/개인 소비 구분과 명의 관리 메뉴를 다음 우선순위로 유지했다.

## 5. 아직 부분 완료인 항목

### 자산 명의/공동 가계부

- 자산 명의 저장, 표시, 명의별 자산 집계는 구현되어 있다.
- 거래 자체에 소비 명의와 공동/개인 소비 구분을 저장하는 구조는 아직 미완료다.
- 더보기에서 명의를 추가·수정·삭제하는 관리 메뉴는 아직 미완료다.
- 현재 자산 명의는 자유 입력 문자열이며 Member 도메인과 직접 연결되지 않는다.
- 인증과 권한 인가는 현재 개인용 범위에서 제외한다.

### 문자 자동 입력

- 클립보드 문자 분석, 카테고리 추천, 거래 폼 자동 입력은 구현되어 있다.
- 사용자 정의 문자 파싱·추천 규칙 관리는 아직 미완료다.

### 영수증 첨부

- 신규/기존 거래의 최대 10장 일괄 첨부, 조회, 미리보기, 개별 삭제는 구현되어 있다.
- 할부 그룹 전체 수정 중 영수증 첨부 정책과 UI는 아직 미완료다.

### 기간 필터

- 기간 조회, 목록 필터, 결과 합계, 요약, CSV 내보내기는 구현되어 있다.
- 기간 필터 기반 통계는 아직 미완료다.

## 8. 개발 메모

- 명의별 집계는 현재 Asset의 `ownerName` 문자열을 정규화해 계산한다.
- 카드 자산 balance가 0인 현재 데이터에서는 카드 미결제 금액이 명의별 부채에 포함되지 않는다. 명의별 카드 미결제 집계는 카드 도메인 확장 과제다.
- 자산 생성·수정·삭제 후 공통 `load()`가 자산 요약 API도 다시 호출하므로 화면이 즉시 갱신된다.
- 브라우저 스모크 테스트 후 Docker Compose 스택은 실행 상태로 유지되어 있다.
