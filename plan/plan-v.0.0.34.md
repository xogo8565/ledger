# 편한가계부 후속 개발 진행 기록 v.0.0.34

작성일: 2026-06-24

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.33.md`
- 이번 개발 범위: v.0.0.34 권장 항목 1순위인 공동/개인 소비 통계
- 검증 방식: Docker Compose 이미지 빌드, Gradle 전체 테스트, Playwright 브라우저 스모크 테스트

## 2. 이번 버전 개발 완료

### 소비 구분별 통계 API

- 월별 요약 응답에 `scopeSpends`를 추가했다.
- 연별 요약 응답에도 같은 소비 구분 집계를 추가했다.
- 각 집계는 `PERSONAL` 또는 `SHARED` 구분, 지출 합계, 거래 건수를 제공한다.
- 수입과 이체는 소비 통계에서 제외한다.
- 소비 구분이 없는 기존 지출은 도메인 기본 정책에 따라 개인 소비로 집계한다.
- 지출 금액 내림차순으로 개인/공동 소비 항목을 정렬한다.

### 월별·연별 통계 UI

- 지출 통계 기준에 `개인/공동` 탭을 추가했다.
- 개인 소비와 공동 소비 각각의 지출 비율, 거래 건수, 금액을 표시한다.
- 월별 통계와 연별 통계에서 동일한 소비 구분 집계를 확인할 수 있다.
- 공동 소비는 별도 색상을 사용해 개인 소비와 쉽게 구분한다.
- 데이터가 없으면 전용 빈 상태 안내를 표시한다.

### 거래 필터 연동

- 가계부 필터에 `소비 구분` 선택 항목을 추가했다.
- 통계의 개인 또는 공동 항목을 누르면 지출 유형과 해당 소비 구분을 자동 선택해 거래 목록으로 이동한다.
- 소비 구분 필터는 검색·카테고리·기간 필터와 함께 사용할 수 있다.
- 수입 또는 이체 유형을 선택하면 소비 구분 필터를 자동으로 비운다.
- 필터 결과 합계와 CSV 내보내기는 선택된 소비 구분 결과를 그대로 사용한다.

### 테스트

- 개인/공동 지출의 금액과 건수를 집계하고 수입을 제외하는 단위 테스트를 추가했다.
- 월별 API에 개인·공동 집계가 모두 포함되는지 브라우저 테스트에서 검증한다.
- 통계 화면에 개인·공동 항목이 표시되는지 검증한다.
- 공동 소비 통계 선택 시 가계부의 `SHARED` 필터와 해당 거래 목록이 열리는지 검증한다.

## 4. 아직 부분 완료인 항목

### 명의별 소비 통계

- 개인 소비 거래에는 Member 명의가 연결되고 개인/공동 통계도 제공된다.
- 개인 소비를 실제 명의별로 나눈 금액·건수 통계는 아직 미완료다.
- 다음 버전에서 월별·연별 `memberSpends` 집계와 명의별 거래 필터를 추가할 수 있다.

### 기존 개인 지출 데이터

- 소비 명의가 없는 기존 개인 지출도 개인 소비 총액에는 포함된다.
- 명의별 통계 구현 시에는 미지정 명의를 별도 항목으로 보여주거나 OWNER로 일괄 마이그레이션하는 정책 결정이 필요하다.

### 기타 후속 항목

- 할부 그룹 전체 수정 중 영수증 첨부 정책과 UI는 미완료다.
- 사용자 정의 문자 파싱·추천 규칙 관리는 미완료다.
- 사용자 지정 기간 기반 통계는 미완료다.

## 7. 개발 메모

- 소비 구분 통계는 거래에 저장된 `ConsumptionScope`를 기준으로 집계하므로 표시 문자열 변경과 무관하다.
- 기존 지출의 null 소비 구분은 `TransactionRecord.getConsumptionScope()`에서 개인 소비로 정규화한다.
- 실행 중인 Docker Compose 스택은 최신 이미지로 재생성했으며 프론트엔드는 `http://localhost:8081`에서 확인할 수 있다.
