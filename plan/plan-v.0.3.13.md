# 편한가계부 초기 명의 및 plan 이미지 기반 자산 엑셀 반영 개발 기록 v.0.3.13

## 개발 목표

- 초기 DB 생성 시 명의를 `석수`, `유진`으로 구분한다.
- `석수`를 기본 명의로 사용한다.
- `plan/*.png` 참고 이미지에 있는 계좌/대출/카드 항목을 기본 자산으로 등록할 수 있도록 초기 자산 엑셀 파일에 반영한다.

## 변경 사항

### 초기 명의 생성

- DB 첫 실행 초기화 시 기본 가계부 회원을 다음과 같이 생성한다.
  - `석수`: `OWNER`
  - `유진`: `EDITOR`
- 초기 거래의 기본 소유자도 `석수`로 연결한다.

### 초기 자산 엑셀 명의 컬럼

- 자산 초기화 엑셀에서 `J` 컬럼을 `명의`로 사용하도록 처리했다.
- `명의` 값이 비어 있거나 등록 정책과 맞지 않으면 기본값 `석수`로 저장한다.
- 현재 허용 명의는 `석수`, `유진`이다.

### plan 이미지 기반 자산 엑셀 추가

- 신규 파일을 추가했다.
  - `backend/src/main/resources/initial-data/assets_plan_20260629.xlsx`
- 반영 항목:
  - 하나은행 은행재원(일시상환) 대출
  - 하나은행 납부계좌
  - 청년 주택드림 청약통장
  - 삼성증권 중개형ISA
  - 삼성증권 종합매매
  - 삼성증권 집합투자증권
  - 삼성증권 연금저축
  - 토스뱅크 저축용
  - NH농협 보험 이체용
  - 토스뱅크 통장
  - NH1934우대통장
  - Discount Plan+
  - KB국민 코웨이III 카드
  - K-패스 삼성카드
  - 삼성카드 taptap O
  - 현대카드M CHECK-경차전용카드(경형승용)
  - 삼성후불하이패스카드

이미지에는 유진 명의로 식별할 수 있는 표시가 없어서, 위 자산은 모두 기본 명의 `석수`로 입력했다. 유진 명의 자산이 필요하면 엑셀 `J` 컬럼을 `유진`으로 수정하면 된다.

### 자산 타입 추론 보강

- `대출`, `차입`, `상환` 포함 자산은 `DEBT`로 처리한다.
- `카드` 포함 자산은 `CARD`로 처리한다.
- `현금` 포함 자산은 `CASH`로 처리한다.
- `증권`, `ISA`, `종합매매`, `집합투자`, `연금저축` 포함 자산은 `OTHER`, 그룹명 `증권`으로 처리한다.
- 그 외는 `BANK`, 그룹명 `계좌`로 처리한다.

## 검증

- 신규 xlsx 파일을 `InitialDataWorkbookReader`로 읽는 테스트를 추가했다.
- 백엔드 전체 테스트 통과.

```text
docker run --rm -v "${PWD}\backend:/workspace" -w /workspace gradle:8.10.2-jdk21 gradle test --no-daemon
BUILD SUCCESSFUL
```

## 운영 반영 참고

- 초기 데이터 주입은 빈 DB에서만 실행된다.
- 이미 운영 DB가 생성된 상태라면 이번 엑셀 추가만으로 기존 DB에 자동 재주입되지 않는다.
- 운영에 즉시 반영하려면 다음 중 하나가 필요하다.
  - DB 초기화 후 재기동
  - 자산 화면에서 수동 등록
  - 별도 마이그레이션/관리 API로 기존 DB에 추가 주입
