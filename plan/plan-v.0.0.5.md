## 2. 이번 버전 개발 완료

### 카드 결제 중복 실행 방지

- 카드 결제 예약 상태에 `PROCESSING`을 추가했다.
- 결제 실행 시 예약 상태를 먼저 `PROCESSING`으로 전환한 뒤 실제 거래 생성을 수행하도록 변경했다.
- 카드 결제 예약 엔티티에 JPA `@Version` 필드를 추가해 동시에 같은 예약을 실행하는 경우 낙관적 잠금으로 중복 처리를 줄이도록 했다.
- `SCHEDULED` 상태가 아닌 결제 예약은 실행할 수 없도록 기존 방어 로직을 유지했다.

### 카드 결제 실패 처리

- 카드 결제 실행 중 결제 계좌 미설정 등 런타임 오류가 발생하면 예약 상태를 `FAILED`로 저장하도록 변경했다.
- 실패 사유를 `failureReason`에 남긴다.
- 실패 사유는 너무 긴 메시지가 저장되지 않도록 500자까지 제한한다.
- 자동 결제 스케줄러는 실행 결과가 `FAILED`이면 실패 사유를 로그에 남기고 다음 예약 처리를 계속한다.

### 카드 결제 UI 상태 표시

- 카드 결제 예약 화면에서 상태 코드를 사용자용 라벨로 표시한다.
  - `SCHEDULED`: 예약됨
  - `PROCESSING`: 처리 중
  - `COMPLETED`: 완료
  - `FAILED`: 실패
- 처리 중인 예약은 실행/취소 버튼을 숨긴다.
- 실패한 처리 내역에는 실패 사유를 함께 표시한다.
- 완료된 처리 내역에는 완료일을 표시한다.

### README 정리

- 구현된 MVP 목록에 카드 결제 중복 실행 방지와 실패 사유 표시를 반영했다.

## 5. 아직 부분 완료인 항목

### 카드 관리

- 카드 결제 예약과 실행은 구현되어 있다.
- 카드 자산 생성/수정 시 카드 프로필 입력과 저장 API 연결은 구현되어 있다.
- 자동 결제 중복 실행 방지는 `PROCESSING` 상태와 낙관적 잠금으로 보강했다.
- 실패 상태 저장과 UI 표시는 구현되어 있다.
- 실제 청구 기간, 말일 처리, 휴일 처리 정책은 보강이 필요하다.
- 실패한 카드 결제를 사용자가 다시 예약하거나 재시도하는 UI는 아직 미완료다.
- 카드 결제 예정 금액을 사용자가 이해하기 쉽게 보여주는 UI 설명도 더 필요하다.

### 할부 관리

- 할부 생성과 조회는 구현되어 있다.
- 할부 그룹 전체 수정/삭제는 아직 미완료다.
- 카드 결제 예정 금액과 할부 청구 월 연결 검증이 필요하다.

### 예산

- 월별 전체/카테고리 예산 저장은 구현되어 있다.
- 카테고리별 예산 사용률, 초과 표시, 전월 예산 복사는 미완료다.

### 통계

- 월별 및 카테고리별 지출 통계는 구현되어 있다.
- 연별 통계, 기간 필터, 카테고리 선택 후 거래 목록 이동은 미완료다.

### 영수증 첨부

- 신규 거래 첨부와 업로드 API는 구현되어 있다.
- 첨부 이미지 조회, 미리보기, 삭제, 다중 첨부 UI는 미완료다.

### 문자 자동 입력

- 원문 붙여넣기와 후보 추출은 구현되어 있다.
- 파싱 결과를 거래 입력 폼으로 확정 저장하는 흐름은 미완료다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

### 공동 가계부

- Household/Member 도메인은 있다.
- 인증, 초대, 권한, 사용자 전환 UI는 미완료다.
- 인증 등, 권한, 사용자 전환 등 기능은 불필요하지만, 자산 명의 / 관련 소비 등에 태그될 수 있도록 구성한다.

## 8. 개발 메모

- 카드 결제 예약 엔티티에 `version` 컬럼이 추가된다. 운영 DB에서는 JPA ddl-auto 정책 또는 별도 마이그레이션 전략을 확인해야 한다.
- 결제 실패 시 예약 상태는 `FAILED`로 남지만, 현재 UI에서는 실패 건을 즉시 재시도하는 기능은 제공하지 않는다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드는 정상 통과했다.
