# 편한가계부 후속 개발 진행 기록 v.0.0.44

작성일: 2026-06-25

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.43.md`
- 이번 개발 범위: v.0.0.44 권장 항목 1순위인 자산·카드 결제 화면 컴포넌트 분리
- 검증 방식: Gradle 전체 테스트, Docker Compose 이미지 빌드, Playwright 브라우저 스모크 테스트

## 2. 이번 버전 개발 완료

### 자산 화면 모듈 분리

- `frontend/src/screens/AssetsScreen.jsx`를 추가했다.
- 다음 자산 화면 구성요소를 `main.jsx`에서 이동했다.
  - 전체 자산·부채·순자산 합계
  - 명의별 자산 요약
  - 자산 그룹별 목록과 그룹 합계
  - 자산 수정·삭제 진입
  - 카드 자산 결제 관리 진입
- 자산 데이터와 생성·수정·삭제 callback은 기존 App에서 전달한다.

### 카드 결제 관리 화면 분리

- 카드 결제 상세와 예약 관리 화면을 자산 모듈로 이동했다.
- 미결제 금액, 결제 예정 금액, 청구 기간과 결제일 보정 안내를 유지했다.
- 결제 예약 생성, 실행, 취소, 실패 재시도와 재예약 callback을 App에서 전달한다.
- 예약·처리 내역 분리와 상태 표시 helper를 자산 모듈 내부로 이동했다.

### 자산 생성·수정 화면 분리

- `AssetFormScreen`을 자산 모듈로 이동했다.
- 일반 자산과 카드 자산의 공통 입력 및 카드 결제 계좌·확정일·결제일·자동 결제 설정을 함께 이동했다.
- 명의 선택과 기존 미등록 명의 안내 흐름을 유지했다.
- 자산 유형 라벨을 자산 모듈에서 export해 App의 폼 초기화에서도 같은 정의를 사용한다.

### 공통 입력 필드 정리

- `LineField`를 `components/ui.jsx`로 이동했다.
- 거래 입력, 반복 거래, 카드 결제, 자산 편집 등 여러 화면이 같은 입력 행 컴포넌트를 사용한다.
- 자산 유형 선택지는 자산 모듈 내부에 두어 자산 화면 책임을 명확히 했다.

### 파일 규모 변화

- `main.jsx`는 약 1,945줄에서 약 1,674줄로 감소했다.
- 자산·카드 결제·자산 폼 표시 로직 약 270줄을 `AssetsScreen.jsx`로 이동했다.
- 프론트엔드 빌드 모듈 수는 37개로 증가했다.

## 4. 아직 부분 완료인 항목

### 프론트엔드 구조

- 거래 입력과 거래 상세 화면은 아직 `main.jsx`에 남아 있다.
- 반복 거래와 할부 일정 화면도 아직 `main.jsx`에 남아 있다.
- 카드 결제·자산 API 요청과 상태 로직은 여전히 App에 있으며 hook/API 모듈 분리는 미완료다.

### 백엔드 구조

- `LedgerService`가 거래, 검색, 예산, 통계, 명의와 자산 관리 책임을 함께 담당한다.
- 도메인별 서비스 분리와 검색 조건 객체 분리가 필요하다.

### 기타 후속 항목

- 검색 결과 페이지네이션과 정렬 선택은 아직 미완료다.
- 영수증 첨부 파일 정리 정책 고도화는 아직 미완료다.

## 7. 개발 메모

- 자산 화면은 표시와 폼을 하나의 도메인 화면 모듈에 배치하되 API 호출은 App에 유지했다.
- 자산 유형 라벨과 선택지를 자산 모듈에서 관리해 자산 관련 표시 정의가 다른 화면으로 퍼지지 않게 했다.
- 공통 입력 행을 먼저 분리해 다음 거래 입력·반복 거래 화면 이동 시 중복 없이 재사용할 수 있게 했다.
