- 검증 방식: Docker Compose 컨테이너 빌드, Gradle 테스트 컨테이너 실행

## 2. 이번 버전 개발 완료

### 자산 명의 구조

- `Asset` 도메인에 `ownerName` 필드를 추가했다.
- 자산 생성/수정 요청과 응답 DTO에 `ownerName`을 포함했다.
- 일반 자산과 카드 자산 생성/수정 시 명의 값을 저장하도록 했다.
- 자산 생성/수정 화면에 `명의` 입력 필드를 추가했다.
- 자산 목록에서 명의가 있으면 자산 유형 앞에 함께 표시하도록 했다.

### 소비 태그 구조

- `TransactionRecord` 도메인에 `spendingTag` 필드를 추가했다.
- 거래 생성/수정 요청과 응답 DTO에 `spendingTag`를 포함했다.
- 지출 거래 입력 화면에 태그 입력 필드를 추가했다.
- 거래 상세 화면에서 소비 태그를 표시하도록 했다.
- 거래 검색 대상에 소비 태그를 포함했다.
- 거래 CSV 내보내기 컬럼에 소비 태그를 추가했다.
- 이체/수입 유형으로 바꿀 때 소비 태그를 초기화하도록 했다.

### 기존 자동 생성 거래 호환

- 카드 자동 결제 거래 생성 시 소비 태그는 비워 두도록 했다.
- 반복 거래 만기 생성 시 소비 태그는 비워 두도록 했다.
- 문자 파싱 확정 흐름은 신규 거래 기본값 기준으로 폼을 채워 이전 태그가 섞이지 않도록 정리했다.

### README 정리

- 구현된 MVP 목록에 자산 명의 저장/표시와 지출 소비 태그 기능을 추가했다.
- 다음 작업 목록에서 자산 명의/소비 태그 구조 항목을 제거했다.
- 다음 우선순위를 기존 거래 수정 중 영수증 추가로 갱신했다.

## 5. 아직 부분 완료인 항목

### 자산 명의/소비 태그

- 자산 명의와 소비 태그의 기본 저장/표시/검색은 구현되어 있다.
- 명의별 자산 집계, 태그별 통계, 태그 추천/관리 UI는 아직 미완료다.
- 반복 거래 규칙 자체에 소비 태그를 저장하는 구조는 아직 미완료다.

### 데이터 내보내기

- 월별/연별 거래 CSV 내보내기는 구현되어 있다.
- CSV에는 소비 태그가 포함되어 있다.
- 예산, 자산, 카테고리, 통계 요약의 통합 내보내기는 아직 미완료다.

### 통계

- 월별/연별 통계와 카테고리 선택 상세 이동은 구현되어 있다.
- 사용자 지정 시작일/종료일 기반 기간 필터는 아직 미완료다.
- 소비 태그별 통계는 아직 미완료다.

### 영수증 첨부

- 신규 거래 첨부와 업로드 API는 구현되어 있다.
- 첨부 이미지 조회, 미리보기, 삭제 UI는 구현되어 있다.
- 기존 거래 수정 중 신규 영수증 추가와 다중 첨부 흐름 고도화는 아직 미완료다.

### 문자 자동 입력

- 원문 붙여넣기, 후보 추출, 거래 입력 폼 확정 이동은 구현되어 있다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

## 8. 개발 메모

- `spring.jpa.hibernate.ddl-auto=update` 설정으로 신규 문자열 컬럼은 컨테이너 기동 시 자동 반영된다.
- 소비 태그는 현재 단일 문자열 필드이며, 별도 태그 마스터 테이블은 두지 않았다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드와 Gradle 테스트는 정상 통과했다.
