# 편한가계부 신규 개발 진행 기록 v.0.1.2

작성일: 2026-06-26

## 1. 기준

- 이전 기록: `plan/plan-v.0.1.1.md`
- 이번 개발 범위: OCR 텍스트 샘플 회귀 테스트 추가 및 영수증 특화 파싱 1차 고도화
- 검증 방식: Gradle 전체 테스트, Docker Compose 이미지 빌드, Playwright 브라우저 스모크 테스트

## 2. 이번 버전 개발 완료

### 영수증 OCR 텍스트 파싱 고도화

- `ImportTextService`에 영수증 합계 금액 우선 추출 로직을 추가했다.
- 품목 금액이 먼저 등장하는 영수증 OCR 텍스트에서도 총액을 거래 금액으로 선택하도록 했다.
- 지원하는 총액 키워드:
  - `합계`
  - `합 계`
  - `총액`
  - `총결제금액`
  - `결제금액`
  - `받을금액`
  - `매출금액`
- 영수증 OCR 텍스트가 여러 줄일 때 첫 유효 라인을 가맹점 후보로 선택하도록 했다.
- 다음과 같은 영수증 노이즈 라인은 가맹점 후보에서 제외한다.
  - 영수증/매출전표
  - 사업자/대표/주소/전화
  - 품명/단가/수량/금액
  - 합계/총액/결제/승인/카드
  - 과세/부가세/면세/받은금액
- 영수증 형태의 입력에는 메모를 `영수증 OCR 자동 입력 후보`로 표시하도록 했다.

### OCR 텍스트 회귀 테스트

- `ImportTextServiceTest`에 영수증 OCR 샘플 텍스트 기반 회귀 테스트를 추가했다.
- 품목 금액보다 합계 금액을 우선 선택하는지 검증한다.
- `합 계 : 3,300`처럼 OCR에서 공백이 섞인 총액 키워드도 인식하는지 검증한다.
- 첫 유효 라인인 가맹점명이 제목 후보로 추출되는지 검증한다.
- 기존 카드/체크카드/입금/승인취소 문자 자동 입력 테스트가 계속 통과하는지 확인했다.

## 4. 아직 부분 완료인 항목

### 실제 이미지 기반 OCR 회귀

- 이번 단계는 Tesseract 결과와 유사한 텍스트 샘플을 고정한 테스트다.
- 실제 이미지 파일을 Tesseract로 처리해 결과를 검증하는 회귀 테스트는 아직 없다.
- 실제 이미지 회귀는 테스트 환경의 Tesseract 설치 여부와 이미지 품질 영향을 받으므로 별도 fixture/조건부 테스트 설계가 필요하다.

### OCR 파싱 추가 고도화

- 카드 승인번호, 사업자번호, 전화번호, 주소 라인 제거는 1차로 반영했지만 케이스가 더 필요하다.
- 할인, 봉사료, 부가세, 면세, 결제수단별 금액, 포인트 사용액 등은 아직 정교하게 처리하지 않는다.
- 여러 날짜가 있는 영수증에서 거래일/발행일/승인일을 구분하는 정책도 추가 검토가 필요하다.

### OCR UX

- 낮은 신뢰도나 빈 OCR 텍스트를 더 명확히 안내하는 UX는 아직 미완료다.
- 사용자가 OCR 원문을 수정해 재파싱하는 UI도 아직 없다.

## 7. 개발 메모

- 실제 Tesseract OCR 정확도 자체는 이미지 품질에 따라 흔들리므로, 우선 OCR 이후 텍스트 파싱 계층을 안정화했다.
- 영수증 OCR 텍스트는 품목 금액이 총액보다 먼저 등장하는 경우가 많아 합계 키워드 기반 금액 추출이 중요하다.
- 이번 변경은 기존 문자 자동 입력 파서와 같은 서비스에 들어갔기 때문에 카드 문자 회귀 테스트도 함께 유지했다.
