# 편한가계부 다건 거래 가져오기 검토/선택 화면 개발 기록 v.0.3.19

작성일: 2026-07-01

## 1. 기준

- 이전 신규 개발항목 정리: `plan/new-development-items.md`
- 이번 개발 범위: P1 `다건 거래 가져오기 검토/선택 화면`
- 제외 범위: `→` 문구의 카드 결제 출금, 보험료, 내부 계좌 이체 판정 규칙 고도화는 추가 정책이 필요해 후속으로 남긴다.

## 2. 구현 내용

### 저장 전 다건 검토 화면 추가

- `ManualTextImportReviewScreen`을 추가했다.
- 문자/거래 목록 분석 결과에 `items`가 여러 건이면 바로 거래 입력으로 이동하지 않고 검토 화면을 먼저 연다.
- 검토 화면에서는 전체 분석 건수, 저장 예정 건수, 확인 필요 건수를 보여준다.
- 사용자는 항목별 체크박스로 저장 대상 거래를 선택하거나 제외할 수 있다.
- 전체 선택/전체 제외 버튼을 제공한다.

### 매칭 실패 경고 표시

- 다건 후보별로 자산명 매칭 상태를 계산한다.
- 일반 수입/지출 후보에서 자산명이 없거나 기존 자산과 매칭되지 않으면 경고를 표시한다.
- 이체 후보는 `→` 기준으로 출금/입금 자산 매칭을 시도하고, 실패한 쪽을 경고로 표시한다.
- 추천 분류가 없는 수입/지출 후보는 `분류 확인 필요` 경고를 표시한다.

### 선택 항목 순차 입력

- 검토 화면에서 선택한 항목만 순차 입력 대기열로 만든다.
- 첫 번째 선택 항목을 거래 입력 화면에 채우고, 저장 성공 후 다음 항목을 자동으로 표시한다.
- 기존 순차 입력 진행 표시를 유지한다.

### 순차 입력 건너뛰기

- 거래 입력 화면의 자동 입력 진행 카드에 `건너뛰기` 버튼을 추가했다.
- 현재 항목을 저장하지 않고 다음 대기열 항목으로 이동한다.
- 마지막 항목에서 건너뛰면 패널을 닫고 목록을 다시 불러온다.

## 5. 남은 과제

1. `→` 문구의 카드 결제 출금, 보험료, 내부 계좌 이체 판정 규칙을 실제 자산 데이터와 연결해 고도화한다.
2. 검토 화면에서 항목별 자산/분류를 직접 수정하는 기능은 아직 없다. 필요하면 선택 화면을 편집 화면으로 확장한다.
3. 다건 가져오기 브라우저 스모크 테스트를 추가하면 회귀 검증 신뢰도를 높일 수 있다.
