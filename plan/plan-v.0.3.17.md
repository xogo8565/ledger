# 편한가계부 plan 카드 확정일/결제일 엑셀 컬럼화 개발 기록 v.0.3.17

## 개발 목표

- `assets_plan_20260629.xlsx` 카드 자산 주입 시 확정일/결제일을 코드 고정값으로 처리하지 않는다.
- 기존 엑셀 파일에 확정일/결제일 항목을 추가하고 해당 값으로 주입한다.
- 같은 이름의 기존 카드 자산이 있어도 코드 기본값이 아니라 엑셀 값 기준으로 갱신한다.

## 변경 사항

### 엑셀 컬럼 추가

- `assets_plan_20260629.xlsx`에 다음 컬럼을 추가했다.
  - `K`: 확정일
  - `L`: 결제일
- 현재 카드 행 값:
  - 확정일: `16`
  - 결제일: `25`

이후 확정일/결제일을 변경하려면 코드 수정 없이 엑셀 K/L 컬럼 값만 변경하면 된다.

### 코드 고정값 제거

- 기존 `PLAN_CARD_STATEMENT_CLOSING_DAY = 16`, `PLAN_CARD_PAYMENT_DAY = 25` 상수를 제거했다.
- 카드 프로필 생성/갱신 시 `K`, `L` 컬럼 값을 읽어 사용한다.
- 값은 `1~31` 범위로 검증한다.

### 유지되는 정책

- `assets_plan_20260629.xlsx` 카드 자산 금액은 `0`으로 주입한다.
- 결제계좌는 엑셀에서 특정하지 않으므로 기존과 동일하게 신규 생성 시 미지정으로 둔다.
- 자동결제는 신규 생성 시 `false`, 기존 프로필이 있으면 기존 값을 유지한다.

## 검증

- `InitialDataWorkbookReaderTest`에 `K=확정일`, `L=결제일` 헤더 검증을 추가했다.
- 카드 행의 확정일 `16`, 결제일 `25` 파싱 검증을 추가했다.

## 운영 참고

- `initial-data` 변경 감지 정책에 따라 `assets_plan_20260629.xlsx` checksum이 바뀌면 서버 재시작 시 재주입된다.
- 기존 같은 이름 카드 자산이 있으면 카드 금액은 `0`, 확정일/결제일은 엑셀 K/L 컬럼 값으로 갱신된다.
