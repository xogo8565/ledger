## 2. 이번 버전 개발 완료

### 기존 개인 지출 명의 마이그레이션 정책

- 지출 거래이면서 소비 구분이 개인이고 소비 명의가 없는 거래를 마이그레이션 대상으로 정의했다.
- 소비 구분 컬럼이 없던 시기의 기존 지출도 현재 도메인 정책상 개인 지출로 간주해 대상에 포함한다.
- 공동 지출, 수입, 이체, 이미 소비 명의가 있는 거래는 대상에서 제외한다.
- 대상 거래는 현재 Household의 `OWNER` 역할 명의로 연결한다.
- 이미 처리된 거래는 다시 대상이 되지 않아 반복 실행해도 추가 변경이 없는 멱등 정책으로 구현했다.

### 마이그레이션 API

- `GET /api/members/consumer-migration`으로 OWNER 명의와 대상 건수를 조회할 수 있다.
- `POST /api/members/consumer-migration`으로 대상 거래를 OWNER 명의에 일괄 연결할 수 있다.
- 실행 결과는 연결된 명의와 실제 처리 건수를 반환한다.
- OWNER 명의가 없으면 데이터 불일치로 간주해 실행을 중단한다.

### 명의 관리 UI

- 명의 관리 화면에 `기존 개인 지출 명의 연결` 영역을 추가했다.
- 실행 전에 현재 대상 건수를 표시한다.
- 대상이 없으면 실행 버튼을 비활성화한다.
- 대상이 있으면 연결될 OWNER 명의를 버튼에 표시하고 확인창을 거쳐 실행한다.
- 완료 후 대상 건수, 거래·통계 데이터와 화면을 다시 불러온다.

### 테스트 및 스모크 보강

- 마이그레이션이 대상 개인 지출만 OWNER에 연결하고 재실행 시 0건을 처리하는 단위 테스트를 추가했다.
- 브라우저 스모크 테스트에서 명의 관리 화면의 마이그레이션 상태 영역과 상태 API 응답을 검증한다.

## 4. 아직 부분 완료인 항목

### 예산·주간 통계

- 예산 화면은 아직 월별 예산 중심이다.
- 연간 예산 화면에서 연도 이동을 기본으로 하는 흐름과 주간별 통계는 아직 미완료다.

### 검색·필터

- 월 내 클라이언트 필터와 사용자 지정 기간 거래 조회는 구현되어 있다.
- 금액 범위, 자산, 다중 조건을 서버에서 조합하는 고급 검색·필터는 아직 미완료다.

### 기타 후속 항목

- 프론트엔드 화면 단위 컴포넌트 분리는 아직 미완료다.
- 백엔드 프로젝트 구조 개선 작업 진행은 아직 미완료다.
- 영수증 첨부 파일 정리 정책 고도화는 아직 미완료다.

## 6. 검증

- `docker run --rm -v "${PWD}\backend:/workspace" -w /workspace gradle:8.10.2-jdk21 gradle test --no-daemon` 통과
- `docker compose build frontend backend` 통과
- `docker compose up -d frontend backend`로 최신 이미지 재생성 완료
- `powershell -ExecutionPolicy Bypass -File .\scripts\browser-smoke.ps1` 통과
- 로컬 `npm run build`는 PATH에 `npm`이 없어 실행하지 못했으며, 동일 프론트 빌드는 Docker 이미지 빌드에서 통과했다.

## 7. 개발 메모

- 기존 데이터를 앱 시작 시 자동 변경하지 않고, 명의 관리 화면에서 대상 건수를 확인한 뒤 사용자가 명시적으로 실행하도록 했다.
- 마이그레이션은 소비 명의만 연결하며 거래 금액, 날짜, 카테고리, 자산 잔액에는 영향을 주지 않는다.
- OWNER가 바뀌는 정책이 추가되기 전까지는 현재 Household의 단일 OWNER를 기준으로 한다.
