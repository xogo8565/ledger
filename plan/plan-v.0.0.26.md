# 가계부 앱 개발 진행 기록 v.0.0.26

작성일: 2026-06-24

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.25.md`
- 이번 개발 범위: v.0.0.26 권장 항목 중 문자 파싱 기반 카테고리 추천
- 검증 방식: Docker Compose 컨테이너 빌드, Gradle 테스트 컨테이너 실행, 실제 문자 분석 API 확인, 브라우저 스모크 테스트

## 2. 이번 버전 개발 완료

### 문자 파싱 카테고리 추천

- 문자 분석 응답에 추천 카테고리 ID, 이름, 추천 근거를 추가했다.
- 같은 가맹점의 최근 거래 이력이 있으면 가장 자주 사용한 카테고리를 우선 추천한다.
- 거래 이력 검색은 최근 200건을 대상으로 한다.
- 가맹점명 비교 시 대소문자, 공백, 특수문자를 정규화한다.
- 같은 가맹점 이력이 없으면 가맹점 키워드 규칙으로 추천한다.
- 수입과 지출의 카테고리 추천 규칙을 분리했다.
- 비활성 카테고리와 거래 유형이 다른 과거 거래는 추천에서 제외한다.

### 기본 키워드 추천

- 식비: 스타벅스, 카페, 커피, 식당, 배달 등의 키워드
- 마트/편의점: CU, GS25, 세븐일레븐, 마트 등의 키워드
- 교통/차량: 택시, 주유, 하이패스, 버스, 지하철 등의 키워드
- 문화생활, 패션/미용, 생활용품, 주거/통신, 건강 관련 기본 키워드
- 급여, 이자, 부수입 관련 수입 키워드
- 현재 가계부에 해당 이름의 활성 카테고리가 있을 때만 추천한다.

### 문자 미리보기와 거래 입력

- 문자 분석 미리보기에 추천 분류와 추천 근거를 표시한다.
- `거래 입력으로 이동` 시 추천 카테고리를 입력 폼에 자동 선택한다.
- 추천이 없으면 기존처럼 카테고리를 비운 상태로 이동한다.
- 사용자는 거래 저장 전에 추천 카테고리를 자유롭게 변경할 수 있다.

### 회귀 테스트

- 스타벅스 문자를 식비로 추천하는 키워드 테스트를 추가했다.
- 같은 가맹점의 과거 거래가 있으면 과거 분류가 키워드 추천보다 우선되는 테스트를 추가했다.
- 기존 카드 승인, 체크카드 출금, 은행 입금, 승인취소 파싱 테스트를 유지했다.

### README 정리

- 구현된 문자 자동 입력 항목에 과거 이력·가맹점 키워드 기반 카테고리 추천을 반영했다.
- 다음 작업에서 문자 파싱 기반 카테고리 추천을 제거했다.
- 인증/권한 없이 사용하는 명의 관리 메뉴를 후속 항목으로 반영했다.

## 5. 아직 부분 완료인 항목

### 문자 자동 입력

- 카드/은행 문자 파싱과 과거 이력·키워드 카테고리 추천은 구현되어 있다.
- 사용자가 추천 키워드나 파싱 규칙을 직접 관리하는 기능은 아직 미완료다.
- 거래 추가 버튼에서 직접 입력/문자 자동 입력을 선택하는 진입 UX는 아직 미완료다.
- 더보기 화면의 문자 붙여넣기 UI를 제거하고 클립보드 기반 진입으로 전환하는 작업은 아직 미완료다.

### 통계

- 월별/연별 카테고리 및 소비 태그 통계는 구현되어 있다.
- 기간 필터 기준 요약 금액 재계산은 아직 미완료다.

### 영수증 첨부

- 신규/기존 거래의 최대 10장 일괄 첨부, 조회, 미리보기, 개별 삭제는 구현되어 있다.
- 할부 그룹 전체 수정 중 영수증 첨부 정책과 UI는 아직 미완료다.

### 자산 명의/공동 가계부

- 자산 명의 저장과 표시, Household/Member 도메인은 구현되어 있다.
- 명의별 자산 집계, 거래의 공동/개인 소비 구분, 더보기 명의 관리 메뉴는 아직 미완료다.
- 인증과 권한 인가는 현재 개인용 범위에서 제외한다.

## 8. 개발 메모

- 과거 거래 추천은 최근 거래 200건을 메모리에서 정규화·집계하는 MVP 방식이다.
- 추천 카테고리는 자동 저장되지 않고 거래 입력 폼의 초기 선택값으로만 사용한다.
- 키워드 규칙은 코드에 정의되어 있으며 사용자 정의 규칙 관리는 후속 과제다.
- PowerShell 콘솔에서는 UTF-8 응답이 깨져 보일 수 있으나 API의 추천 ID와 파싱 금액은 정상 확인했다.
- 브라우저 스모크 테스트 후 Docker Compose 스택은 실행 상태로 유지되어 있다.
