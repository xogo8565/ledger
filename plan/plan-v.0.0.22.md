## 2. 이번 버전 개발 완료

### 기간 거래 조회 API

- `GET /api/transactions/range?startDate=YYYY-MM-DD&endDate=YYYY-MM-DD` API를 추가했다.
- 시작일과 종료일 사이의 거래를 최신순으로 조회한다.
- 종료일이 시작일보다 빠르면 예외를 발생시키도록 했다.
- 기존 월별 거래 조회 API는 유지했다.

### 가계부 기간 필터 UI

- 가계부 필터 영역에 시작일/종료일 날짜 입력을 추가했다.
- 시작일과 종료일을 모두 입력하면 서버 기간 조회 API를 호출해 해당 기간 거래 목록을 표시한다.
- 기간 필터가 켜져 있을 때 선택 기간 배너를 표시한다.
- 기간 조회 결과에도 기존 검색어, 유형, 카테고리 필터를 함께 적용한다.
- 기간 중 하나만 입력한 경우에는 현재 로드된 거래 목록 안에서 로컬 날짜 필터를 적용한다.
- 초기화 버튼으로 검색어, 유형, 카테고리, 기간 필터를 함께 초기화하도록 했다.

### README 정리

- 구현된 MVP 목록에 사용자 지정 기간 거래 조회/필터를 추가했다.
- 다음 작업 목록에서 사용자 지정 기간 필터 항목을 제거했다.
- 다음 우선순위를 필터 결과 기반 CSV 내보내기로 갱신했다.

### 통계

- 월별/연별 통계와 카테고리 선택 상세 이동은 구현되어 있다.
- 소비 태그별 통계는 아직 미완료다.

### 문자 자동 입력

- 카드 승인, 체크카드 출금, 은행 입금, 취소/환불 기본 규칙은 구현되어 있다.

- 문자 파싱 기반 카테고리 추천은 아직 미완료다.
- 문자 자동 입력 UX 변경 : 초기 화면에서 + 클릭 시, 햄버거 창 표기 -> 직접 입력 / 문자 자동 입력 중 선택하여 진행

- 신규 거래 첨부, 기존 거래 수정 중 첨부, 조회, 미리보기, 삭제 UI는 구현되어 있다.
- 다중 첨부 일괄 업로드와 할부 그룹 수정 중 첨부 흐름은 아직 미완료다.

## 8. 개발 메모

- 시작일/종료일이 모두 입력된 경우 서버 기간 조회 결과를 사용한다.
- 월 요약 수치는 현재 선택 월 기준이며, 기간 필터 기준 요약 재계산은 다음 개선 과제로 남겼다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드와 Gradle 테스트는 정상 통과했다.
