작성일: 2026-06-23

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.9.md`
- 이번 개발 범위: v.0.0.10 권장 항목 중 실패한 카드 결제 재시도/재예약 UI
- 검증 방식: Docker Compose 컨테이너 빌드

## 2. 이번 버전 개발 완료

### 실패 카드 결제 재예약 API

- `POST /api/cards/payment-schedules/{scheduleId}/reschedule` API를 추가했다.
- `FAILED` 상태인 카드 결제 예약만 재예약할 수 있다.
- 재예약 시 상태를 `SCHEDULED`로 되돌리고 완료일과 실패 사유를 비운다.
- 실패가 아닌 예약에 재예약을 요청하면 오류를 반환한다.

### 실패 카드 결제 재시도 API

- `POST /api/cards/payment-schedules/{scheduleId}/retry` API를 추가했다.
- `FAILED` 상태인 카드 결제 예약을 먼저 재예약한 뒤 즉시 실행한다.
- 기존 결제 실행 로직을 재사용하므로 실행 성공 시 `COMPLETED`, 재실패 시 다시 `FAILED` 상태로 남는다.

### 카드 결제 UI

- 카드 결제 처리 내역에서 실패한 항목에 `재시도` 버튼을 추가했다.
- 카드 결제 처리 내역에서 실패한 항목에 `재예약` 버튼을 추가했다.
- `재시도`는 즉시 실행 API를 호출하고 카드 결제 데이터와 전체 데이터를 다시 불러온다.
- `재예약`은 실패 건을 예약 목록으로 되돌리고 카드 결제 데이터를 다시 불러온다.

### README 정리

- 구현된 MVP 목록의 카드 결제 항목에 실패 결제 재시도/재예약을 반영했다.

## 5. 아직 부분 완료인 항목

### 카드 관리

- 카드 결제 예약과 실행은 구현되어 있다.
- 카드 자산 생성/수정 시 카드 프로필 입력과 저장 API 연결은 구현되어 있다.
- 자동 결제 중복 실행 방지는 `PROCESSING` 상태와 낙관적 잠금으로 보강했다.
- 실패 상태 저장과 UI 표시는 구현되어 있다.
- 실패한 카드 결제 재시도와 재예약 UI는 구현되어 있다.
- 결제 예정 금액은 다음 결제 예정일에 대응하는 청구 기간 기준으로 산정한다.
- 말일 처리는 구현되어 있다.
- 공휴일/주말 결제일 보정 정책은 아직 미완료다.

### 할부 관리

- 할부 생성, 조회, 그룹 단위 수정/삭제는 구현되어 있다.
- 그룹 수정 시 기존 회차 수 변경은 아직 미지원이다.
- 카드 결제 예정 금액과 할부 청구 월 연결 검증이 필요하다.

### 예산

- 월별 전체/카테고리 예산 저장은 구현되어 있다.
- 카테고리별 예산 사용률과 초과 표시는 구현되어 있다.
- 전월 예산 복사는 구현되어 있다.

### 통계

- 월별 및 카테고리별 지출 통계는 구현되어 있다.
- 연별 통계, 기간 필터, 카테고리 선택 후 거래 목록 이동은 미완료다.

### 영수증 첨부

- 신규 거래 첨부와 업로드 API는 구현되어 있다.
- 첨부 이미지 조회, 미리보기, 삭제, 다중 첨부 UI는 미완료다.

### 문자 자동 입력

- 원문 붙여넣기와 후보 추출은 구현되어 있다.
- 파싱 결과를 거래 입력 폼으로 확정 저장하는 흐름은 미완료다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

### 공동 가계부

- Household/Member 도메인은 있다.
- 인증, 초대, 권한, 사용자 전환 UI는 미완료다.
- 인증, 권한, 사용자 전환은 불필요하더라도 자산 명의나 관련 소비 태그로 활용할 구조가 필요하다.

## 8. 개발 메모

- 재시도는 기존 실패 예약의 날짜와 금액을 그대로 사용한다.
- 실패 예약의 금액이나 날짜를 바꿔 재예약하는 흐름은 아직 없다. 필요하면 실패 건 수정 UI와 함께 확장하는 편이 좋다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드는 정상 통과했다.
