# 편한가계부 초기 데이터 주입 개발 기록 v.0.3.0

작성일: 2026-06-26

## 1. 개발 목표

- `초기거래/자산설정_초기거래.xlsx` 파일을 기준으로 DB 최초 실행 시 자산과 잔액을 주입한다.
- `초기거래/6월_가계부.xlsx` 파일을 기준으로 DB 최초 실행 시 거래 내역을 주입한다.
- 이미 데이터가 존재하는 DB에서는 초기 데이터가 중복 주입되지 않도록 한다.

## 2. 구현 범위

- 엑셀 원본을 `backend/src/main/resources/initial-data/`에 포함해 Docker/배포 환경에서도 동일하게 읽을 수 있도록 했다.
- 런타임 외부 라이브러리 추가 없이 JDK ZIP/XML 파서로 `.xlsx` 파일을 읽는 `InitialDataWorkbookReader`를 추가했다.
- 기존 하드코딩 데모 데이터 초기화 로직을 엑셀 기반 초기화로 교체했다.
- 빈 DB 조건은 `households`, `assets`, `categories`, `transactions`가 모두 비어 있는 경우로 제한했다.
- 자산설정 파일의 잔액은 기준 잔액으로 유지하고, 거래 내역 저장 시 자산 잔액을 추가 증감하지 않는다.

## 3. 컬럼 매핑

| 엑셀 컬럼 | 의미 | DB 반영 |
| --- | --- | --- |
| A | 날짜 | 거래일, Excel serial date 변환 지원 |
| B | 자산 | 자산명 매핑, 없으면 0원 자산 자동 생성 |
| C | 분류 | 카테고리명 우선값 |
| D | 소분류 | 분류가 비어 있을 때 보조 카테고리명 |
| E | 내용 | 거래 제목 |
| F | 금액(원) | 거래 금액/자산 잔액 우선값 |
| G | 수입/지출 | 거래 유형 |
| H | 메모 | 거래 메모 |
| I | 금액 | F가 비어 있을 때 보조 금액 |
| J | 통화 | 현재는 원화 기준으로 참고만 함 |

## 4. 검증

- `InitialDataWorkbookReaderTest` 추가
  - 자산설정 파일 9행 파싱 확인
  - 6월 가계부 파일 124행 파싱 확인
  - 주요 금액/수입지출 컬럼 파싱 확인
- 실행 결과:

```powershell
docker run --rm -v "${PWD}\backend:/workspace" -w /workspace gradle:8.10.2-jdk21 gradle test --no-daemon
```

결과: 성공

## 5. 후속 권장 사항

1. 최초 구동 데이터가 실제 운영 시작 잔액인지, 거래 내역 반영 후의 현재 잔액인지 정책을 확정한다.
2. 초기 데이터 파일 교체가 자주 필요하면 `resources` 번들 방식 대신 외부 경로 설정값을 추가한다.
3. 운영 DB에는 최초 1회만 적용되므로, 이미 데이터가 있는 DB에 재주입이 필요하면 별도 import 기능으로 분리한다.
