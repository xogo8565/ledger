# 편한가계부 신규 개발 계획 v.0.1.0

작성일: 2026-06-26

## 1. 기준

- 이전 개발 라인: `plan-v.0.0.*`
- v0.0 종결 기록: `plan/plan-v.0.0.57.md`
- 신규 개발 라인: `plan-v.0.1.*`
- 신규 개발 주제: 영수증 업로드 시 OCR 텍스트 변환 및 자동 입력 기능

## 2. v0.1 목표

영수증 이미지를 업로드하면 OCR로 텍스트를 추출하고, 추출된 텍스트를 거래 입력 폼에 자동 반영할 수 있게 한다.

기존 클립보드 문자 자동 분석 기능의 사용 경험을 영수증 이미지 입력에도 확장한다.

## 3. 핵심 사용자 흐름

### 초기 화면 진입

- 앱 초기 화면에서 영수증 업로드 진입점을 제공한다.
- 사용자는 거래를 직접 입력하기 전에 영수증 사진부터 올릴 수 있다.
- 업로드 후 OCR 분석 결과를 확인하고 거래 입력 폼으로 이어진다.

### 햄버거 바 진입

- 클릭 시 열리는 햄버거 바 또는 더보기 메뉴에 `영수증 업로드` 항목을 추가한다.
- 메뉴 항목을 누르면 영수증 업로드 화면 또는 업로드 패널로 이동한다.
- 기존 거래 입력 플로우와 분리된 독립 진입점으로 제공한다.

### OCR 자동 입력

- 사용자가 영수증 이미지를 업로드한다.
- 백엔드는 이미지를 저장하고 OCR 처리를 수행한다.
- OCR 결과에서 다음 후보 값을 추출한다.
  - 거래일
  - 금액
  - 가맹점/제목
  - 메모 원문 또는 주요 텍스트
  - 카테고리 추천 후보
- 프론트엔드는 추출 결과를 검토 가능한 상태로 표시한다.
- 사용자가 확인하면 거래 입력 폼에 자동 채운다.

## 4. 백엔드 개발 항목

### OCR 처리 구조

- `ReceiptOcrService` 신규 도입
- OCR 엔진 연동 어댑터 분리
  - 초기 구현은 로컬/외부 OCR 선택 가능성을 열어둔다.
  - OCR 엔진이 미설정된 환경에서는 명확한 오류 또는 mock/stub 처리 정책을 둔다.
- OCR 원문 텍스트와 파싱 결과를 분리한다.

### API 설계 초안

- `POST /api/receipts/ocr`
  - multipart image 업로드
  - OCR 텍스트 추출 및 거래 후보 반환
- `POST /api/receipts/ocr/parse`
  - OCR 원문 텍스트를 거래 후보로 재파싱
  - 재분석/수동 보정 흐름 지원

### 응답 DTO 초안

- 원본 파일명
- OCR 원문 텍스트
- 거래일 후보
- 금액 후보
- 제목/가맹점 후보
- 메모 후보
- 카테고리 추천 후보
- 신뢰도 또는 경고 목록

## 5. 프론트엔드 개발 항목

### 초기 화면 UX

- 초기 화면 또는 주요 홈 영역에 `영수증 업로드` CTA를 추가한다.
- 사용자가 파일을 선택하면 OCR 분석 진행 상태를 표시한다.
- 분석 결과를 확인한 뒤 거래 입력 화면으로 이어진다.

### 햄버거 바 UX

- 햄버거 바 또는 더보기 메뉴에 `영수증 업로드` 항목을 추가한다.
- 메뉴 항목은 OCR 업로드 패널을 열거나 OCR 전용 화면으로 이동한다.
- 기존 하단 탭/더보기 흐름과 충돌하지 않도록 화면 상태를 분리한다.

### 자동 입력 UX

- OCR 결과를 즉시 저장하지 않고, 거래 입력 폼에 초안으로 채운다.
- 사용자는 금액·날짜·카테고리·자산을 확인하고 저장한다.
- OCR 원문 보기/접기 UI를 제공한다.
- 분석 실패 시 수동 입력으로 자연스럽게 전환한다.

## 6. 데이터 및 저장 정책

- OCR 업로드 파일을 기존 영수증 첨부 저장소와 공유할지, 임시 저장소로 둘지 결정해야 한다.
- 거래 저장 전 업로드된 OCR 파일은 임시 파일로 취급하고 정리 정책이 필요하다.
- 거래 저장 후에는 기존 영수증 첨부로 연결할 수 있는 흐름을 검토한다.

## 7. 보안 및 운영 고려사항

- 영수증 이미지는 개인정보와 결제 정보를 포함할 수 있으므로 로그에 원문 이미지를 남기지 않는다.
- OCR 원문 텍스트도 민감정보일 수 있으므로 저장 여부를 별도 결정한다.
- 외부 OCR API를 사용할 경우 API 키, 비용, 장애 fallback, 요청 제한 정책이 필요하다.

4. OCR 결과를 거래 입력 폼 초안으로 연결

3. 영수증 임시 파일 정리 정책 구현

## 9. 완료 기준

- 초기 화면에서 영수증 업로드를 시작할 수 있다.
- 햄버거 바 또는 더보기 메뉴에서 영수증 업로드를 시작할 수 있다.
- 이미지 업로드 후 OCR 텍스트 추출 결과를 볼 수 있다.
- OCR 결과가 거래 입력 폼에 자동 채워진다.
- 사용자는 자동 입력된 거래를 수정 후 저장할 수 있다.
- OCR 실패 시에도 수동 입력으로 전환할 수 있다.

- Playwright 브라우저 스모크에 영수증 업로드 진입점 확인 추가
- 실제 OCR 엔진 연동 후 샘플 영수증 이미지 기반 회귀 테스트
