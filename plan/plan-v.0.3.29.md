# plan-v.0.3.29

작성일: 2026-07-01

## 작업 범위

- `P3. OCR 후보 선택 이력 기반 추천 개선` 항목을 진행했다.
- OCR 후보 선택 UI에서 사용자가 고른 후보를 브라우저 로컬 이력으로 집계하고, 다음 OCR 결과의 후보 표시 순서에 반영했다.

## 변경 내용

- `frontend/src/screens/TransactionScreens.jsx`
  - OCR 후보 선택 이력 저장소를 `localStorage` 기반으로 추가했다.
  - 날짜, 품명/가맹점, 금액 후보 선택 횟수를 저장한다.
  - 금액 후보는 현재 품명/가맹점별 선택 이력을 우선 반영하고, 전체 금액 선택 빈도를 보조로 사용한다.
  - 후보 칩을 직접 선택할 때와 `거래 입력에 사용`을 누를 때 현재 후보를 이력에 기록한다.
  - 이전 선택 이력이 적용된 경우 OCR 후보 카드에 안내 문구를 표시한다.
- `README.md`
  - OCR 후보 선택 이력 기반 정렬을 기능 목록에 추가했다.
- `plan/new-development-items.md`
  - 해당 항목을 부분 완료로 최신화했다.

## 저장 범위

- 서버 DB에는 저장하지 않는다.
- 브라우저 `localStorage`에 후보 문자열과 선택 횟수만 저장한다.
- OCR 원문 전체나 영수증 이미지는 저장하지 않는다.

## fallback

- 저장된 이력이 없거나 `localStorage`를 사용할 수 없으면 기존 OCR 후보 순서를 그대로 사용한다.
- 이력 정렬은 후보 목록을 바꾸지 않고 순서만 조정한다.

## 남은 작업

- 서버 기반 후보 선택 이력 저장이 필요한지 결정한다.
- 품명/가맹점별 자주 선택한 카테고리를 서버 추천에 반영할지 검토한다.
- 이력 초기화 UI가 필요한지 검토한다.
