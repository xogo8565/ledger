# 편한가계부 후속 개발 진행 기록 v.0.0.42

작성일: 2026-06-25

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.41.md`
- 이번 개발 범위: v.0.0.42 권장 항목 1순위인 가계부 화면과 거래 필터 컴포넌트 분리
- 검증 방식: Gradle 전체 테스트, Docker Compose 이미지 빌드, Playwright 브라우저 스모크 테스트

## 2. 이번 버전 개발 완료

### 가계부 화면 모듈 분리

- `frontend/src/screens/LedgerScreen.jsx`를 추가했다.
- `main.jsx`에 있던 다음 가계부 화면 구성을 전용 모듈로 이동했다.
  - 가계부 탭과 월 탐색
  - 월 합계
  - 일일 거래 목록
  - 달력 거래 요약
  - 월별 요약
  - 거래·자산·예산 요약
  - 메모 거래 목록
  - 거래 행과 할부 진입 UI
- App은 거래 데이터, 필터 상태, 화면 전환 callback을 전달하고 가계부 모듈이 표시와 화면 내 파생 계산을 담당한다.

### 거래 필터 컴포넌트 분리

- 검색어, 유형, 카테고리, 소비 구분, 명의, 자산, 기간, 금액 범위 필터를 가계부 모듈로 이동했다.
- 필터 초기값과 활성 여부 판별도 `LedgerScreen.jsx`에서 export한다.
- 서버 검색 결과에 대한 화면 내 최종 필터와 합계 계산을 가계부 모듈 내부로 모았다.
- 필터 결과 CSV callback은 기존 App의 다운로드 흐름을 그대로 사용한다.

### 가계부 공통 표시 컴포넌트 정리

- `MonthNav`, `YearNav`, `Metric`, `KeyValue`를 가계부 화면 모듈에서 export하도록 정리했다.
- 통계, 예산, 자산, 상세 화면은 이 공통 표시 컴포넌트를 재사용한다.
- 거래 상세 화면에서 사용하는 이체 라벨과 거래 유형 아이콘 helper도 가계부 모듈에서 공유한다.

### 포맷·CSV 유틸 분리

- `frontend/src/utils/format.js`를 추가했다.
- 금액, 숫자, 날짜, 월 라벨, 월 이동과 거래 색상 helper를 이동했다.
- `frontend/src/utils/csv.js`를 추가했다.
- CSV 셀 escaping, 브라우저 다운로드, 필터 파일명 생성 helper를 이동했다.
- 공통 진행률 UI인 `ProgressBar`를 `components/ui.jsx`로 이동했다.

### 반복 거래 저장 목록 안정화

- 브라우저 회귀 검증 중 반복 거래 저장 직후 재조회 결과에 의존하던 목록 갱신 타이밍 문제를 확인했다.
- 생성·수정 API 응답을 즉시 `recurringRules` 상태에 반영하도록 변경했다.
- 저장 직후에도 생성 행이 안정적으로 표시되며 이후 목록 재진입 시 서버 데이터를 다시 조회한다.
- 브라우저 스모크 테스트에 저장 응답 제목 검증을 추가했다.

### 파일 규모 변화

- `main.jsx`는 약 3,003줄에서 약 2,436줄로 감소했다.
- 가계부 표시와 필터 로직 약 400줄을 `LedgerScreen.jsx`로 이동했다.
- 화면·공통 UI·유틸 경계가 분리되어 다음 통계·자산 화면 분리의 기반을 마련했다.

## 4. 아직 부분 완료인 항목

### 프론트엔드 구조

- 통계·예산 화면은 아직 `main.jsx`에 남아 있다.
- 자산, 카드 결제, 거래 입력·상세 화면도 아직 `main.jsx`에 남아 있다.
- API 요청과 상태 로직을 custom hook 또는 API 모듈로 분리하는 작업은 아직 미완료다.

### 백엔드 구조

- `LedgerService`가 거래, 검색, 예산, 통계, 명의와 자산 관리 책임을 함께 담당한다.
- 도메인별 서비스 분리와 검색 조건 객체 분리가 필요하다.

### 기타 후속 항목

- 검색 결과 페이지네이션과 정렬 선택은 아직 미완료다.
- 영수증 첨부 파일 정리 정책 고도화는 아직 미완료다.

## 7. 개발 메모

- 화면 모듈이 서버 요청을 직접 소유하지 않고 App에서 받은 데이터와 callback을 사용하도록 해 상태 흐름을 유지했다.
- 가계부 화면 내부에서만 필요한 필터 후처리와 요약 계산은 화면 모듈에 배치해 응집도를 높였다.
- 반복 거래 목록 문제는 모듈 분리 자체의 회귀가 아니라 저장 후 즉시 GET에 의존하던 기존 갱신 방식이 원인이었으며, API 응답 기반 상태 반영으로 보완했다.
