# 편한가계부 신규 개발 진행 기록 v.0.1.6

작성일: 2026-06-26

## 1. 기준

- 이전 기록: `plan/plan-v.0.1.5.md`
- 이번 개발 범위: OCR 실패/낮은 신뢰도 UX 보강
- 배경: OCR 분석이 실패하거나 결과가 빈약할 때 사용자가 다음 행동을 판단하기 어렵다. 실패 메시지, 품질 경고, 수동 입력 전환, 촬영/재분석 가이드를 보강한다.

## 2. 이번 버전 개발 완료

### 백엔드 OCR 품질 경고

- `ReceiptOcrService`에서 OCR 결과와 파싱 후보를 기반으로 사용자용 경고를 추가했다.
- 추가 경고 기준:
  - OCR 텍스트가 비어 있음
  - OCR 텍스트가 지나치게 짧음
  - 금액 후보를 찾지 못함
  - 가맹점/품명 후보가 불확실함
- Tesseract stderr 중 `Estimating resolution...` 같은 정보성 메시지는 사용자 경고에서 제외하도록 분리했다.

### 프론트엔드 OCR 실패 대응

- OCR 분석 실패 메시지 영역에 `직접 입력으로 전환` 버튼을 추가했다.
- 실패 시에도 빈 지출 입력 초안으로 이동할 수 있어 업로드 흐름에서 막히지 않는다.
- OCR 결과 화면에 촬영/재분석 가이드를 추가했다.
- 사용자는 영수증 전체가 보이게 다시 촬영하거나 OCR 원문에서 품명·합계·금액을 수정한 뒤 재분석할 수 있다.

### 테스트 추가

- `ReceiptOcrServiceTest`를 추가했다.
- 검증 항목:
  - 정보성 Tesseract 메시지를 경고에서 제외한다.
  - OCR 텍스트가 빈 경우 사용자 행동 경고를 반환한다.
  - 짧은 텍스트, 금액 없음, 가맹점 불확실 상태를 경고한다.

## 4. 아직 부분 완료인 항목

### 검증 상태

- Docker Desktop이 수동 일시정지 상태라 Docker 기반 검증을 실행하지 못했다.
- 로컬에는 Gradle wrapper가 없고 `npm` 명령도 PATH에 없어 대체 빌드도 제한됐다.
- `git diff --check`는 통과했다.
- Docker Desktop 재개 후 아래 검증이 필요하다.

```powershell
docker run --rm -v "${PWD}\backend:/workspace" -w /workspace gradle:8.10.2-jdk21 gradle test --no-daemon
docker compose build backend
docker compose build frontend
```

### UX 고도화

- OCR 결과 후보 다중 선택 UI는 아직 없다.
- 품목/금액/가맹점 후보를 사용자가 선택하는 인터랙션은 다음 단계로 남긴다.

### 파일 정책

- 영수증 임시 파일 정리 정책 검토는 아직 남아 있다.

## 7. 개발 메모

- 이번 변경은 OCR 정확도 자체를 높이는 것이 아니라 실패·저신뢰도 상황에서 사용자가 멈추지 않도록 하는 흐름 개선이다.
- 품질 경고는 서버에서 계산해 응답에 포함하므로, 프론트는 동일한 경고 목록을 표시하기만 하면 된다.
