# 가계부 앱 개발 진행 기록 v.0.0.4

작성일: 2026-06-23

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.3.md`
- 이번 개발 범위: v.0.0.4 권장 항목 중 기존 카드 프로필 수정 UI 추가
- 검증 방식: Docker Compose 컨테이너 빌드

## 2. 이번 버전 개발 완료

### 기존 카드 프로필 수정 UI

- 자산 목록에서 기존 카드 자산을 눌러 수정할 때 카드 결제 설정 영역이 함께 표시되도록 변경했다.
- 기존 카드 프로필 값을 자산 수정 폼에 채운다.
  - 결제 계좌
  - 결제금액 확정일
  - 결제일
  - 자동 결제 여부
- 카드 자산 수정 저장 시 일반 자산 수정 API가 아니라 `PUT /api/assets/{id}/card`를 호출하도록 연결했다.
- 카드 결제 설정 영역은 신규 카드 생성과 기존 카드 수정에서 모두 같은 입력 UI를 사용한다.

### 카드 프로필 수정 API

- `PUT /api/assets/{id}/card` API를 추가했다.
- 카드 자산의 이름, 잔액, 그룹, 메모와 카드 프로필 정보를 한 번에 갱신한다.
- 기존 카드 프로필이 없으면 수정 저장 시 새로 생성하도록 했다.
- 신규 카드 생성 직후 응답에도 카드 프로필 정보가 안정적으로 포함되도록 카드 자산의 역방향 참조를 세팅했다.

### README 정리

- 구현된 MVP 목록의 카드 자산 항목을 생성/수정 기준으로 최신화했다.

## 5. 아직 부분 완료인 항목

### 카드 관리

- 카드 결제 예약과 실행은 구현되어 있다.
- 카드 자산 생성/수정 시 카드 프로필 입력과 저장 API 연결은 구현되어 있다.
- 실제 청구 기간, 말일 처리, 휴일 처리, 중복 자동 결제 방지 정책은 보강이 필요하다.
- 카드 결제 예정 금액을 사용자가 이해하기 쉽게 보여주는 UI 설명도 더 필요하다.

### 할부 관리

- 할부 생성과 조회는 구현되어 있다.
- 할부 그룹 전체 수정/삭제는 아직 미완료다.
- 카드 결제 예정 금액과 할부 청구 월 연결 검증이 필요하다.

### 예산

- 월별 전체/카테고리 예산 저장은 구현되어 있다.
- 카테고리별 예산 사용률, 초과 표시, 전월 예산 복사는 미완료다.

### 통계

- 월별 및 카테고리별 지출 통계는 구현되어 있다.
- 연별 통계, 기간 필터, 카테고리 선택 후 거래 목록 이동은 미완료다.

### 영수증 첨부

- 신규 거래 첨부와 업로드 API는 구현되어 있다.
- 첨부 이미지 조회, 미리보기, 삭제, 다중 첨부 UI는 미완료다.

### 문자 자동 입력

- 원문 붙여넣기와 후보 추출은 구현되어 있다.
- 파싱 결과를 거래 입력 폼으로 확정 저장하는 흐름은 미완료다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

### 공동 가계부

- Household/Member 도메인은 있다.
- 인증, 초대, 권한, 사용자 전환 UI는 미완료다.

## 8. 개발 메모

- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드는 정상 통과했다.
- 프론트엔드는 여전히 `frontend/src/main.jsx` 단일 파일 비중이 크다. 다음 기능 확장 전에 화면 단위 컴포넌트 분리를 고려하는 것이 좋다.
