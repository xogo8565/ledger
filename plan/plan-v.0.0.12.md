# 가계부 앱 개발 진행 기록 v.0.0.12

작성일: 2026-06-23

## 1. 기준

- 이전 기록: `plan/plan-v.0.0.11.md`
- 이번 개발 범위: v.0.0.12 권장 항목 중 할부 회차 수 변경 지원
- 검증 방식: Docker Compose 컨테이너 빌드

## 2. 이번 버전 개발 완료

### 할부 회차 수 변경 지원

- `PUT /api/transactions/installments/{installmentGroupId}`에서 기존 회차 수와 다른 `installmentMonths`를 받을 수 있도록 변경했다.
- 할부 그룹 수정 시 기존 회차의 자산 반영을 모두 되돌린다.
- 기존 할부 회차를 모두 삭제한 뒤 같은 `installmentGroupId`로 새 회차를 다시 생성한다.
- 새 회차 수 기준으로 전체 금액을 다시 분배한다.
- 마지막 회차에 나머지 금액을 반영해 합계 오차를 줄인다.
- 시작일 기준으로 각 회차 거래일을 월 단위로 다시 계산한다.

### 기존 UI 재사용

- 기존 할부 전체 수정 화면의 할부 개월 선택값을 그대로 API에 전달한다.
- 사용자는 할부 전체 수정 화면에서 금액, 시작일, 카테고리, 자산, 내용과 함께 회차 수를 변경할 수 있다.

### README 정리

- 구현된 MVP 목록의 할부 항목에 회차 수 변경 지원을 반영했다.

## 5. 아직 부분 완료인 항목

### 카드 관리

- 법정 공휴일/대체공휴일 데이터 기반 결제일 보정은 아직 미완료다.
- 카드 결제 예정 금액과 할부 청구 월 연결 검증이 필요하다.

### 할부 관리

- 할부 생성, 조회, 그룹 단위 수정/삭제, 회차 수 변경은 구현되어 있다.
- 회차 수 변경은 기존 회차를 모두 삭제 후 재생성하는 정책이다.
- 이미 확정/결제된 회차 보호 정책은 아직 없다.

### 통계

- 연별 통계, 기간 필터, 카테고리 선택 후 거래 목록 이동은 미완료다.

### 영수증 첨부

- 첨부 이미지 조회, 미리보기, 삭제, 다중 첨부 UI는 미완료다.

### 문자 자동 입력

- 파싱 결과를 거래 입력 폼으로 확정 저장하는 흐름은 미완료다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

## 8. 개발 메모

- 할부 회차 수 변경은 가장 단순하고 일관적인 방식인 전체 재생성 정책으로 구현했다.
- 향후 결제 확정/청구 반영 상태가 생기면 삭제/재생성 전에 보호 규칙이 필요하다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드는 정상 통과했다.
