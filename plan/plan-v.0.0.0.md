# 가계부 앱 기획서

## 1. 프로젝트 개요

### 앱 목적

개인 사용을 목적으로 하는 모바일 가계부 앱을 개발한다. 참고 앱은 "편한가계부"이며, 핵심 사용 흐름은 빠른 수입/지출 입력, 자산별 잔액 관리, 카드 결제 예정 금액 관리, 월별 예산 관리다.

### 핵심 가치

- 수입, 지출, 이체를 빠르게 기록한다.
- 현금, 은행, 카드 등 자산별 잔액과 부채를 한 화면에서 확인한다.
- 카드 지출은 결제 예정 금액과 미결제 금액을 구분해 관리한다.
- 월별 예산은 지출 카테고리와 연결해 예산 대비 사용액을 확인한다.
- 월별, 카테고리별 소비 흐름을 통계로 파악한다.

### 디자인 방향

- 기본 배경은 흰색과 따뜻한 아이보리 톤을 사용한다.
- 주요 포인트 색상은 연두색 계열로 사용해 가볍고 편안한 인상을 준다.
- 보조 색상은 우드 브라운 계열로 사용해 자산/가계부의 생활감과 안정감을 만든다.
- 경고, 초과 예산, 삭제 등 강한 상태 표현을 제외하고 원색 사용을 최소화한다.
- 화면은 카드형 요소를 과하게 쓰지 않고, 여백과 구분선 중심으로 단정하게 구성한다.

### 우선 개발 범위

1. 거래 입력: 수입, 지출, 이체
2. 거래 목록: 일일, 달력, 월별, 요약
3. 자산 관리: 현금, 은행, 카드
4. 카드 관리: 결제일, 결제금액 확정일, 결제 예정 금액
5. 카테고리 관리: 수입/지출 분류
6. 월별 예산: 지출 카테고리별 예산 설정
7. 통계: 월별 수입/지출, 카테고리별 지출 비중

## 2. 주요 사용자 시나리오

### 지출 기록

1. 사용자는 메인 가계부 화면에서 추가 버튼을 누른다.
2. 입력 화면에서 `지출` 탭을 선택한다.
3. 날짜, 금액, 분류, 자산, 내용을 입력한다.
4. 저장하면 선택한 자산의 잔액 또는 카드 미결제 금액에 반영된다.
5. 지출 카테고리와 연결된 월 예산 사용액도 함께 증가한다.

### 수입 기록

1. 사용자는 입력 화면에서 `수입` 탭을 선택한다.
2. 날짜, 금액, 분류, 입금 자산, 내용을 입력한다.
3. 저장하면 해당 자산 잔액이 증가한다.
4. 월별 수입 통계에 반영된다.

### 자산 간 이체

1. 사용자는 입력 화면에서 `이체` 탭을 선택한다.
2. 금액, 출금 자산, 입금 자산, 수수료, 내용을 입력한다.
3. 저장하면 출금 자산은 감소하고 입금 자산은 증가한다.
4. 수수료가 있으면 별도 지출로 처리하거나 이체 비용 항목으로 저장한다.

### 카드 지출 관리

1. 사용자는 카드 자산을 생성할 때 결제일과 결제금액 확정일을 설정한다.
2. 카드로 지출을 입력하면 해당 카드의 미결제 금액에 포함된다.
3. 결제금액 확정일 기준으로 다음 결제일에 청구될 결제 예정 금액을 계산한다.
4. 카드 결제일이 되면 연결된 결제 계좌에서 결제 예정 금액이 출금되고 카드 미결제 금액이 감소한다.

### 월 예산 관리

1. 사용자는 통계 화면의 `예산` 탭에서 월별 예산을 설정한다.
2. 전체 월 예산과 카테고리별 예산을 입력한다.
3. 지출을 기록하면 해당 카테고리 예산 사용액에 반영된다.
4. 예산 화면에서 남은 예산, 사용률, 카테고리별 사용액을 확인한다.

## 4. 기능 요구사항

### 거래 관리

- 거래는 수입, 지출, 이체 중 하나의 유형을 가진다.
- 거래는 날짜, 금액, 자산, 카테고리, 내용 값을 가진다.
- 지출 거래는 지출 카테고리와 연결되어야 한다.
- 수입 거래는 수입 카테고리와 연결되어야 한다.
- 이체 거래는 출금 자산과 입금 자산을 모두 가져야 한다.
- 거래는 수정, 삭제할 수 있어야 한다.
- 거래 수정/삭제 시 자산 잔액, 카드 미결제 금액, 예산 사용액이 다시 계산되어야 한다.

### 자산 관리

- 자산 유형은 현금, 은행, 카드, 기타, 부채로 구분한다.
- 현금/은행/기타 자산은 거래 발생 시 잔액이 직접 증가 또는 감소한다.
- 카드 자산은 지출 시 실제 은행 잔액을 즉시 줄이지 않고 카드 미결제 금액을 증가시킨다.
- 부채 자산은 사용자가 별도 입력하거나 이체/상환 거래로 관리한다.
- 자산 목록에서는 자산, 부채, 순자산 합계를 표시한다.

### 카드 결제 규칙

카드에는 `결제금액 확정일`과 `결제일`이 있다.

예시:

- 결제금액 확정일: 매월 1일
- 결제일: 매월 1일
- 5월 1일 ~ 5월 31일 사용분은 6월 1일 결제 예정 금액
- 6월 1일 ~ 6월 30일 사용분은 7월 1일 결제 예정 금액

규칙:

- 카드 지출은 사용일 기준으로 기록한다.
- 카드 결제 예정 금액은 결제금액 확정일 기준 청구 기간에 포함된 카드 지출 합계다.
- 결제일이 도래하면 연결 결제 계좌에서 결제 예정 금액만큼 출금 처리한다.
- 자동 결제가 켜져 있으면 결제일에 자동으로 이체/출금 거래를 생성한다.
- 자동 결제가 꺼져 있으면 사용자가 수동으로 카드 대금 결제 거래를 등록한다.
- 카드 할부 거래는 각 월별 청구 금액으로 나누어 결제 예정 금액에 반영한다.

### 카테고리 관리

- 수입 카테고리와 지출 카테고리를 분리한다.
- 카테고리는 이름, 아이콘, 색상, 정렬 순서를 가진다.
- 지출 카테고리는 월 예산과 연결될 수 있다.
- 카테고리 삭제 시 기존 거래 보존을 위해 `미분류`로 이동하거나 삭제를 제한한다.

### 예산 관리

- 예산은 월 단위로 생성한다.
- 전체 월 예산과 카테고리별 월 예산을 설정할 수 있다.
- 카테고리별 예산 합계가 전체 예산과 다를 수 있도록 허용한다.
- 지출 거래가 저장되면 해당 월과 카테고리의 사용액에 반영된다.
- 예산 화면은 남은 예산, 사용액, 사용률을 표시한다.
- 예산 초과 카테고리는 시각적으로 구분한다.
- 전월 예산을 복사해 다음 달 예산으로 생성할 수 있다.

### 통계 관리

- 월별/연별 통계를 제공한다.
- 수입과 지출을 분리해서 볼 수 있다.
- 카테고리별 비중과 금액을 표시한다.
- 특정 카테고리를 선택하면 해당 거래 목록으로 이동한다.
- 자산별 지출 통계는 후순위 기능으로 둔다.

### 영수증 사진 첨부

- 사용자는 수입/지출 거래에 영수증 사진을 첨부할 수 있다.
- 사진은 카메라 촬영 또는 갤러리 선택으로 등록한다.
- 거래 상세 화면에서 첨부 이미지를 미리보기로 확인할 수 있다.
- 한 거래에 여러 장의 이미지를 첨부할 수 있도록 확장 가능하게 설계한다.
- MVP에서는 OCR 자동 인식은 제외하고, 사진 첨부/조회/삭제를 우선 구현한다.
- 추후 OCR을 도입할 경우 가맹점명, 금액, 날짜 후보를 거래 입력 폼에 자동 채운다.

### 문자 텍스트 기반 자동 입력

- 사용자는 카드 승인 문자, 은행 입출금 문자 등의 텍스트를 붙여넣어 거래를 자동 생성할 수 있다.
- 시스템은 문자 내용에서 날짜, 금액, 가맹점/내용, 카드명 또는 계좌명 후보를 추출한다.
- 추출 결과는 즉시 저장하지 않고 입력 확인 화면에서 사용자가 수정/확정한다.
- 문자 패턴은 카드사/은행별로 다르므로 정규식 기반 파서와 수동 보정 규칙을 함께 둔다.
- 자동 분류는 가맹점명 키워드와 사용자가 이전에 선택한 카테고리 이력을 기준으로 추천한다.
- 파싱 실패 시 원문을 보존하고 사용자가 직접 입력할 수 있도록 한다.


### 확장 아이디어

- 영수증 OCR 자동 인식
- 문자 복사/붙여넣기로 반자동 등록
- 목표 저축 관리
- 고정 지출 관리

### 1차

## 8. 정책 및 예외 처리

- 거래 금액은 0원 이하로 저장할 수 없다.
- 삭제된 카테고리가 기존 거래에 연결되어 있으면 거래 내역은 유지한다.
- 카드 결제일이 말일보다 큰 경우 해당 월의 마지막 날로 처리한다.
- 결제 계좌 잔액이 부족해도 카드 결제 거래는 생성할 수 있으며, 계좌 잔액은 음수가 될 수 있다.
- 이체 거래는 수입/지출 통계에 기본 포함하지 않는다.
- 카드 대금 결제는 소비 지출로 중복 집계하지 않는다.
