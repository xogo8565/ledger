## 2. 이번 버전 개발 완료

### 할부 그룹 단위 수정 API

- `PUT /api/transactions/installments/{installmentGroupId}` API를 추가했다.
- 같은 `installmentGroupId`를 가진 모든 회차를 한 번에 수정한다.
- 수정 시 기존 각 회차의 자산 반영을 먼저 되돌린 뒤 새 금액/날짜/카테고리/자산 정보를 적용한다.
- 전체 금액은 회차 수로 다시 분배한다.
  - 마지막 회차에 나머지 금액을 반영해 합계 오차를 줄인다.
- 시작일 기준으로 각 회차 거래일을 월 단위로 다시 계산한다.
- 현재 버전에서는 기존 회차 수 변경은 막았다.

### 할부 그룹 단위 삭제 API

- `DELETE /api/transactions/installments/{installmentGroupId}` API를 추가했다.
- 같은 `installmentGroupId`를 가진 모든 회차의 자산 반영을 되돌린 뒤 전체 삭제한다.
- 존재하지 않는 할부 그룹이면 오류를 반환한다.

### 할부 일정 UI

- 할부 일정 화면에 `수정` 버튼을 추가했다.
- `수정`을 누르면 기존 거래 입력 화면을 할부 전체 수정 모드로 재사용한다.
- 할부 전체 수정 모드에서는 첫 회차 정보를 기반으로 폼을 채우고, 금액은 전체 회차 합계로 표시한다.
- 저장 시 단일 거래 수정 API가 아니라 `PUT /api/transactions/installments/{installmentGroupId}`를 호출한다.
- 할부 일정 화면에 `전체 삭제` 버튼을 추가했다.
- 삭제 전 브라우저 확인창을 표시해 전체 회차 삭제 실수를 줄이도록 했다.

- 구현된 MVP 목록에 할부 그룹 단위 수정/삭제를 반영했다.

## 5. 아직 부분 완료인 항목

### 카드 관리

- 카드 결제 예약과 실행은 구현되어 있다.
- 카드 자산 생성/수정 시 카드 프로필 입력과 저장 API 연결은 구현되어 있다.
- 자동 결제 중복 실행 방지는 `PROCESSING` 상태와 낙관적 잠금으로 보강했다.
- 실패 상태 저장과 UI 표시는 구현되어 있다.
- 결제 예정 금액은 다음 결제 예정일에 대응하는 청구 기간 기준으로 산정한다.
- 말일 처리는 구현되어 있다.
- 공휴일/주말 결제일 보정 정책은 아직 미완료다.
- 실패한 카드 결제를 사용자가 다시 예약하거나 재시도하는 UI는 아직 미완료다.

### 할부 관리

- 할부 생성, 조회, 그룹 단위 수정/삭제는 구현되어 있다.
- 그룹 수정 시 기존 회차 수 변경은 아직 미지원이다.
- 카드 결제 예정 금액과 할부 청구 월 연결 검증이 필요하다.

### 예산

- 월별 전체/카테고리 예산 저장은 구현되어 있다.
- 카테고리별 예산 사용률, 초과 표시, 전월 예산 복사는 미완료다.

### 통계

- 월별 및 카테고리별 지출 통계는 구현되어 있다.
- 연별 통계, 기간 필터, 카테고리 선택 후 거래 목록 이동은 미완료다.

### 영수증 첨부

- 신규 거래 첨부와 업로드 API는 구현되어 있다.
- 첨부 이미지 조회, 미리보기, 삭제, 다중 첨부 UI는 미완료다.

### 문자 자동 입력

- 원문 붙여넣기와 후보 추출은 구현되어 있다.
- 파싱 결과를 거래 입력 폼으로 확정 저장하는 흐름은 미완료다.
- 카드사/은행별 파싱 규칙과 카테고리 추천은 고도화가 필요하다.

### 공동 가계부

- Household/Member 도메인은 있다.
- 인증, 초대, 권한, 사용자 전환 UI는 미완료다.
- 인증, 권한, 사용자 전환은 불필요하더라도 자산 명의나 관련 소비 태그로 활용할 구조가 필요하다.

## 8. 개발 메모

- 할부 그룹 수정은 기존 회차 수 유지 조건으로 구현했다. 회차 수 변경은 기존 회차 삭제/재생성 정책이나 이미 결제된 회차 보호 정책을 정한 뒤 확장하는 편이 안전하다.
- 로컬 PATH에서 `npm`, `git`을 찾지 못했고 백엔드에는 Gradle wrapper가 없어 로컬 직접 빌드는 실행하지 못했다.
- 프로젝트 기준 검증 방식인 Docker Compose 빌드는 정상 통과했다.
