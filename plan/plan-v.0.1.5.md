# 편한가계부 신규 개발 진행 기록 v.0.1.5

작성일: 2026-06-26

## 1. 기준

- 이전 기록: `plan/plan-v.0.1.4.md`
- 이번 개발 범위: OCR 품목표 여러 행 메모 요약
- 배경: v0.1.3에서 `품명/단가/수량/금액` 품목표의 첫 품목을 거래 제목 후보로 쓰도록 했지만, 여러 품목 영수증에서는 나머지 품목 정보가 거래 입력 초안에서 사라지는 문제가 있었다.

## 2. 이번 버전 개발 완료

### 품목표 전체 행 추출

- `ImportTextService`의 영수증 품목표 파싱을 첫 행 단건 추출에서 여러 행 리스트 추출로 확장했다.
- 첫 번째 품목은 기존과 동일하게 거래 제목 후보와 합계 누락 시 금액 fallback에 사용한다.
- 이후 품목들도 `ReceiptLineItem` 목록으로 보존해 메모 생성에 활용한다.

### 품목 메모 요약

- 영수증 OCR 메모에 품목 요약을 추가했다.
- 형식은 `품목: 품명 금액원, 품명 금액원...` 형태다.
- 최대 5개 품목까지 메모에 표시하고, 6개 이상이면 `외 N건`으로 축약한다.
- 메모 기본 문구는 `영수증 OCR 자동 입력 후보`로 정리했다.

### 회귀 테스트

- `ImportTextServiceTest`에 여러 품목 행을 포함한 영수증 OCR 텍스트 테스트를 추가했다.
- 검증 항목:
  - 합계 금액을 거래 금액으로 사용한다.
  - 첫 번째 품목명을 거래 제목 후보로 사용한다.
  - 여러 품목이 메모 요약에 포함된다.

## 4. 아직 부분 완료인 항목

### OCR UX

- OCR 실패/낮은 신뢰도 안내는 아직 더 보강할 수 있다.
- 촬영 가이드, 빈 텍스트 안내, 수동 입력 전환 CTA를 더 명확하게 만들 수 있다.

### 품목 상세 UX

- 여러 품목 중 대표 품목을 사용자가 선택하는 UI는 아직 없다.
- 품목별 상세 저장 모델은 아직 검토 전이다.

### 파일 정책

- OCR 분석 중 서버 임시 파일은 요청 처리 후 삭제된다.
- 거래 저장 전 OCR 원본 파일은 프론트의 `File` 객체로만 유지되므로, 장시간 작성 중 이탈/새로고침 정책은 별도 검토가 필요하다.
