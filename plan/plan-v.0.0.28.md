## 2. 이번 버전 개발 완료

### 거래 추가 입력 방식 선택

- 가계부와 통계 화면의 `+` 버튼을 누르면 입력 방식 선택 시트를 표시한다.
- `직접 입력`과 `문자 자동 입력` 두 경로를 제공한다.
- 배경 또는 닫기 버튼을 누르면 선택 시트를 닫을 수 있다.
- 직접 입력을 선택하면 기존 지출 입력 화면으로 이동한다.
- 선택 메뉴는 모바일 하단 시트 형태로 구성했다.

### 클립보드 문자 자동 입력

- 문자 자동 입력을 선택하면 브라우저 클립보드의 텍스트를 읽는다.
- 클립보드 내용을 기존 문자 분석 API로 즉시 전송한다.
- 분석된 거래 유형, 날짜, 금액, 가맹점, 메모, 추천 카테고리를 거래 입력 폼에 자동 반영한다.
- 별도 미리보기 확인 단계를 거치지 않고 사용자가 거래 폼에서 값을 검토·수정하도록 했다.
- 클립보드가 비어 있으면 안내창을 표시하고 입력 방식 선택 메뉴를 유지한다.
- 클립보드 권한이 없거나 읽기에 실패하면 권한 확인 안내창을 표시한다.
- 문자 분석 API가 실패하면 오류 안내창을 표시한다.

### 더보기 화면 정리

- 더보기 화면의 기존 문자 붙여넣기 textarea와 분석 미리보기 UI를 제거했다.
- 문자 자동 입력의 진입점을 거래 추가 버튼으로 통합했다.
- 카테고리 관리, 반복 거래, 영수증, CSV 메뉴는 기존대로 유지했다.

### 브라우저 회귀 테스트

- `+` 버튼에서 입력 방식 선택 메뉴가 열리는지 검증한다.
- 직접 입력 선택 후 기존 거래 생성 흐름이 정상 동작하는지 검증한다.
- 빈 클립보드에서 안내창이 표시되고 선택 메뉴가 유지되는지 검증한다.
- 카드 승인 문자를 클립보드에 넣고 문자 자동 입력을 선택한다.
- 문자 분석 API 호출 성공과 거래 입력 화면 이동을 확인한다.
- 분석 금액 8,900원이 입력 폼에 자동 반영되는지 검증한다.
- Docker 스모크 환경은 보안 컨텍스트가 아니므로 테스트에서 표준 Clipboard API 인터페이스를 주입한다.

### README 정리

- 구현된 거래 입력 항목에 직접 입력/클립보드 자동 입력 선택을 반영했다.
- 문자 자동 입력 설명을 클립보드 기반 흐름으로 최신화했다.
- 다음 작업에서 입력 방식 선택과 더보기 문자 UI 제거 항목을 삭제했다.

## 5. 아직 부분 완료인 항목

### 문자 자동 입력

- 클립보드 문자 분석, 과거 이력·키워드 카테고리 추천, 거래 폼 자동 입력은 구현되어 있다.
- Clipboard API는 브라우저 보안 정책상 HTTPS 또는 localhost 같은 보안 컨텍스트와 사용자 권한이 필요하다.
- 사용자가 파싱·추천 규칙을 직접 관리하는 기능은 아직 미완료다.

### 자산 명의/공동 가계부

- 자산 명의 저장과 표시, Household/Member 도메인은 구현되어 있다.
- 명의별 자산 집계, 거래의 공동/개인 소비 구분, 더보기 명의 관리 메뉴는 아직 미완료다.
- 인증과 권한 인가는 현재 개인용 범위에서 제외한다.

### 영수증 첨부

- 신규/기존 거래의 최대 10장 일괄 첨부, 조회, 미리보기, 개별 삭제는 구현되어 있다.
- 할부 그룹 전체 수정 중 영수증 첨부 정책과 UI는 아직 미완료다.

### 기간 필터

- 기간 조회, 목록 필터, 결과 합계, 요약, CSV 내보내기는 구현되어 있다.
- 기간 필터 기반 통계는 아직 미완료다.

## 8. 개발 메모

- 문자 자동 입력은 클립보드 읽기 직후 기존 `/api/import/text/parse` API를 재사용한다.
- 더보기의 별도 문자 상태와 미리보기 상태를 제거해 문자 입력 흐름을 하나로 통합했다.
- 실제 서비스 주소 `http://localhost:8081`은 Chromium에서 보안 컨텍스트로 취급되지만 원격 HTTP 주소에서는 Clipboard API가 제한될 수 있다.
- Docker 브라우저 테스트의 `host.docker.internal`에는 테스트용 Clipboard API를 주입해 동일 사용자 흐름을 검증한다.
- 브라우저 스모크 테스트 후 Docker Compose 스택은 실행 상태로 유지되어 있다.
