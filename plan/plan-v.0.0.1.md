# 가계부 앱 - 진행 상황 보고서 (v.0.0.1)

## 1. 프로젝트 개요

### 앱 목적
개인 사용을 목적으로 하는 모바일 가계부 앱 개발

### 핵심 가치
- 수입, 지출, 이체를 빠르게 기록
- 자산별 잔액과 부채를 한 화면에서 확인
- 카드 지출 관리 (결제 예정 금액 vs 미결제 금액)
- 월별 예산 관리 및 통계
- 월별, 카테고리별 소비 흐름 분석

## 4. 기술 스택 및 구성

### 백엔드
- Java 17+
- Spring Boot 3.x
- Gradle 빌드 시스템
- MySQL 8.0
- REST API 구조

### 프론트엔드
- React 18+
- Vite (빌드 도구)
- CSS3 (반응형 디자인)
- 상태 관리: 미정 (Context API 또는 Redux)

### 인프라
- Docker & Docker Compose
- Nginx (리버스 프록시/정적 파일 서빙)
- MySQL 자동 백업

### 핵심 기술 구현

**거래 수정/삭제 시 데이터 무결성 보장**
```
거래 수정 흐름:
1. 기존 거래 찾기
2. 기존 거래의 자산 변화 역계산 (reverseAssetChange)
3. 새로운 데이터로 거래 업데이트
4. 새로운 자산 변화 적용 (applyAssetChange)
5. 저장

거래 삭제 흐름:
1. 거래 찾기
2. 자산 변화 역계산하여 복구 (reverseAssetChange)
3. 거래 삭제
4. 저장
```

**카드 지출 처리 (버그 수정)**
- 문제: 카드 지출 시 카드 잔액이 음수가 되는 버그
- 해결: AssetType.CARD인 경우 지출 시 잔액 변경 방지
- 결과: 카드는 미결제 금액만 추적, 잔액은 0으로 유지

**청구 기간 계산 (월말 대응)**
- 2월 같은 짧은 달에서도 정확한 청구 기간 계산
- Math.min(statementClosingDay, lengthOfMonth()) 사용
