# 편한가계부 신규 개발 진행 기록 v.0.1.9

작성일: 2026-06-26

## 1. 기준

- 이전 기록: `plan/plan-v.0.1.8.md`
- 이번 개발 범위: OCR 날짜 후보 선택 UI
- 배경: OCR 결과 후보 선택 UI에서 제목/품명과 금액은 선택 가능해졌지만, 영수증 날짜가 여러 형식으로 인식되거나 잘못 선택된 경우 사용자가 거래일 후보를 선택할 수 없었다.

## 2. 이번 버전 개발 완료

### 날짜 후보 추출

- OCR 원문에서 날짜 후보를 추출하는 프론트 헬퍼를 추가했다.
- 지원 형식:
  - `YYYY-MM-DD`
  - `YYYY.MM.DD`
  - `YYYY/MM/DD`
  - `MM/DD`
  - `MM월 DD일`
- 연도가 없는 날짜는 현재 연도를 붙여 `YYYY-MM-DD` 형식으로 정규화한다.
- 유효하지 않은 월/일 조합은 후보에서 제외한다.

### 날짜 후보 선택 UI

- OCR 결과 화면의 `OCR 후보 선택` 카드에 날짜 후보 그룹을 추가했다.
- 현재 파싱된 거래일을 우선 후보로 표시한다.
- 사용자가 날짜 후보 칩을 선택하면 `preview.transactionDate`가 갱신된다.
- `거래 입력에 사용`을 누르면 선택된 날짜가 거래 입력 초안에 반영된다.

## 4. 아직 부분 완료인 항목

### 후보 추출 위치

- 현재 제목/금액/날짜 후보 추출은 프론트에서 OCR 원문을 기준으로 수행한다.
- 서버가 명시적인 후보 DTO를 내려주는 구조는 아직 없다.

### 후보 품질

- 날짜 후보의 연도 보정은 현재 연도 기준이다.
- 연말/연초에 전년도 영수증을 입력하는 경우 추가 보정 규칙이 필요할 수 있다.
