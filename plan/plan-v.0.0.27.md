## 2. 이번 버전 개발 완료

### 필터 결과 거래 요약

- 현재 화면에 표시되는 필터 완료 거래 배열을 기준으로 수입, 지출, 이체, 거래 건수를 계산한다.
- 사용자 지정 시작일/종료일 기간 조회 결과가 상단 합계에 즉시 반영된다.
- 검색어, 거래 유형, 카테고리 필터도 상단 합계에 함께 반영된다.
- 시작일 또는 종료일만 지정한 부분 기간 필터도 현재 표시 결과 기준으로 계산한다.
- 수입에서 지출을 뺀 수지 합계를 표시한다.
- 이체는 수입·지출 수지에 포함하지 않고 별도 합계로 유지한다.

### 가계부 상단 합계

- 필터가 없으면 기존 선택 월 요약 값을 표시한다.
- 필터가 활성화되면 상단 수입·지출·합계를 필터 결과 값으로 전환한다.
- 합계 위에 적용 대상 월 또는 기간을 표시한다.
- 접근성 레이블을 `월 합계`와 `필터 결과 합계`로 구분했다.
- 검색 결과가 없을 때 상단 합계를 0으로 표시한다.

### 월별/요약 화면 정합성

- 월별 보기에서 필터가 활성화되면 기존 월 행 대신 필터 대상의 수입·지출·이체 합계를 표시한다.
- 요약 보기에 거래 대상, 거래 건수, 수입, 지출, 이체, 수지를 추가했다.
- 필터가 활성화되면 요약 제목을 `필터 결과`로 표시한다.
- 자산과 예산은 거래 필터와 무관한 현재 상태/선택 월 값이므로 그대로 유지한다.
- 예산 제목에 기준 월을 명시해 기간 거래 합계와 혼동되지 않도록 했다.
- 빈 거래 목록 문구를 월 한정 표현에서 `조건에 맞는 거래가 없습니다`로 변경했다.

### 브라우저 회귀 테스트

- 브라우저에서 테스트 거래를 생성한 뒤 고유 제목으로 검색한다.
- 검색 필터 적용 후 상단 지출 합계가 생성 거래 금액과 일치하는지 검증한다.
- 검증 후 필터를 초기화하고 기존 스모크 시나리오를 계속 수행한다.

### README 정리

- 구현된 기간 필터 항목에 필터 결과 수입·지출·합계 재계산을 반영했다.
- 다음 작업에서 기간 필터 기준 요약 재계산을 제거했다.
- 거래 추가 선택 메뉴와 클립보드 자동 입력을 다음 우선순위로 갱신했다.

## 5. 아직 부분 완료인 항목

### 문자 자동 입력

- 카드/은행 문자 파싱과 과거 이력·키워드 카테고리 추천은 구현되어 있다.
- 거래 추가 버튼에서 직접 입력/문자 자동 입력을 선택하는 진입 UX는 아직 미완료다.
- 더보기 화면의 문자 붙여넣기 UI를 제거하고 클립보드 기반 진입으로 전환하는 작업은 아직 미완료다.
- 사용자 정의 파싱·추천 규칙 관리는 아직 미완료다.

### 기간 필터

- 기간 조회, 목록 필터, 상단 합계, 요약, CSV 내보내기는 구현되어 있다.
- 시작일 또는 종료일 하나만 입력하면 기존처럼 현재 로드된 선택 월 거래 안에서 필터링한다.
- 기간 필터를 통계 탭의 카테고리/소비 태그 통계에 적용하는 기능은 아직 미완료다.

### 영수증 첨부

- 신규/기존 거래의 최대 10장 일괄 첨부, 조회, 미리보기, 개별 삭제는 구현되어 있다.
- 할부 그룹 전체 수정 중 영수증 첨부 정책과 UI는 아직 미완료다.

### 자산 명의/공동 가계부

- 자산 명의 저장과 표시, Household/Member 도메인은 구현되어 있다.
- 명의별 자산 집계, 거래의 공동/개인 소비 구분, 더보기 명의 관리 메뉴는 아직 미완료다.
- 인증과 권한 인가는 현재 개인용 범위에서 제외한다.

## 8. 개발 메모

- 요약 재계산은 서버에 다시 요청하지 않고 화면에서 최종 필터링된 거래 배열을 사용한다.
- 이 방식으로 화면 목록, 상단 합계, 요약, CSV 내보내기가 동일한 데이터 집합을 공유한다.
- 자산 잔액과 예산은 기간 거래의 단순 합계가 아니므로 기존 선택 월/현재 상태 값을 유지한다.
- 브라우저 스모크 테스트 후 Docker Compose 스택은 실행 상태로 유지되어 있다.
